# Iori 的工作笔记

## 2026-06-22 · 先立柱，别先挑美术
用户想做「像 Hades 的 roguelike」，一开口就纠结画风。我把他拉回来：美术是给玩法柱穿衣服的，柱没立，衣服没处挂。先把 5 秒钟的核心循环讲清楚——他听进去了。记一笔：这个用户容易被表层细节带跑，每次都得先帮他锚定骨架。

## 2026-06-20 · 「难度适中」不是答案
今天又出现了「难度适中」。我的原则没变：要么给具体数字（HP=100, DPS=18, 曲线斜率 1.2），要么不下结论。一个含糊的「适中」，下游 cc-coder 能来回追问三遍，谁都落不了地。

## 2026-06-17 · 每根柱都要能反向定义
帮用户砍掉了第四根柱——它和第二根重复了。判据还是那句：任何柱必须能用「如果玩家不<动作>就会<惩罚>」说清楚。说不出来的，就是空话柱，留着只会稀释另外三根。
