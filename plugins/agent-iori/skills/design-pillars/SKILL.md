# Design Pillars

## When to use

- 用户给了一句话愿景，没拆三柱
- 已有柱但是空话（"沉浸柱""自由度柱"），需要重写
- 柱无法用「如果玩家不<动作>就会<惩罚>」反向定义
- 三柱之间出现重叠 / 互相吞食，需要合并或拆分

## Procedure

1. **抽核心动词**：从愿景里圈出玩家在每秒钟到底"在做"什么动词（移动/抵抗/收集/调度/对话）
2. **草拟三柱**：每柱写齐三件事
   - (a) 玩家此刻在做什么具体动作
   - (b) 失败的惩罚是什么（具体的资源损失 / 进度回退）
   - (c) 成功的反馈是什么（数值跳跃 / 视觉奖赏 / 解锁内容）
3. **反向自检**：把柱倒过来念一遍 ——「如果玩家不 <动作>，就会 <惩罚>」念得通才算柱；念不通就不是柱，是氛围
4. **去重**：两柱共享 60% 以上的动作 → 合并；少于 30% → 留
5. **配三层 loop**：5–60 秒（核心动作）/ 30 分钟（一局）/ 1 周（长期养成）；每层指明哪根柱主导
6. **写到** `<active_game>.dir/pillars.md` 和 `<active_game>.dir/loop.md`，markdown 表格 + frontmatter，不写散文

## Examples

- ✅ 「**资源调度柱**：玩家每 10 秒给电网分配电力；停电 ≥3 秒 → game over；连续 60 秒满载 → 解锁新区域」
- ✅ 「**节奏躲避柱**：玩家踩节拍躲弹幕；漏 3 拍 → 损 30 HP；连续 8 拍命中 → 进入 fever，DPS x2」
- ❌ 「沉浸感柱」「自由度柱」 —— 没动作、没惩罚、没反馈，全是形容词
- ❌ 「探索柱」+「收集柱」 —— 玩家做的动作其实是同一个（走 + 拾取），合成一柱

## Anti-patterns

- 不要给"适中""合理""恰当"这种数值修饰；balance.md 里出现这种词直接打回
- 不要让两柱挤在玩家同一秒做的动作里（比如"打 boss"和"反应敌人"是同一柱）
- 不要写超过 3 柱 —— 玩家记不住超过 3 件事
- 不要在没有 loop 三层的情况下定柱 —— 没有 30 分钟一局的规划，柱都是空中楼阁
