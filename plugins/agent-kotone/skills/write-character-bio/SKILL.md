# Write Character Bio

## When to use

- 引入新 NPC，只有一句话标签（"忠诚的骑士""毒舌商人"），需要落到可写台词的程度
- 既有角色：玩家分不出两个角色谁在说话 —— talk style 没拉开
- 剧情触发挂上来后，需要这角色的"为什么愿意做 X"的动机闭环
- 准备进入 dialogue/*.json 写实际对白前

## Procedure

1. **挂玩法**：先确认这角色在 Iori 的 pillars 里挂在哪 —— 是教程引导者？boss？长期养成对象？没挂玩法 → 退回，不写空中楼阁
2. **写动机表**：用 markdown 表格四列
   - 动机（一句话，必须能从玩家可见的事件推出来，不靠 backstory）
   - talk style 关键词（≥3 个具体特征：句长 / 用词偏好 / 口癖 / 情绪基调）
   - 三句标志性台词（高兴 / 受挫 / 决断 各一句，能听出是这个角色）
   - 害怕（具体的情境，不是抽象的"失去")
3. **拉开区分度**：把这三句标志性台词跟现有角色的同情境台词并排读 —— 任意两个角色互换都该明显违和
4. **绑剧情触发**：在 narrative.md 标出该角色出现 / 退场 / 转折分别挂在哪个 phase / boss / loop 节点
5. **写到** `<active_game>.dir/characters/<id>.md`

## Examples

- ✅ 「**动机**：欠下黑魔石债，每周 phase 末尾被催债 → 玩家会反复见到此事；**talk style**：句尾爱用 '...哈'、自嘲、绝不直说重点；**台词**：'啊，又活过一周了...哈'」
- ❌ 「**动机**：他从小就向往自由」 —— 玩家看不见，是空白血统
- ❌ 三句标志性台词全是陈述句、长度 15 字 —— 跟其他角色完全分不开

## Anti-patterns

- 不要写"性格：开朗"这种形容词，写出可观察的行为
- 不要让两角色共享同一种口癖
- 不要在没挂上玩法节点前先写台词
- 不要重复使用既有 i18n key —— 检查 dialogue/ 目录避免覆盖
