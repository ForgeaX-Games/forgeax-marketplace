# tier1-narrative-full

为**叙事驱动型游戏**提供完整的叙事生产管线配置。

叙事是核心产品力，几乎所有玩法都围绕故事体验设计。

---

## 触发条件

用户说要做以下类型的游戏时触发：

| 用户描述 | 品类 | 预设文件 |
|---------|------|---------|
| 视觉小说 / AVG / 文字冒险 | visual-novel | presets/visual-novel.md |
| 互动叙事 / 互动电影 / 分支故事 | interactive-drama | presets/interactive-drama.md |
| JRPG / 日式 RPG / 回合制 RPG | jrpg | presets/jrpg.md |
| CRPG / WRPG / 西式 RPG / 博德之门式 | crpg | presets/crpg.md |
| 开放世界 RPG / 仙侠 RPG / 武侠 RPG | open-world-rpg | presets/open-world-rpg.md |

**不触发**：用户说"我要做动作游戏"、"我要做塔防"、"叙事不重要"。

---

## 管线策略：完整启动

```
Phase 1 → Phase 2 → Phase 4 → Phase 5
创意策划  完整大纲   内容生成   打磨优化
```

所有阶段全量执行。见 `pipeline-config.md` 了解各阶段具体配置。

## 执行模式

| 情况 | 模式 | 做什么 |
|-----|------|-------|
| 用户给了基本想法 | **完整生成** | 识别品类 → 读品类预设 → 按完整管线生成 |
| 用户有主题方向 | **定向生成** | 品类预设 + 用户方向融合，保留用户核心设定 |
| 用户有完整设定 | **结构化输出** | 100% 保留用户设定，补充缺失模块，结构化整理 |
| 用户要修改局部 | **节点级迭代** | 精确定位修改范围，最小化影响面重新生成 |

## Agency 规划（Tier 1 专有）

Tier 1 游戏必须规划 Agency 选择点，明确区分两种类型：

| Agency 类型 | 描述 | 推荐密度 |
|------------|------|---------|
| **Expressive Agency** | 风格化选择，不改变世界状态（语气/态度） | 每场景 1-3 个 |
| **Branching Agency** | 改变世界状态，需要全局追踪 | 每章 ≤ 2-3 个 |

**必须规划收敛点**：防止分支无限膨胀，控制内容量。

---

## 数据卫生规则（Tier 1 专有）

内容生产前必须确认：
- 对话节点 ID 命名规范：`{chapter}_{scene}_{node_index}`
- 角色 ID 规范：`char_{name_pinyin}`
- Flag/变量规范：`flag_{category}_{name}`
- 本地化字段预留
- 导出格式确认（JSON / Markdown）
