# 互动叙事 / 互动电影叙事预设

## 品类特征

| 维度 | 详情 |
|------|------|
| **核心体验** | 玩家是导演，每个选择都在"写"故事 |
| **Agency 类型** | Branching Agency 为主，选择即叙事 |
| **内容形式** | 电影级演出 + 分支选择 + 道德困境 |
| **代表作** | 底特律：变人、隐形守护者、奇异人生、直到黎明 |

---

## 叙事结构核心：蝴蝶效应网络

互动叙事的关键是**选择的重量感**：玩家应感受到每个选择都有真实后果。

```
选择点 A ─┬─ 路径A1 ─┐
           │          ├─ 收敛点1 ─┬─ 选择点B ─┐
           └─ 路径A2 ─┘           │           ├─ ...
                                  └─ 路径直通 ─┘
```

**设计原则**：
- 每个重大选择必须在后续剧情中**被提及/产生后果**
- 收敛点不等于"选择无意义"——保留选择的**痕迹差异**
- 避免"选A和选B最终完全一样"的假分支

---

## 核心叙事需求

### 剧情结构 (S) ★★★
- 叙事弧线：角色 vs 道德困境 vs 系统性压迫
- 主题必须有**争议性**（对错不明确，玩家各有解读）
- 结局设计：建议 3-6 个主要结局，通过变量累积触发

### 角色设计 (C) ★★★
- 主角：玩家代入载体，**道德立场模糊**或由玩家决定
- 关键 NPC：每个都有自己的利益和动机，非单纯好/坏
- 角色关系：会因选择变化（信任值 / 好感度 / 存活状态）

### 对话剧本 (D) ★★★
- 时间压力选择（限时）：强化紧迫感
- 沉默也是选择：允许不选，有对应后果
- 分支对话后必须有**状态差异反馈**（角色反应/表情/后续引用）

### 环境叙事 (E) ★★★
- 环境细节传达角色过去和世界状态
- 探索触发隐藏信息（增强沉浸）

## Agency 设计：道德困境模板

```
情境设定：
[描述一个没有完美答案的两难局面]

选项A：[牺牲个体保护群体]
  → 立即后果：[角色X感激/愤怒]
  → 延迟后果：[章节Y中影响Z]

选项B：[保护个体放弃群体]
  → 立即后果：[角色Y信任度+1，角色Z信任度-1]
  → 延迟后果：[章节W中角色Z不再支持你]

选项C：（可选）[第三条路，代价不明]
  → 需要前置条件：[之前做了某件事]
  → 后果：[意想不到的连锁反应]
```

## 变量系统设计

互动叙事必须有变量/Flag系统，驱动分支和结局：

| 变量类型 | 示例 | 影响 |
|---------|------|------|
| 关系变量 | `rel_alex_trust: 0-100` | 角色是否信任你 |
| 道德倾向 | `moral_pragmatic: 0-100` | 务实 vs 理想主义 |
| 存活状态 | `char_alice_alive: bool` | 角色是否在场 |
| 世界状态 | `city_on_fire: bool` | 世界状态变化 |
| 知识变量 | `knows_secret: bool` | 是否掌握关键信息 |

## 结局触发设计模板

```markdown
## 结局：[结局名称]

触发条件：
- rel_alex_trust >= 70 AND
- char_bob_alive == true AND
- moral_pragmatic <= 40

叙事描述：
[用 2-3 句话描述这个结局的核心画面和情感]

主题表达：
[这个结局传达了什么主题/价值观]

与其他结局的区别：
[什么样的选择路径才能到达这里]
```

## 节奏设计

| 场景类型 | 特征 | 节奏控制 |
|---------|------|---------|
| 探索场景 | 收集信息、互动环境 | 缓慢，玩家主导 |
| 对话场景 | 角色互动、关系建立 | 中速，情感投入 |
| 危机场景 | 时间压力、限时选择 | 快速，肾上腺素 |
| 后果揭露 | 展示选择结果 | 变速，情感落地 |

## 体量参考

| 目标体量 | 分支节点 | 对话节点 | 结局数 |
|---------|---------|---------|-------|
| 短篇（1-2小时） | 15-25 个 | 200-500 个 | 3-4 个 |
| 标准（4-6小时） | 40-80 个 | 800-2000 个 | 5-8 个 |
| 大型（10小时+） | 100+个 | 3000+个 | 8-15 个 |

## 质量检查

- [ ] 每个重大选择在后续是否有可见的后果？
- [ ] 是否避免了"假分支"（选什么都一样）？
- [ ] 主题是否具有争议性/多元解读空间？
- [ ] 变量系统是否完整设计（无遗漏 Flag）？
- [ ] 结局触发条件是否合理且可达？
- [ ] 限时选择是否传达了足够的紧迫感？
