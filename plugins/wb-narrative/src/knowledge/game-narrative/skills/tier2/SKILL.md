# tier2-narrative-enhanced

为**叙事增强型游戏**提供标准叙事管线配置。

叙事是重要卖点，但与系统玩法并重，叙事设计必须考虑与游戏系统的接口。

---

## 触发条件

用户说要做以下类型的游戏时触发：

| 用户描述 | 品类 | 预设文件 |
|---------|------|---------|
| 抽卡 RPG / Gacha RPG / 手游 RPG / 原神式 | gacha-rpg | presets/gacha-rpg.md |
| ARPG / 动作 RPG / 魂类 / 暗黑类 | arpg | presets/arpg.md |
| MMORPG / 大型多人在线 / MMO | mmorpg | presets/mmorpg.md |
| 战棋 RPG / SRPG / 回合制战棋 / 火焰纹章式 | srpg | presets/srpg.md |
| 模拟经营 / 经营类 / SIM / 种菜/开店 | simulation | presets/simulation.md |

**不触发**：视觉小说、互动叙事、JRPG、CRPG（用 Tier 1）；卡牌/塔防（用 Tier 3）；超休闲（用 Tier 4）。

---

## 管线策略：标准启动

```
Phase 1 → Phase 2 → Phase 4
 创意策划    完整大纲    内容生成
（Phase 5 按需可选）
```

Phase 5 打磨优化只在全量内容生产完毕后按需执行。

## 叙事内容需求矩阵

| 内容类型 | 需求等级 | 说明 |
|---------|---------|------|
| W 世界观 | ★★★ 核心必需 | 模块化世界观，支持持续扩展 |
| C 角色设计 | ★★★ 核心必需 | 重角色人设，驱动系统 |
| S 剧情结构 | ★★ 重要推荐 | 主线弱于 Tier 1，支撑系统运营 |
| D 对话剧本 | ★★ 重要推荐 | 章节制剧情，活动剧情 |
| Q 支线任务 | ★★ 重要推荐 | 角色个人线，日常任务 |
| E 环境叙事 | ★★ 重要推荐 | 因品类而异，ARPG/开放世界需求高 |
| I 物品叙事 | ★★ 重要推荐 | 装备/道具/角色技能 Lore |
| U UI文案 | ★★★ 核心必需 | 系统界面文案、技能名/描述 |
| L Lore 碎片 | ★★ 重要推荐 | 图鉴、档案系统 |

## 执行模式

| 情况 | 模式 | 做什么 |
|-----|------|-------|
| 用户给了游戏基本设想 | **协同生成** | 叙事与系统接口同步设计 |
| 用户有世界观/角色设定 | **扩展生成** | 在现有设定基础上完善叙事系统 |
| 用户需要运营内容 | **活动叙事模板** | 生成可复用的活动叙事框架 |
| 用户要更新章节 | **章节级迭代** | 新章节叙事，保持世界观一致性 |

---

## 系统叙事接口（Tier 2 专有）

Tier 2 游戏叙事必须与游戏系统深度绑定：

| 系统 | 叙事接口 | 设计要求 |
|-----|---------|---------|
| 角色/英雄系统 | 角色人设 + 语音台本 | 每角色需独立人设，语音风格文档 |
| 活动/赛季系统 | 活动剧情模板 | 短平快，独立完整，不影响主线 |
| 任务系统 | 任务叙事挂钩 | 任务目标有叙事理由，非"杀X个怪" |
| 成就/收集系统 | 文案 + Lore | 成就名称/描述有叙事风味 |
| 世界/地图系统 | 地区历史 + 事件 | 每地区有背景故事 |

## 长线运营叙事结构（适用 Gacha / MMORPG）

```
[世界观核心]（固定，不可变）
       │
  ┌────┴────┐
主线版本剧情  活动剧情
（季度更新）  （月度更新）
       │
  分支到角色个人剧情
```

设计要求：
- 世界观核心建立后不可轻易更改（影响所有内容）
- 版本主线推进世界历史
- 活动剧情独立完整，可单独体验
