# Tier 2 管线配置

> 叙事增强型游戏（40-70%）标准管线配置参考。

---

## 管线执行策略：Standard Pipeline

```
Phase 1 → Phase 2 → Phase 4
 创意策划    完整大纲    内容生成
（Phase 5 按需执行）
```

---

## Phase 1：创意策划（与系统协同）

**与 Tier 1 的差异**：
- High Concept 需要包含"叙事 × 系统"双卖点
- 规模预估必须考虑**运营周期**（日运营 / 版本制）
- 题材选择需评估与系统玩法的兼容性

**输出步骤**：
- `core_concept`：叙事 + 系统双卖点
- `system_architecture`：品类 + 叙事强度 + 运营模式
- `value_framework`：叙事支柱（聚焦角色 + 世界）
- `design_doc`：内容规划（主线章节数 + 活动频次）

---

## Phase 2：系统化叙事设计

**输出步骤**：
- `worldview_generation`：模块化世界观
- `character_generation`：角色档案 + 系统接口（语音台本框架）
- `story_framework`：主线叙事框架（章节制 / 版本制）
- `outline_batch`：阶段性内容规划（主线 + 活动）

**Tier 2 特殊要求**：
- `worldview_generation` 必须明确**扩展锚点**（可添加新地区/角色/故事线）
- `character_generation` 包含**UI文案锚点**（技能名/描述风格指南）
- `story_framework` 分离**主线框架**和**运营内容框架**
- Agency 规划简化为主要剧情节点，无需精细到 Expressive

---

## Phase 4：内容生成（系统协同）

**执行策略**：

| 内容模块 | 步骤 | 优先级 |
|---------|------|-------|
| 主线叙事 | `scene_generation` Layer 1-2 | 必须 |
| 角色剧情 | `character_generation` 详细版 | 必须 |
| 任务叙事 | `quest_generation` | 必须 |
| UI文案 | 独立输出 | 必须 |
| 活动叙事模板 | 模板化生成 | 推荐 |
| 环境叙事 | `scene_generation` 环境描写 | 按品类 |
| 物品 Lore | 批量生成 | 推荐 |

**Tier 2 特殊要求**：
- 优先生成**可上线内容**（主线 + 核心角色 + UI文案）
- 活动叙事用**模板化**方式，降低维护成本
- 物品 Lore 批量生成，保持风格统一

---

## 各品类 Phase 4 重点

| 品类 | Layer 1 重点 | Layer 2 重点 | 特殊需求 |
|-----|-------------|-------------|---------|
| 抽卡 RPG | 主线章节骨架 + 角色登场 | 角色个人剧情 + 活动剧情 | UI文案（技能名/突破文案） |
| ARPG | 主线章节 + 地区叙事骨架 | 环境叙事 + 物品 Lore | BOSS 叙事（与战斗绑定） |
| MMORPG | 版本主线骨架 + 地区设计 | 任务叙事 + 日常任务 | 公会叙事模板 |
| 战棋 RPG | 章节战役叙事 + 角色弧光 | 战斗前后对话 | 战棋特有 Agency（战术选择）|
| 模拟经营 | 经营叙事框架 + NPC 性格 | 日常对话 + 事件叙事 | 长线事件系统 |

---

## 节点级重跑配置

| 修改范围 | 影响评估 |
|---------|---------|
| 修改世界观核心 | 影响所有角色/地区/任务 → 建议全量重跑 Phase 4 |
| 修改角色人设 | 影响该角色的所有对话/任务 → 节点级重跑 |
| 修改章节/版本主线 | 影响该章节的场景/任务 → 章节级重跑 |
| 修改 UI 文案风格 | 仅影响 UI 文案模块 → 独立重跑 |
| 修改活动叙事模板 | 影响该活动所有内容 → 活动级重跑 |

---

## 体量 × 管线深度

| 目标 | 推荐配置 |
|-----|---------|
| 上线前版本（首发内容） | Phase 1 + 2 + 主线 Phase 4 |
| 版本更新内容 | 章节增量 + 活动叙事 |
| 全量内容（含质量审计） | Phase 1 → 2 → 4 → 5 |
