# 抽卡 RPG 叙事预设

## 品类特征

| 维度 | 详情 |
|------|------|
| **核心驱动** | 角色收集 + 抽卡期待 + 长线运营 |
| **叙事核心** | 角色人设驱动收藏欲，活动剧情保持新鲜感 |
| **世界观** | 模块化，支持无限扩展（新地区/新角色/新故事） |
| **代表作** | 原神、明日方舟、崩坏：星穹铁道、蔚蓝档案 |

---

## 叙事结构：章节制 + 活动制并行

```
[世界观核心]（固定基石）
       │
  ┌────┴────────────────┐
版本主线剧情              角色个人剧情
（推进世界历史）           （驱动角色价值）
       │                      │
活动剧情（月度）          语音台本 + 羁绊故事
（独立完整，短平快）
```

## 角色设计（最高优先级）

抽卡 RPG 的**一切叙事都服务于角色价值**：

### 角色人设档案模板
```markdown
## [角色名] — [职位/身份]

**定位关键词**（3个，驱动玩家印象）：
[形容词1] × [形容词2] × [形容词3]

**人设核心**：
- 表面形象：[第一印象]
- 深层性格：[深入了解后的真实面]
- 行为驱动：[Why this character does what they do]

**技能主题叙事**：
- 技能1名称：[XX之剑/星光降临...]（有意象感）
- 技能2名称：[...]
- 技能主动效果的叙事诠释：

**语音台本风格**：
- 常规语气：[轻松/严肃/傲娇/温柔...]
- 口头禅/特殊习惯：
- 禁忌话题反应：
- 与主角亲密度提升时的变化：

**出场叙事钩子**：
[一句话概述这个角色的登场场景和第一印象钩子]
```

## 世界观：模块化设计

| 模块 | 要求 |
|-----|------|
| **核心规则** | 一次定型，不可轻易改动（魔法体系/科技规则/种族关系） |
| **地区模块** | 每个地区独立历史，可随版本解锁 |
| **角色子世界** | 每个主要角色背后有一个迷你世界观 |
| **扩展锚点** | 设计明确的"未知区域/待解谜团"供后续版本填充 |

---

## 叙事内容类型 × 优先级

| 内容类型 | 优先级 | 说明 |
|---------|-------|------|
| 角色人设 + 语音台本 | P0 最高 | 驱动抽卡欲望 |
| 主线章节剧情 | P1 必须 | 建立世界观，吸引玩家 |
| 角色个人剧情/羁绊故事 | P1 必须 | 深化角色价值 |
| UI文案（技能/突破/档案） | P1 必须 | 直接影响游戏体验 |
| 活动叙事 | P2 重要 | 保持长线内容节奏 |
| 任务/委托叙事 | P2 重要 | 探索世界动力 |
| 物品/道具 Lore | P3 推荐 | 深度内容，粉丝向 |

## UI 文案规范（Tier 2 核心需求）

### 技能文案模板
```
技能名：[2-5字，有意象感，避免普通描述性词语]
描述（功能）：[XX对单个/群体敌人造成XX%物理/魔法伤害]
风味文案（可选）：[1-2句有叙事感的描述，体现角色特质]
```

### 突破/觉醒文案模板
```
[角色名]完成了XX次突破。
曾经的[状态描述]，如今的[变化描述]。
[角色与玩家的关系变化，情感层面的感谢或认可]
```
