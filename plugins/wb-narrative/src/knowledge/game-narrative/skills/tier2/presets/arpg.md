# ARPG / 动作 RPG 叙事预设

## 品类特征

| 维度 | 详情 |
|------|------|
| **核心体验** | 战斗即叙事——每次战斗都在讲故事 |
| **叙事载体** | 装备 Lore、环境叙事、BOSS 传说 > 对话剧情 |
| **叙事密度** | 碎片化、被动触发、玩家主动探索 |
| **代表作** | 暗黑之魂、艾尔登法环、巫师3、哈迪斯、暗黑破坏神4 |

---

## 叙事核心原则：碎片考古

ARPG 的叙事不是"讲给玩家听"，而是让玩家**自己发现和拼凑**。

```
标准叙事：NPC 告诉你"这个世界发生了XX"
ARPG叙事：玩家在废墟中发现：
  - 装备描述："这把剑属于最后的守护者..."
  - 环境：一具尸骸旁有半张求救信
  - BOSS 名字：[堕落之王 XXX] — 暗示了什么？
```

## 核心叙事需求

### 环境叙事 (E) ★★★（ARPG 核心）
- 场景考古：废墟/战场/遗迹背后的故事
- 视觉叙事：建筑风格/植被/尸骸排列传递历史
- 环境道具：信件/笔记/日记（短小精悍）
- 每个 Sub-区域有独立的叙事背景

### 物品叙事 (I) ★★★（ARPG 核心）
- 每件命名武器/防具有 Lore 文字（50-150字）
- Lore 从物品视角讲述世界历史
- 不同装备的 Lore 拼凑完整历史线
- 稀有装备的 Lore 应揭露世界核心秘密

### Lore 碎片 (L) ★★★
- 图鉴/典籍系统（可选择读/不读）
- 环境触发的隐藏日志
- NPC 的只言片语（非完整对话，隐晦暗示）

### 世界观 (W) ★★★
- **深度 Lore，但碎片化呈现**
- 世界史是隐藏信息，等待玩家拼凑
- 神明/古代文明/诅咒等主题

### 角色设计 (C) ★★
- **BOSS 是最重要的角色**（比 NPC 更重要）
- BOSS 设计需要背景故事和堕落原因
- 主要 NPC 少而精，不需要完整弧光

## BOSS 叙事设计模板

```markdown
## [BOSS 名称] — [头衔/称号]

**来历**：[他/她/它是谁，曾经是什么]
**堕落原因**：[什么让他走到今天这一步]
**与世界的关系**：[他在整体世界史中扮演的角色]
**战斗主题**：[这场战斗在叙事上象征什么]
**击败后的意义**：[玩家杀死他意味着什么]

**相关物品 Lore**（2-3件）：
- [物品1]：[揭露 BOSS 过去的某个片段]
- [物品2]：[揭露 BOSS 与世界关系的某个片段]

**关联环境叙事**：
[玩家抵达 BOSS 房间前会看到什么场景，暗示什么]
```

## 装备 Lore 写作模板

```markdown
**[装备名称]**
[类型]

[50-150字的 Lore 文本]

写作规范：
- 用第三人称，但有叙事视角（曾经的主人/制作者/见证者）
- 揭露世界某个角落的历史片段
- 不直接说明，用暗示和细节
- 好的结尾留白（"他再也没有回来"）
- 与其他装备 Lore 形成拼图关系（可选）
```

## 主线对话设计原则

ARPG 的主线对话要**少而精**：

| 准则 | 说明 |
|-----|------|
| 对话尽量简短 | 玩家在行动流里，长对话打断节奏 |
| 不解释，只暗示 | "你应该知道自己在做什么"的态度 |
| 每句话都有重量 | NPC 说的每句话都有叙事意义 |
| 允许不理解 | 玩家第一次可以不懂，鼓励二周目 |
| 谜语式叙事 | 说一半，留一半 |

## 环境叙事节点类型

| 类型 | 触发方式 | 内容 |
|-----|---------|------|
| 考古现场 | 玩家到达特定位置 | 尸骸/残骸背后的故事 |
| 书/日记 | 拾取/交互 | 角色视角的历史记录 |
| 壁画/碑文 | 观察/扫描 | 神话/历史的图像叙事 |
| NPC 碎言 | 对话触发 | 隐晦的信息暗示 |
| 环境变化 | 探索进度 | 世界对玩家行为的反应 |

## 质量检查

- [ ] BOSS 是否都有背景故事和堕落原因？
- [ ] 不同装备的 Lore 是否形成可拼凑的历史线？
- [ ] 环境叙事是否能独立传达故事（无需 NPC 解释）？
- [ ] 主线对话是否简洁有力（无冗余解释）？
- [ ] 核心世界秘密是否通过碎片叙事隐藏可发现？
