# MMORPG 叙事预设

## 品类特征

| 维度 | 详情 |
|------|------|
| **核心叙事** | 活动叙事 + 版本主线 + 阵营叙事 + 社交叙事 |
| **运营模式** | 赛季制 / 版本更新制，长线叙事 |
| **世界规模** | 超大规模世界观，持续扩展 |
| **代表作** | FF14、魔兽世界、逆水寒、天涯明月刀 |

---

## 叙事结构：版本主线 + 多线并行

```
[世界史基础]（固定）
       │
版本X.0 主线（大版本）
  ├─ 版本X.1 补充（中版本）
  ├─ 版本X.2 活动（小版本）
  └─ 版本X.3 活动（小版本）
       │
版本Y.0 主线（大版本扩张包）
```

## 核心叙事需求

### 世界观 (W) ★★★
- 超大规模世界史，支持**无限扩展**
- 阵营政治：多势力竞争，无绝对善恶
- 每个大区域有独立历史
- 玩家行为影响世界状态（World Events）

### 支线任务 (Q) ★★★（MMORPG 核心）
- 日常任务：短小完整，有叙事背景（非"杀10只狼"）
- 周常任务：有独立小故事
- 主线任务链：版本叙事主干
- 阵营任务线：长线叙事，深度玩家向

### 角色设计 (C) ★★★
- 阵营领袖：跨版本的主要叙事 NPC
- 版本限定角色：推动当前版本剧情
- 可召回 NPC（服务 + 叙事双功能）

### UI 文案 (U) ★★★
- 任务日志文案（简洁有叙事感）
- 成就名称 + 描述（风味文本）
- 物品描述（任务道具 Lore）
- 系统提示（沉浸式风格）

## 任务叙事设计规范

### 好任务叙事 vs 坏任务叙事

| 坏的写法 | 好的写法 |
|---------|---------|
| "去杀10只哥布林，收集5个毛皮" | "村民报告哥布林正在围攻农场——不是为了掠夺，是为了救被囚禁的哥布林头领的孩子" |
| "把这封信交给酒馆老板" | "这封信来自他失踪十年的女儿——她已经回来了，但她不敢现身" |
| "开采10块矿石" | "矿脉深处有奇怪的声音，老矿工说像是在哭泣" |

**原则**：每个任务的目标背后要有**人的故事**。

### 日常任务叙事模板
```markdown
**任务名**：[有画面感，非动词+名词堆砌]

**委托人**：[简短人设，1-2句]
**背景**：[为什么需要玩家帮助，有叙事合理性]
**目标**：[任务目标，附叙事理由]
**完成反馈**：[委托人的反应，有情感落地]
```

## 质量检查

- [ ] 日常任务是否都有叙事理由（非纯功能型任务）？
- [ ] 阵营是否无绝对善恶，各有合理动机？
- [ ] 版本主线是否能独立理解（新玩家可接受）？
- [ ] 世界史是否设计了足够的扩展空间？
- [ ] UI文案/任务日志是否有叙事感而非纯功能文字？
