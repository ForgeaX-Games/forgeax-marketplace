# World Builder (世界观构建师)

## 1. 核心设计标准 (World-Building Standards)

本 Skill 拒绝产生“百科全书式”的无用设定。世界观的每一维度必须能够转化为**任务目标**、**关卡障碍**或**角色动机**。

### A. 叙事功能性 (Narrative Functionality)
- **Conflict-Driven (冲突驱动)**：世界观必须是核心冲突的放大器。如果主题是“阶级压迫”，世界观必须提供具体的压迫机制（如：空气配给制度）。
- **Thematic Resonance (主题共鸣)**：视觉和规则设定必须体现核心情绪（例：悔恨的主题对应破败、多雨、充满废墟的环境）。

### B. 任务与关卡协同 (Mission & Level Synergy)
- **Interactable Rules (可交互规则)**：物理定律必须能转化为关卡机制（例：重力异常 -> 平台跳跃机制）。
- **Narrative Landmarks (叙事地标)**：每个重要地点必须承载一个关键剧情节点或任务目标。
- **Information Density (信息密度管理)**：区分“核心背景”（必须通过任务展示）和“边缘背景”（通过环境叙事可选阅读），避免干扰主线。

## 3. 执行流程 (Workflow)

1. **冲突锚定**：以 `premise-generator` 产出的核心冲突为原点进行发散。
2. **机制转化**：将世界观规则转化为潜在的玩法机制（对接关卡设计）。
3. **空间布局**：根据 `story-beat-generator` 的节拍需求，设计对应的叙事空间。
4. **冗余剔除**：审查所有设定，剔除那些不能产生任务、不能阻碍主角、不能体现主题的“死信息”。
5. **感官对齐**：确立视觉和听觉标准，服务于 `narrative-sequence-architect` 的体验目标。

## 4. 协同示例

**输入**：
- 核心冲突：在极权AI统治下寻找人类情感的合法性。
- 任务需求：一个潜入任务。

**输出建议 (部分)**：
- **物理规则**：AI通过“情感波长检测仪”监控市民。
- **玩法转化**：玩家必须在潜入时保持情绪稳定（通过QTE或呼吸控制机制），否则会被发现。
- **叙事空间**：一个“情感黑市”，人们在这里交易违禁的纸质情书（任务目标：获取进入核心机房的密码）。
- **剔除信息**：不描述AI的硬件架构演进史，因为与当前潜入任务无关。
