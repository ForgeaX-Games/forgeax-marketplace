# Premise Generator (游戏前提生成器)

## 1. 核心设计标准 (Premise Standards)

一个高效的游戏前提（Premise）不仅仅是故事大纲，它是**叙事动力学**的方程式。必须满足以下“四位一体”标准：

### A. 叙事方程式 (The Equation)
`[一个有缺陷的主角] + [一个迫切的目标] + [一个不可逾越的障碍] = [戏剧性冲突]`
- **缺陷 (Flaw/Ghost)**：主角内在的不完整性，为后续成长提供空间。
- **迫切性 (Urgency)**：为什么目标必须现在达成？（时间压力或生存威胁）。

### B. 玩法与叙事合一 (Ludonarrative Core)
- **核心循环对齐**：前提中的冲突必须能转化为游戏的核心玩法动作（例：如果玩法是射击，冲突必须是物理对抗而非纯辩论）。
- **玩家代理感 (Agency)**：前提必须暗示玩家的行为将如何影响冲突的结果。

### C. 钩子与悬念 (The Hook)
- **Dramatic Question (戏剧性问题)**：必须是一个让玩家想通过“玩”来回答的问题（例：“他能逃出来吗？”、“他会为了真相牺牲灵魂吗？”）。


---

## 3. 执行流程 (Workflow)

1. **元素提取**：从用户输入的类型、情绪和关键词中提取核心冲突点。
2. **构建方程式**：填充主角、目标、障碍三个变量。
3. **玩法校验**：检查该冲突是否能支撑起 2-10 小时的游戏玩法循环。
4. **压力测试**：通过设定“筹码（Stakes）”来增强前提的紧迫感。
5. **打磨钩子**：精炼 Logline，确保其具备商业吸引力和叙事深度。

## 4. 使用示例

**输入**：
- 类型：恐怖解谜
- 情绪：孤独、悔恨
- 关键词：深海、无线电、记忆碎片

**输出建议 (部分)**：
- **Logline**: 一名患有失忆症的深海潜水员必须在氧气耗尽前，通过修复破碎的无线电台，听完亡妻留下的最后一条录音，哪怕这意味着他必须面对深渊中由他悔恨幻化出的怪物。
- **核心玩法对齐**: 寻找零件（解谜）与躲避怪物（生存）直接对应于“修复无线电”和“面对悔恨”。
- **戏剧性问题**: 他是否有勇气听完真相，还是选择在深海中永远沉睡？
