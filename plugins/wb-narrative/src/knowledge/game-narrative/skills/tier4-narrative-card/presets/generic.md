# 通用休闲游戏叙事框架

当用户的游戏类型无法匹配到具体品类时，使用此通用框架。

## 组合逻辑

- [主角] 想要 [完成目标]
- 通过 [核心操作] 一步步接近目标
- 失败 = [失败条件] 触发
- 成功 = 目标达成，愿望实现

**故事必须解释**：
- 为什么要做这个操作？（游戏核心玩法的叙事理由）
- 成功后能得到什么？

## 要素库

### 主角类型
| 类型 | 变体示例 |
|-----|---------|
| 可爱动物 | 小猫/小狗/小鸟/小兔/小熊 |
| 小人物 | 小骑士/小精灵/小忍者/小厨师 |
| 拟人物品 | 小方块/小星星/小糖果/小机器人 |

### 目标类型
| 类型 | 变体示例 |
|-----|---------|
| 收集类 | 收集足够的XX/集齐所有XX |
| 到达类 | 到达终点/到达最高处/到达目的地 |
| 得分类 | 获得高分/打破纪录 |
| 生存类 | 坚持足够长时间/活到最后 |
| 解救类 | 救出伙伴/解除封印 |
| 建造类 | 建成XX/完成XX |

### 核心操作
| 类型 | 变体示例 |
|-----|---------|
| 点击类 | 点击/敲击/触碰 |
| 滑动类 | 滑动/拖拽/划线 |
| 长按类 | 长按蓄力/按住不放 |
| 方向类 | 控制方向/左右移动/上下移动 |
| 时机类 | 在正确时机操作 |

### 失败条件
| 类型 | 变体示例 |
|-----|---------|
| 时间类 | 时间耗尽/倒计时结束 |
| 生命类 | 生命值归零/机会用完 |
| 碰撞类 | 碰到障碍/碰到敌人 |
| 掉落类 | 掉出屏幕/掉进陷阱 |
| 资源类 | 资源耗尽/空间不足 |

### 动机类型
| 类型 | 叙事解释 |
|-----|---------|
| 救援 | 救伙伴/救家人/救世界 |
| 回家 | 找妈妈/回到家 |
| 梦想 | 实现梦想/成为最强 |
| 收集 | 集齐所有/完成图鉴 |
| 守护 | 保护重要的东西 |

### 场景递进
| 环境线 | 示例 |
|-------|------|
| 自然线 | 草地→森林→山地→云端 |
| 难度线 | 简单→普通→困难→极限 |
| 区域线 | 第一区→第二区→...→最终区 |

## 使用指南

当无法匹配具体品类时：

1. **提取核心玩法**：用户描述中的核心动作是什么？
2. **确定目标类型**：玩家要达成什么？
3. **明确失败条件**：什么情况下会失败？
4. **选择主角和动机**：用可爱角色+经典动机包装
5. **套用通用公式**：生成叙事

## 组合示例

### 示例1：未知玩法A（假设是某种点击收集类）
- 主角：小松鼠
- 目标：收集足够的松果过冬
- 操作：点击收集松果
- 失败：时间耗尽/被老鹰抓走
- 结局：松果满仓，温暖过冬

### 示例2：未知玩法B（假设是某种躲避生存类）
- 主角：小萤火虫
- 目标：在黑夜中存活到天亮
- 操作：控制方向躲避
- 失败：被蝙蝠吃掉/光芒熄灭
- 结局：迎来黎明，找到伙伴

### 示例3：未知玩法C（假设是某种建造类）
- 主角：小蚂蚁
- 目标：建造蚁穴
- 操作：拖拽放置
- 失败：结构倒塌/资源不足
- 结局：蚁穴建成，蚁后入住
