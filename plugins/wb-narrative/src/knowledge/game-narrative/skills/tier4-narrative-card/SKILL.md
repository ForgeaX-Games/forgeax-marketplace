# tier4-narrative-card

为休闲小游戏生成叙事包装，让游戏有灵魂。

## 触发条件

用户说要做：贪食蛇/三消/跑酷/2048/合成/放置/IO/塔防/切水果/打地鼠/堆高高/音游/解谜/棋牌/模拟经营等休闲游戏。

**不触发**：用户说"不需要叙事"、已有明确主题、或做重叙事游戏（RPG/AVG）。

## 执行模式

| 情况 | 模式 | 做什么 |
|-----|------|-------|
| 首次做游戏 | **生成** | 识别品类 → 读预设 → 组合要素 → 输出 |
| 用户带主题 "僵尸塔防" | **融合** | 按用户方向从要素库选取 |
| 用户带完整设定 | **润色** | 100%保留用户设定，只做结构整理 |
| 用户要修改 "换个主题" | **迭代** | 保持品类，局部调整（见下表） |

**迭代类型**：

| 用户说 | 做什么 |
|-------|-------|
| "换个主题" | 保持品类，换要素组合 |
| "太可爱了" | 保持要素，换语气 |
| "主角换成猫" | 只换指定元素 |
| "第二章呢" | 沿用世界观延伸 |
| "重新来" | 避开上次，全新生成 |

### 未匹配品类的处理

当用户输入的游戏类型无法精确匹配时：

1. **智能匹配**：分析核心玩法特征，匹配最接近的品类
   - 提取核心动词（点击/滑动/躲避/收集/消除...）
   - 找到玩法最相似的品类
   - 使用该品类的框架生成

2. **通用兜底**：如果实在无法匹配，使用 `presets/generic.md`
   - 确保六要素完整

**匹配参考**：
| 核心动词 | 可能匹配的品类 |
|---------|--------------|
| 点击/敲击 | timing/action/rhythm |
| 滑动/拖拽 | puzzle/match3/runner |
| 消除/配对 | match3/connect/bubble |
| 躲避/闪避 | runner/timing/shooter |
| 收集/吃 | snake/io/timing |
| 建造/放置 | tower/idle/simulation |
| 瞄准/发射 | sling/shooter/bubble |

### 预设文件结构

每个预设文件包含：
- **组合逻辑**：这个品类的故事骨架（谁+做什么+失败/成功）
- **要素库**：各类可选元素（角色/目标/场景/机制等）
- **组合示例**：5个不同要素组合参考
- **关卡拓展维度**：变化方向

**生成时**：理解组合逻辑 → 从要素库选取元素填入 → 按「叙事模块」展开成完整故事

## 写作核心（必读）

### 一句话公式

```
[动词] + [角色/对象] + [做什么] + [目标/限制]！
```

**好例子**：
- 控制黑洞，吞掉整座城市，成为最大的那个！
- 切水果得分，但别切到炸弹！
- 三个同色消掉！6点前搞定就能下班！
- 吃水果变长！但别撞到自己的屁股！

**坏例子**：
- ❌ 在时间的长河中，你是文明的引导者...（太文艺）
- ❌ 玩家控制角色收集物品避开障碍（太说明书）

### 故事三段式

故事分三段写，**不加小标题**，自然分段：

**第一段：世界发生了什么**（2-3句）
- 建立背景和世界观
- 制造一个"问题"或"缺失"
- 让读者好奇接下来会怎样

**第二段：你要做什么**（3-5句）
- 介绍角色（你是谁）
- 说明动机（为什么要做）
- 描述行动（怎么做）
- 这是故事的核心，要有画面感

**第三段：为什么要赢**（1-2句）
- **成功愿景**：描绘胜利后的具体画面（不是"获得奖励"）
- **悬念钩子**：制造好奇心，让玩家想知道结局
- **挑战召唤**：激发斗志，让玩家想证明自己
- 三选一，让结尾有「我想玩下去」的冲动

### 写作原则

**像讲童话，不像写说明书**：
- 用**具体画面**代替抽象描述（"星星会漏下去" vs "游戏失败"）
- 用**情感语言**代替功能语言（"找回失散的伙伴" vs "收集目标物"）
- 用**留白**代替解释（不用解释为什么三消会消失，玩家会自己理解）
- 用**短句**保持节奏（避免从句套从句）

**机制解释的处理**：
- 简单机制：不解释，让玩家自己发现
- 复杂机制：用一句话自然带过，不要专门解释
- 错误示范：❌ "根据古老的法则，当三个同色宝石排列在一起时..."
- 正确示范：✅ "三颗星星连成线，就会化作流星飞走"

### 情感节奏

**好故事有呼吸感**：
- 第一段：平静或紧张（建立情境）
- 第二段：行动和希望（角色出发）
- 第三段：温暖或期待（美好愿景）

**避免**：
- 每句话都在交代新信息
- 全程紧绷没有停顿
- 结尾像系统提示（"你将获得..."）

### 结尾多样性

**结尾要有「游戏感」**——让玩家想继续玩、想赢、想看到结局。

**三种结尾方向**（每次选一种）：

**1. 成功愿景型**（描绘胜利后的画面）
- 「当最后一块拼图归位，沉睡的王国将重新苏醒。」
- 「所有萤火虫重新亮起的那一刻，整片森林会变成银河。」
- 「传说中的黄金厨房，只为最优秀的厨师敞开大门。」

**2. 悬念钩子型**（制造好奇，想继续玩）
- 「塔顶究竟藏着什么？只有爬上去才知道。」
- 「传说集齐2048的人，能实现一个愿望——是真的吗？」
- 「深海最深处的那条鱼，从来没有人见过它的真面目。」

**3. 挑战召唤型**（激发斗志，想证明自己）
- 「全城最快的送餐员？这个称号，你要不要？」
- 「它们说你撑不过第10波。证明它们错了。」
- 「最高分的位置还空着，写上你的名字。」

**避免**：
- ❌ 纯童话式：「从此过上了幸福的生活」（没有游戏感）
- ❌ 纯系统式：「你将获得丰厚奖励」（像说明书）
- ❌ 空洞激励：「加油，你可以的！」（没有具体画面）

### 故事示例

**贪食蛇 - 好的写法**：

> 森林里的萤火虫突然失去了光芒，整片树林陷入黑暗。
>
> 小萤火虫「点点」是最后一只还能发光的。它决定独自出发，用微弱的尾灯照亮前路，把散落在树洞、花丛、蘑菇下的光点一颗颗找回来。每找到一颗，身后就多一只苏醒的伙伴，队伍越来越长——但森林小路很窄，一不小心就会撞上自己的队伍。
>
> 当所有萤火虫重新亮起，整片森林会变成银河。

**贪食蛇 - 差的写法**：

> ❌ 你是一只萤火虫，需要在森林中收集光点。每收集一个光点，你的身体就会变长，这是因为光点会转化为身体能量。你需要避免撞到墙壁或自己的身体，否则游戏结束。收集足够的光点后，你将获得胜利奖励并解锁下一关。
