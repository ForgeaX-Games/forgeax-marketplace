# Archetype D — Emergent Narrative Baseline

> 适用品类家族：沙盒生存、4X策略、开放世界沙盒、模拟经营、太空探索
> 当品类缺少独立 .skill.ts 时，PromptComposer 回退到此 baseline。

---

## 核心叙事哲学

Emergent 叙事的本质是「系统讲故事」——叙事不来自预编排的剧情线，而是从游戏系统的交互中自然涌现。作者的角色从「讲述者」变为「世界规则设计者」。

关键支柱：
- **玩家能动性至上**：每位玩家的故事独一无二，因为选择和时序不同
- **事件模板 × 上下文**：预制事件骨架 + 动态填充当前世界状态 = 每次触发都不同
- **系统产生戏剧性**：资源匮乏、派系冲突、生存压力本身就是叙事引擎

## 世界观构建指引

### 结构原则
- 世界是「活的生态系统」而非「故事舞台」——有没有玩家，世界都在运转
- 资源分布/地理要素驱动冲突（高地 vs 盆地 / 水源 vs 荒漠 / 贸易线 vs 劫掠线）
- 派系/文明不是静态势力——有人口、经济、军事的动态模拟
- 历史不重要，「当下的紧张关系」才重要

### 世界观的「叙事种子」
- 不需要编排完整主线，而是准备一组「初始紧张关系」：
  - 两个派系争夺同一资源
  - 一场即将到来的自然灾害/季节变化
  - 一位流浪商人带来的违禁品
  - 远方传来的战争流民潮
- 这些种子被系统拾取后，会与玩家行为交织产生独特故事

### 关键检查项
- [ ] 世界有至少 3 种可被争夺的核心资源
- [ ] 至少 2 对派系/势力存在结构性矛盾
- [ ] 有明确的环境压力机制（季节/灾害/衰变）
- [ ] 世界变化对所有 NPC/派系都有影响（非仅针对玩家）

## 角色/派系塑造指引

### 派系作为角色
- 每个派系有独立的价值观 / 治理方式 / 对外态度
- 派系内部有鸽派/鹰派分裂——玩家的行为可能影响内部权力天平
- 派系有「国民性格」——影响外交对白风格和事件反应模式

### NPC 设计
- 重要 NPC 有「议程」——无论玩家做什么，NPC 都在追求自己的目标
- NPC 关系网独立于玩家存在（A 和 B 是宿敌，无论玩家是否介入）
- NPC 对玩家的态度由行为累积决定——不是预设的好感度初始值

### 玩家角色
- 玩家 = 白板，叙事通过「玩家做了什么」来定义角色
- 提供「玩家身份标签」系统：其他 NPC 会根据你的行为给你贴标签（建设者/征服者/商人/隐者）
- 标签影响后续事件触发和 NPC 初始态度

## 事件模板系统指引

### 事件设计结构
```
事件模板 = {
  触发条件: [世界状态 + 玩家状态 + 随机权重],
  事件骨架: [阶段1 → 阶段2 → 结果],
  动态填充: [当前相关NPC / 资源 / 地点 / 天气],
  后果链: [对派系关系 / 资源 / 区域状态的影响]
}
```

### 事件类型清单
- **危机事件**：旱灾/瘟疫/入侵/叛乱 → 迫使玩家做出取舍
- **机遇事件**：发现遗迹/商队到来/新技术 → 引发争夺
- **社交事件**：求婚/决斗/节日/审判 → 丰富人物关系
- **连锁事件**：A 事件的结果触发 B 事件 → 蝴蝶效应
- **个人事件**：依据玩家标签触发的独属于玩家的叙事片段

### 事件文案原则
- 描述「发生了什么」而非「你应该做什么」——不替玩家做判断
- 提供信息但不提供答案——「商队被劫了，北方有蛮族营地，南方有盗贼窝」
- 结果文案反映实际世界变化——「村庄 A 因为你的决定繁荣/衰败了」

## 场景生成指引

### 场景作为「时刻」
- Emergent 叙事的场景不是线性推进的，是「此刻世界状态的快照」
- 场景描述必须包含环境变量：天气/时间/季节/附近NPC活动
- 同一地点在不同世界状态下呈现完全不同的叙事色彩

### 生成原则
- 优先使用「感官描写」：看到什么/听到什么/闻到什么
- 暗示系统状态而非直接陈述：「空气中弥漫着焦味，远处有浓烟」而非「火灾正在发生」
- 留白给玩家想象：描述当前状态，不预设玩家反应

## UI/日志叙事指引

### 事件日志文案
- 简洁 + 有调性：「第37天：暴风雨摧毁了东部围墙。」而非「围墙耐久度降为0」
- 日志本身构成一部「微型编年史」——回看日志应该能感受到一段独特的历史
- 重要事件用特殊格式标记（颜色/图标/字体变化）

### 加载/过渡文案
- 反映当前世界状态：「你的领地正在经历第三个寒冬。储备还够撑半个月。」
- 偶尔加入NPC视角：「附近的旅人说起你的名字时，语气里有敬畏。」
