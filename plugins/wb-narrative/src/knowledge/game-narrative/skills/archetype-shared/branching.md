# Archetype B — Branching Narrative Baseline

> 适用品类家族：视觉小说、互动电影、侦探推理、恋爱模拟、Galgame、互动戏剧
> 当品类缺少独立 .skill.ts 时，PromptComposer 回退到此 baseline。

---

## 核心叙事哲学

Branching 叙事的本质是「选择即身份」——玩家通过一系列有意义的决策，亲手塑造属于自己的故事版本。每个分支不是简单的 A/B 路径，而是价值观表达。

关键支柱：
- **选择有后果**：没有任何选择是装饰性的——短期、中期、长期后果逐层涟漪
- **角色关系追踪**：好感度/信任度/怀疑度作为隐形叙事变量，影响对白、场景触发、结局走向
- **多结局即多主题**：每个结局是「如果你坚持这个价值观，世界会变成什么样」的完整答案

## 世界观构建指引

### 结构原则
- 世界需要「封闭舞台」特质：有限空间（学校/岛屿/宅邸/城市区块）内关系密集交织
- 时间压力装置：限定天数 / 事件倒计时 / 不可逆的时间流逝
- 信息不对称 = 悬念引擎：不同角色掌握不同碎片，玩家选择决定信息获取路径
- 世界规则必须自洽——分支再多，物理/社会规则不变

### 关键检查项
- [ ] 舞台边界清晰（玩家活动空间有明确限制）
- [ ] 至少有 1 个「不可逆事件窗口」（错过 = 永远无法触发）
- [ ] 信息碎片分配到不同角色路线中
- [ ] 存在隐藏真相需要多周目/多路线拼合才能获知

## 角色塑造指引

### 可攻略/可调查角色
- 每位角色的「表面形象」与「真实内核」必须有落差（人设反转是核心魅力）
- 好感度阶梯：每 N 点解锁新的对白层次（从客套→信任→坦诚→亲密/依赖）
- 好感与怀疑可以共存——「我喜欢你但不信任你」是高级情感状态

### 关系系统
- 关系变量不止二元（好感/恶感）——引入信任/占有/嫉妒/愧疚等多轴
- 角色之间存在关系网：对 A 好可能让 B 疏远（社交资源有限性）
- NPC 之间有独立关系发展——不以玩家为唯一中心

### 反派/嫌疑人
- 真凶/反派的动机必须在复盘时完全自洽（所有行为可解释）
- 红鲱鱼角色需要「差点是真的」——其嫌疑必须建立在合理推理上

## 分支结构指引

### 分支设计原则
- **选择类型**：价值观抉择 > 信息收集方向 > 对话语气 > 数值累积
- **后果可见性**：短期（立即反馈）+ 中期（数场景后回响）+ 长期（结局级影响）
- **假选择检测**：如果两个选项最终收束到完全相同的结果，这是假选择——删除或增加差异
- **瓶颈节点**：所有分支在关键剧情节点收束，确保核心叙事推进

### 结局系统
- 结局数量建议：1 True End + 2-4 Normal End + 1-2 Bad End
- 每个结局必须是某种价值观选择的「完整回答」
- Bad End 不是惩罚——是「如果你坚持这样选，世界确实会这样」
- True End 应需要多周目信息拼合（不是最好的，是最完整的）

### 变量管理
- 使用标记系统（flag）追踪关键决策点
- 好感度 / 信任度 / 线索收集度作为阈值判定条件
- 变量之间允许交互（角色 A 的好感度影响角色 B 事件触发）

## 剧本写作指引

### 对白层次
- 同一场景的对白根据关系变量呈现不同层次（冷淡版/友好版/亲密版）
- 角色说谎时的微妙矛盾——给玩家「直觉上觉得哪里不对」的信号
- 问话/审讯场景：玩家选择问什么 = 选择获得什么信息碎片

### 节奏控制
- 每个调查/对话场景控制在 3-5 分钟体验时长
- 重大抉择前必须有「犹豫时间」——给玩家一段思考性独白/总结
- 高潮场景用时间压力（倒计时 / 不可暂停的对话流）

### 叙事诡计
- 叙述者不可靠：第一人称视角可以省略/扭曲信息
- 时间线打乱：通过回忆/闪回揭示过去，但不破坏当前推理逻辑
- 循环构造：多周目解锁不同侧面，用「似曾相识」暗示元叙事

---

## 技术约束

- 单个分支点的选项数控制在 2-4 个（避免认知过载）
- 分支深度上限建议 4-5 层（超出需要瓶颈节点收束）
- 每条路线的总文本量差异不超过 30%（避免某路线体验过薄）
- 变量名使用语义化命名（`rei_trust` 而非 `var_003`）
