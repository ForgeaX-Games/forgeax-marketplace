# Archetype G — Micro Narrative Baseline

> 适用品类家族：超休闲、Hyper-casual、小游戏合集、广告变现型游戏（Tier4）
> 当品类缺少独立 .skill.ts 时，PromptComposer 回退到此 baseline。

---

## 核心叙事哲学

Micro 叙事的本质是「一眼即懂的世界感」——玩家在 3 秒内理解游戏世界是什么，30 秒内感受到角色的存在，不需要阅读任何文字。叙事完全融入视觉和交互设计。

关键支柱：
- **美术风格即叙事**：色调、造型、动画风格本身传达全部世界观
- **零文字门槛**：任何叙事信息都不能依赖阅读理解
- **UI文案即全部剧本**：按钮/弹窗/成就名/角色名是唯一的文字叙事载体

---

## 世界前提设计

### 一句话世界观
整个游戏的世界观必须能用 1 句话（< 15 字）概括：
- 「猫咪开餐厅」
- 「水果忍者大战」
- 「火柴人逃出办公室」
- 「像素小人建塔」

### 设计原则
- 概念即画面：听到概念就能想象出游戏画面
- 无需解释前因后果——世界存在即合理
- 角色/世界的「可爱」/「酷」/「蠢萌」= 唯一需要的情感锚点
- 冲突动机用视觉暗示（大 vs 小 / 追 vs 逃 / 建 vs 毁）

### 一句话模板
```
[角色/主体] + [在做什么] + [可选：在哪里/对抗什么]
```

## 角色设计

### 最小角色定义
- **造型辨识度**：剪影就能认出来
- **1个标志色**：品牌色 = 角色色 = 游戏色
- **动作个性**：跑/跳/胜利/失败动画有性格（笨拙/优雅/爆裂）
- 不需要名字/背景/对白——形象本身就是角色

### 角色家族
- 如果有多个角色，用简单规则生成差异：
  - 颜色变化（红/蓝/绿/金）
  - 体型变化（胖/瘦/高/矮）
  - 配件变化（帽子/眼镜/翅膀）
- 角色间不需要叙事关系——「同一个世界的不同住民」即可

## 视觉叙事

### 美术风格选择即世界观
| 风格 | 暗含叙事 | 适合品类 |
|------|----------|----------|
| 扁平矢量 | 现代/科技/简洁 | 益智/策略 |
| 像素复古 | 怀旧/冒险/8bit | 平台跳跃/动作 |
| 涂鸦手绘 | 童趣/即兴/DIY | 绘画/创意类 |
| 3D卡通 | 活力/动态/沉浸 | 跑酷/竞速 |
| 极简几何 | 冥想/纯粹/专注 | 解谜/relaxing |

### 环境叙事（无文字）
- 背景层次暗示空间深度和世界规模
- 颜色温度变化 = 难度/情绪变化（暖→冷 = 安全→危险）
- 场景元素的动态行为暗示世界是活的（鸟飞过/云移动/灯闪烁）

## UI文案设计

### 核心原则
- 每条文案 ≤ 10 字（中文）/ 5 词（英文）
- 能用图标替代文字就用图标
- 文案语气 = 游戏语气（可爱游戏用可爱文案）
- 不用术语/行话——5岁小孩能懂

### UI文案清单
| 位置 | 要求 | 示例 |
|------|------|------|
| 游戏名 | 2-4字，有画面感 | 「切水果」「叠叠乐」 |
| 开始按钮 | 动词，有活力 | 「开玩！」「冲！」 |
| 失败画面 | 轻松，不打击 | 「差一点！」「再来」 |
| 胜利画面 | 夸张，快乐 | 「太强了！」「完美✨」 |
| 成就名 | 有梗/有趣 | 「手速怪物」「一次都没死（骗人的）」 |
| 关卡名 | 可选，画面感 | 「果冻森林」「熔岩蹦床」 |
| 引导提示 | 动词+方向 | 「滑动切开！」「点击跳跃」 |

### 数字反馈的叙事包装
- 分数/金币用视觉效果代替数字理解（金币雨 > 「+100金币」）
- combo 用越来越夸张的视觉/音效表达（而非单纯数字+1）
- 进度条/等级可以用视觉隐喻（种子→发芽→开花 = 经验增长）

## 广告/留存叙事

### 插屏广告过渡
- 广告前后的游戏场景保持视觉连贯（不要突兀切断）
- 可选：用角色动画做过渡（角色「搬开」广告牌）

### 次日留存叙事钩子
- 不需要「剧情悬念」——用「进度可视化」替代（你的花园长大了）
- 每日登录的角色表情变化（今天开心/难过/期待）
- 累积性视觉奖励比文字叙事更有效（7天拼完一幅画）

---

## 约束

- 全游戏文字总量 < 500 字（含UI/提示/成就全部）
- 无需过场动画/CG/对白——所有信息通过交互和视觉传递
- 本地化成本极低（< 200 字翻译量）
- 叙事决策不能增加任何加载时间或额外资源包大小
