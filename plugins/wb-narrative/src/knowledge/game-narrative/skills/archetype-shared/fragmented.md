# Archetype C — Fragmented Narrative Baseline

> 适用品类家族：魂系、银河城、恐怖、Roguelike、步行模拟器
> 当品类缺少独立 .skill.ts 时，PromptComposer 回退到此 baseline。

---

## 核心叙事哲学

Fragmented 叙事的本质是「世界已经发生过了」——玩家不是在经历故事，而是在废墟中拼凑真相。叙事的主体从「事件流」转变为「考古学」。

关键支柱：
- **环境即叙事**：建筑布局、物品摆放、光影变化本身讲述故事，不依赖对白
- **碎片拼图**：玩家主动收集并关联 Lore 碎片，每个玩家构建的理解图谱可能不同
- **暗示优于明说**：永远不把答案摊开——宇宙恐怖/宿命/轮回等主题用「接近但不触及」的方式呈现

## 世界观构建指引

### 结构原则
- 世界有「黄金时代→灾变→当下废墟」的三段历史
- 灾变原因 = 核心谜团，贯穿整个游戏的 Lore 拼图
- 空间叙事：区域间的高低/明暗/开合对应叙事阶段（深渊 = 真相核心）
- 规则宇宙学：世界运行的底层规则本身就是谜（什么是火/暗/轮回/死亡？）

### 关键检查项
- [ ] 世界有清晰的「灾变前/灾变后」断层
- [ ] 核心谜团至少有 2 种合理解读（允许社区争论）
- [ ] 地图布局暗含叙事隐喻（上升=觉醒/下降=堕落）
- [ ] 存在 1-2 个「真相过于恐怖所以被隐藏」的禁忌知识

## 角色塑造指引

### 角色存在方式
- 大多数角色以「残留」形式出现：幽灵/日记/尸体/雕像/回声
- 活着的 NPC 极少，每位都值得玩家警觉（为什么他还活着？）
- 对白极简：每个 NPC 对话总量 < 200 字，每句话都有信息密度

### Boss 作为叙事节点
- 每个 Boss 代表灾变历史中的一个「角色」
- Boss 的攻击模式/场地/音乐暗示其生前故事
- 击败 Boss 后解锁的 Lore = 该角色的真相碎片

### 不可靠叙述者
- 少数 NPC 的说辞互相矛盾——玩家需要判断谁在说谎或谁记忆扭曲
- 物品描述可以与 NPC 说法冲突——物品更可信（沉默的证据）

## 碎片化 Lore 设计指引

### 投递媒介
- **物品描述**：每条 < 80 字，3 句以内，聚焦一个信息点
- **环境叙事**：尸体姿态 / 物品摆放 / 建筑损毁方式 / 涂鸦文字
- **Boss/精英台词**：战前 1 句 + 死亡 1 句，暗示性而非说明性
- **场景变化**：重访区域时的微妙变化暗示时间/因果

### Lore 架构原则
- 同一事件从 3-5 个不同视角/物品分散叙述
- 碎片之间从不直接引用彼此——靠共同的名词/意象关联
- 核心真相需要累积 60-70% 碎片才能推断——不给 100% 确定性
- 关键碎片放在危险区域 / 隐藏路径（付出代价才能获得真相）

### 层次结构
1. **表层 Lore**（60%）：所有玩家都会获得，构建基础世界认知
2. **深层 Lore**（30%）：需要探索/挑战，揭示灾变真相
3. **禁忌 Lore**（10%）：隐藏极深，动摇世界底层认知

## 场景/关卡叙事指引

### 环境叙事手法
- **物件叙事**：桌上的半杯茶 = 有人匆忙离开；地板的拖痕 = 有东西被拖走
- **建筑叙事**：封闭的门 / 坍塌的桥 / 被焊死的窗 = 有人试图阻止什么
- **光影叙事**：唯一光源的方向引导视线 / 黑暗区域隐藏的东西
- **声音叙事**：远处的歌声 / 重复的滴水 / 不应存在的脚步声

### 空间情绪曲线
- 区域进入时：压迫/未知感
- 探索中段：理解加深，恐惧从「未知」变为「已知但不可阻止」
- Boss 前：沉默，给玩家时间消化刚获得的 Lore
- Boss 后：短暂的释放 + 新信息带来的更大不安

## 剧本写作指引

### 对白原则
- 少即是多：能用 5 个字绝不用 20 个字
- 禁止说教/解释：NPC 不会主动告诉你「这个世界的真相是……」
- 破碎语法：某些 NPC 的对白可以不完整/重复/自相矛盾（暗示精神状态）
- 暗示性句式：「你也来了……和他们一样」（谁是他们？玩家自己想）

### 物品文案
- 结构：名称 + 1-2 句描述 + 隐含的叙事信息
- 风格参考：「曾属于某位不愿被遗忘的人。刀刃上的锈迹不是铁锈的颜色。」
- 禁忌：不直接点明关联（「这把剑属于Boss A」→ 改为用细节暗示）

### 氛围基调
- 底色：寂静、宿命、哀悼
- 情绪波动范围窄而深——不需要欢乐场景，但需要「微光」（片刻温暖后的更深绝望）
- 恐怖来自于理解，不来自于 jumpscare
