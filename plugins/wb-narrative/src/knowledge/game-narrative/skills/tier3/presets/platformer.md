# 2D 横版 / Metroidvania 叙事预设

## 品类特征

| 维度 | 详情 |
|------|------|
| **叙事载体** | 环境叙事 + 收集品 Lore + 简短 NPC 对话 |
| **叙事节奏** | 玩家主动探索发现故事，非被动接受 |
| **Metroidvania 特点** | 区域有历史，能力解锁和叙事解锁同步 |
| **代表作** | 空洞骑士、星之卡比、奥里与黑暗森林、死亡细胞、蔚蓝 |

---

## 叙事结构：探索驱动

```
[玩家探索新区域]
        │
  环境叙事（场景传达历史）
        │
  收集品 Lore（碎片化历史）
        │
  NPC 短暂对话（关键信息锚点）
        │
  Boss 战（叙事高潮）
```

---

## 核心叙事需求

### 环境叙事 (E) ★★★（横版核心）
- 场景名称有叙事意义（「遗弃的铸造场」vs「熔岩区域2」）
- 场景背景美术传达历史（废墟/战场/被遗弃的家园）
- 场景道具/装饰暗示曾经发生的事

### 物品叙事 (I) ★★★
- 收集品/记忆/日记碎片（Metroidvania 标配）
- 每件收集品 50-100 字的背景文本
- 收集品拼凑完整的世界历史

### 角色设计 (C) ★★
- 主角：有动机但可以沉默（《空洞骑士》式）
- NPC：少量但有深度，非功能性存在
- Boss：每个 Boss 都是世界历史的一部分

### 对话剧本 (D) ★
- 对话极简（玩家在行动流，不读长文）
- 每次对话传达 1 个关键信息
- NPC 对话随进度变化（再访有不同反应）

## 场景叙事设计模板

```markdown
## [区域名称]

**叙事定位**：[这个区域曾经是什么，发生了什么]
**当前状态**：[现在是什么状态，为什么变成这样]
**视觉叙事锚点**：
- [视觉元素A]：暗示 [历史事件]
- [视觉元素B]：暗示 [世界规则/历史]
**收集品数量**：[X 件，讲述 [主题] 的故事]
**关联 NPC**：（若有）[NPC 名字 + 与区域的关系]
```

---

## 收集品 Lore 写作规范

```markdown
**收集品名称**：[有叙事感，非「道具A」]
**类型**：日记 / 信件 / 遗物 / 图腾 / 记忆碎片

**文本**（50-150字）：
[用第一人称或目击者视角，1-2段，描述发生了什么]

写作要点：
- 不要直接说明世界背景，通过个人视角暗示
- 结尾留白（"之后他再也没有回来" / "那声音还在我耳边"）
- 与其他收集品形成拼图关系
```

**示例**：
> 《铸造工的日记，第7页》
> 今天大师说要停工了。那些蛋——越来越多的蛋——开始孵化。
> 我听见了声音。不是正常的声音。
> 大师说这不是我们的事，我们只是工人。
> 但工坊里的其他人，一个个地，消失了。

## Boss 叙事（精简版）

Metroidvania 的 Boss 是**活生生的叙事节点**：

```markdown
## [Boss 名称]

**身份**：[是这个区域历史的什么角色]
**为什么变成这样**：[1-2句，堕落/被困/守护原因]
**战斗氛围**：[愤怒/悲伤/疯狂/守护...]
**击败后叙事**（可选）：[死亡时说什么？揭露什么？]
**关联收集品**：[哪些收集品揭露了这个 Boss 的背景]
```

---

## Metroidvania 叙事节奏

| 能力解锁 | 配套叙事解锁 |
|---------|------------|
| 获得双跳 | 解锁高处区域的历史遗迹 |
| 获得破墙技能 | 发现被封闭的禁忌场所 |
| 获得水下呼吸 | 进入溺水区域，发现沉没文明 |
| 获得传送 | 进入隐藏区域，叙事核心揭露 |

**设计原则**：每次能力解锁都是"新的叙事钥匙"。

## NPC 对话简化原则

```
第1次遇见：[建立存在感，1-2句]
第2次：[对玩家进度有反应，1句]
第3次+：[轻微推进叙事或增加世界感，1-2句]
完成特定条件后：[触发深度对话，3-5句，揭露关键信息]
```

## 质量检查
