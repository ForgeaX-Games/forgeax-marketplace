# 塔防叙事预设

## 品类特征

| 维度 | 详情 |
|------|------|
| **叙事作用** | 交代"为什么要防守"，给战斗提供情感理由 |
| **叙事载体** | 关卡简介 + 单位/建筑描述 + 关卡故事背景 |
| **叙事密度** | 最低密度，玩家专注于策略，叙事是背景色 |
| **代表作** | 植物大战僵尸、保卫萝卜、王国保卫战、Bloons TD |

---

## 叙事核心：给战斗一个理由

塔防的叙事目标很单纯：
1. **为什么要守？**（守护什么）
2. **敌人是谁？**（为什么要来攻击）
3. **守住后怎样？**（胜利的意义）

---

## 世界观框架（极简）

```markdown
**核心设定**（1-2句）：
[交代世界背景，保卫的是什么]

**玩家角色**（1句）：
[玩家是谁，有什么能力/资源]

**威胁来源**（1-2句）：
[敌人是谁，为什么要进攻]

**最终目标**（1句）：
[守住/击退/反攻的终极目标]
```

**示例**：
> 这片古老的王国被黑暗帝国包围。
> 你是最后一位防卫工程师，拥有失落的古代建造知识。
> 怪物军团受黑暗水晶驱动，它们不会停止，直到摧毁一切。
> 守住王国的心脏，就有机会逆转这场战争。

---

## 关卡叙事设计

每个关卡有 3 个叙事锚点：

```markdown
## 关卡 X：[关卡名称]

**战前简介**（1-3句）：
[交代这个战场的背景，为什么要在这里防守]

**关卡独特敌人/事件**（若有）：
[本关特殊机制的叙事理由]

**胜利叙事**（1-2句）：
[守住后发生了什么，进度感/情感落地]
```

---

## 单位/建筑描述规范

```markdown
**[单位/建筑名]**
功能描述：[清晰的战术效果说明]
背景（可选，1句）：[叙事风味文字]

示例：
「古老投石车」
功能：远程攻击，每次攻击范围内所有敌人，伤害30
背景："据说这台机器见证过五百年前的最后一场战役。"
```

**命名原则**：
- 有叙事感的名字（「烈焰守卫者」vs「防御塔2」）
- 名字暗示功能（「冰霜魔法师」不需要解释能放冰）
- 派系/阵营感（「帝国弩手」vs「精灵弓箭手」）

---

## 敌人设计（叙事层面）

每类敌人有简短的叙事定位：

```markdown
## [敌人类型]

**叙事定位**（1句）：[这类敌人是什么，来自哪里]
**外观描述关键词**：[3-4个形容词]
**为什么进攻**：[受什么驱使，有什么目的]
**独特性**（可选）：[有没有特殊叙事背景]
```

---

## 主要 NPC（若有）

塔防有时有指挥官/盟友 NPC：

```markdown
## [NPC名字] — [角色定位]

**功能**：[在游戏中的功能，提供建议/技能/商店]
**性格**（2-3个关键词）：
**台词风格**：
- 关卡开始：（1句）
- 危机时：（1-2句）
- 胜利时：（1句）
- 失败时：（1句）
```

---

## 世界地图叙事（若有大地图）

如果游戏有关卡选择地图，地图节点需要叙事感：

| 区域类型 | 命名风格 | 叙事描述 |
|---------|---------|---------|
| 开始区 | 家园/起点（温暖感） | "这里是你们最初的家园" |
| 中期区 | 险境/废墟（挑战感） | "帝国军的先头部队已经抵达这里" |
| 终章区 | 核心/要塞（决战感） | "守住这里，才能守住一切" |
