# tier3-narrative-accent

为**叙事点缀型游戏**提供精简叙事配置。

叙事是游戏的**背景板**，玩法才是核心。叙事的作用是：让游戏有"灵魂"，让玩法有"理由"。

---

## 触发条件

用户说要做以下类型的游戏时触发：

| 用户描述 | 品类 | 预设文件 |
|---------|------|---------|
| 卡牌构筑 / deck-building / 肉鸽卡牌 | card-game | presets/card-game.md |
| Roguelike / Roguelite / 随机地牢 / 以撒式 / 哈迪斯式 | roguelike | presets/roguelike.md |
| 2D 横版 / 平台跳跃 / Metroidvania | platformer | presets/platformer.md |
| 塔防 / Tower Defense / 防御类 | tower-defense | presets/tower-defense.md |

**不触发**：叙事驱动型游戏（用 Tier 1）；抽卡 RPG/ARPG（用 Tier 2）；超休闲/消除（用 Tier 4）。

---

## 管线策略：精简启动

```
Phase 1 → Phase 2（精简）
 创意策划    基础世界观 + 角色框架
（Phase 4 仅核心内容，Phase 5 跳过）
```

**精简原则**：
- 世界观：有即可，不铺陈
- 角色：核心角色设定，无需完整弧光
- 剧情：有框架，无需章节细化
- 对话：必要的触发式对话，无长篇叙事

## 执行模式

| 情况 | 模式 | 做什么 |
|-----|------|-------|
| 用户要完整叙事设定 | **基础框架** | 世界观 + 角色 + 世界规则，不细化章节 |
| 用户要卡牌/道具文案 | **文案生成** | 批量生成有叙事感的功能文案 |
| 用户要故事背景 | **背景速成** | 用 1 页纸交代清楚世界背景 |
| 用户要角色 | **轻量角色** | 关键词式角色设定，不需要完整弧光 |

## Tier 3 叙事设计核心原则

### "一页纸世界观"
Tier 3 的世界观应该能用**一页纸**说清楚：
- 世界是什么样的？（1-2 句）
- 为什么玩家要行动？（1 句）
- 有什么力量/规则？（2-3 句）
- 最终目标是什么？（1 句）

### 叙事服务玩法
- 卡牌名称有叙事意象，但首要保证功能清晰
- 每次运行有"原因"，即使是 Roguelike 重复内容
- 升级/解锁有叙事感（非纯数字提升）

### 碎片化叙事
- 不要求玩家读完，先玩后了解
- 解锁内容揭露更多背景（粉丝向）
- 每局 Roguelike 的随机事件有世界观背景
