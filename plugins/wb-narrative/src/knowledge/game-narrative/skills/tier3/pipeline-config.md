# Tier 3 管线配置

> 叙事点缀型游戏（15-40%）精简管线配置参考。

---

## 管线执行策略：Simplified Pipeline

```
Phase 1 → Phase 2（精简）→ Phase 4（仅核心）
 创意策划    基础框架          必要文案
```

Phase 5 跳过。

---

## Phase 1：轻量创意策划

**与 Tier 1/2 的差异**：
- High Concept 聚焦玩法，叙事是辅助说明
- 不需要详细规模预估（叙事体量小）
- 输出物精简：一页纸概念 + 角色关键词

**输出步骤**（精简版）：
- `core_concept`：玩法 High Concept + 叙事背景一句话
- `value_framework`：世界观关键词（3-5个词，非完整描述）

---

## Phase 2：基础框架（精简）

**目标**：建立最小可用的叙事框架

**输出步骤**：
- `worldview_generation`（精简版）：一页纸世界观
- `character_generation`（精简版）：核心角色关键词表

**Tier 3 特殊约束**：
- `worldview_generation` 输出 ≤ 500 字
- `character_generation` 每个角色 ≤ 150 字
- 不生成 `story_framework`（除非用户明确要求）
- 不生成 `outline_batch` / `detailed_outline`

---

## Phase 4：核心文案生成

**只生成必要的叙事内容**：

| 内容模块 | 优先级 | 是否必须 |
|---------|-------|---------|
| UI文案（系统/卡牌/能力描述） | P0 | 必须 |
| 主要角色简短对话 | P1 | 推荐 |
| 卡牌/物品风味文本 | P1 | 推荐（卡牌游戏必须） |
| Roguelike 随机事件文本 | P2 | 按品类 |
| 环境场景名称/描述 | P2 | 可选 |
| Lore 碎片（图鉴/解锁内容） | P3 | 深度内容，可选 |

---

## 各品类 Phase 4 重点

| 品类 | 必须内容 | 推荐内容 |
|-----|---------|---------|
| 卡牌构筑 | 卡牌描述（功能+风味） | NPC/Boss 对话、事件文本 |
| Roguelike | 随机事件文本、能力描述 | 环境叙事、死亡台词 |
| 2D横版 | 场景名称/描述、主角台词 | 收集品 Lore、环境道具文字 |
| 塔防 | 单位/建筑描述、关卡说明 | 关卡故事背景、胜负台词 |

---

## 文案风格规范

Tier 3 游戏的文案风格必须与玩法氛围一致：

| 玩法氛围 | 推荐文案风格 | 示例 |
|---------|------------|------|
| 黑暗/严肃 | 短句、沉重感 | "黑暗将吞噬一切——除非你够快。" |
| 幽默/轻松 | 自嘲、流行语 | "史上最失败的英雄冒险——又一次。" |
| 神秘/悬疑 | 暗示性、留白 | "它说它来自未来。也许吧。" |
| 热血/战斗 | 命令式、简洁 | "撑住。还有最后一波。" |

---

## 节点级重跑配置（简化版）

Tier 3 叙事量小，通常不需要节点级重跑：

| 修改范围 | 处理方式 |
|---------|---------|
| 修改世界观关键词 | 重跑所有文案（量小，全量更快） |
| 修改角色设定 | 重跑该角色相关对话/描述 |
| 修改文案风格 | 全量重跑文案（风格影响所有文案） |
| 增加新内容类型 | 独立生成，无需重跑已有内容 |
