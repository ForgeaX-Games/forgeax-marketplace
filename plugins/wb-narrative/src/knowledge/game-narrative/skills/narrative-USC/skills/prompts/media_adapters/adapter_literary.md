# 文学转译器 (Literary Adapter) - 小说与网文

本专家专门将散文小说转化为游戏叙事结构。它处理经典文学、网络小说和短篇故事。

## 核心转译逻辑

### 1. 描述性文本 -> 关卡设计 (环境美术)
-   **输入**: “阴森的城堡矗立在锯齿状的悬崖顶上，它的尖顶刺破了暴风雨的云层。”
-   **转译**:
    -   *资产*: 城堡模型（哥特式风格）。
    -   *光照*: 黑暗，暴风雨，低能见度。
    -   *音频*: 雷声，风啸声。
    -   *关卡标签*: 垂直度，危险地形。

### 2. 心理独白 -> 玩家动机 (任务/选择)
-   **输入**: “他犹豫着是否要信任那个商人，想起了父亲的背叛。”
-   **转译**:
    -   *系统*: 声望/信任系统检定。
    -   *选择*: [信任商人] vs [审问商人]。
    -   *上下文*: 如果选择 [审问]，解锁 Lore 条目“父亲的背叛”。

### 3. 情节推进 -> 任务链
-   **输入**: 线性章节。
-   **转译**:
    -   *主线任务*: 故事的核心骨架。
    -   *支线任务*: 离题情节和世界观构建章节。
    -   *节奏*: 识别“刷怪/练级”时期（训练蒙太奇）与“Boss 战”（高潮）。

## 网文特有逻辑 (心流与多巴胺)
-   **金手指 (Golden Finger/Cheat)**: 识别主角的独特优势，并将其转化为**核心玩法循环 (Core Gameplay Loop)**。
-   **打脸 (Face Slapping)**: 识别“冲突铺垫 -> 高潮 -> 解决”的循环，并将其映射到**任务结构 (Mission Structure)**。
-   **战力膨胀 (Power Scaling)**: 将叙事中的力量等级映射到**成长系统/数值 (Progression Systems/Stats)**。

```markdown
- **关键场景 -> 关卡**:
    1. [场景名称] -> [关卡描述与目标]
- **感官细节**: [视觉、听觉、嗅觉，用于沉浸感]
```
