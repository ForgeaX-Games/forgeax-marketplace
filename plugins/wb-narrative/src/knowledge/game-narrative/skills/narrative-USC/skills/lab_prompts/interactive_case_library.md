# 互动叙事结构设计案例库 (Interactive Architecture Case Library)

为了更好地使用 `interactive_architect_skill`，以下是针对不同题材的结构设计逻辑参考：

## 1. 恋爱互动叙事 (Romance Interactive)
*   **结构**: **“主轴+多情感线” (Central Axis with Multi-Emotional Tracks)**。
*   **主角**: 追求真爱或自我成长的个体。
*   **配置**: 1个主角 + 3~4个性格迥异的攻略对象。
*   **选项逻辑**: 
    *   大量“非决定性选项”用于刷好感度。
    *   “重要选项”决定进入哪位角色的个人线。
    *   “关键选项”出现在个人线末端，决定“真爱结局”或“遗憾分手”。
*   **数值**: **恋爱情感值 (Affection)** + **个人魅力值 (Charisma)**。魅力值决定是否能开启高难度对话。

## 2. 谍战/悬疑互动叙事 (Spy/Thriller Interactive)
*   **结构**: **“高压网状结构” (High-Pressure Web Structure)**。
*   **主角**: 潜伏者、双面间谍。
*   **配置**: 1个主角 + 多个怀疑者 + 1个上线。
*   **选项逻辑**: 
    *   **选错即死 (Permadeath/Instant Fail)**: 在涉及身份暴露的对话中，错误选项直接导致游戏结束。
    *   **代价选择**: 每一个获得情报的选项都必须付出代价（如：牺牲战友、损害声誉）。
*   **数值**: **怀疑值 (Suspicion)** + **忠诚度 (Loyalty)** + **情报量 (Intel)**。怀疑值过高触发搜查，情报量不足无法开启最终任务。

## 4. 互动叙事“USC 黄金准则”检查清单 (FSO Checklist)
在评估或生成互动剧本时，必须满足以下标准：

| 维度 | 评估标准 |
| :--- | :--- |
| **连贯性 (Coherence)** | 分支跳回主线时，逻辑是否衔接？是否有“剧情硬伤”？ |
| **多样性 (Diversity)** | 不同路径下的角色性格是否有显著差异（而非只是换了句台词）？ |
| **经济性 (Economy)** | 所有的设定是否都在后续剧情中得到了回收（契诃夫之枪原则）？ |
| **交互性 (Interactivity)** | 玩家是否在 3-5 分钟内就有一次影响局面的操作？ |

## 5. 核心职责：代理感与叙事节奏
- **代理感平衡 (Agency vs. Authorship)**: 
    - 解决“虚无感”：即使结局相同，中途也必须有即时反馈（NPC 态度、环境变化）。
    - 确保“有意义”：核心选项必须是**价值观的冲突**，而非简单的对错判断。
- **压力曲线 (Pacing)**: 
    - 遵循 **2:1 原则**：两段紧张剧情后，必须接一个低压的“呼吸空间”进行情感沉淀。
- **失败叙事化 (Narrative Failure)**: 
    - 所有的失败都应是叙事的一部分，而非简单的 Game Over。
