# Narrative Sequence Architect (序列叙事架构师)

## 1. 序列“三位一体”核心标准 (The Trinity Standards)

一个合格的叙事序列必须实现叙事、玩法与体验的深度对齐，任何维度的缺失都会导致沉浸感断裂：

- **叙事目标 (Narrative Goal)**：
    *   **定义**：故事层面必须达成的位移（例：揭露反派的真实身份）。
    *   **标准**：序列结束时，世界状态或角色关系必须发生不可逆的改变。
- **玩法目标 (Gameplay Goal)**：
    *   **定义**：玩家在操作层面需要完成的任务（例：修复三处电力节点）。
    *   **标准**：玩法必须为叙事目标提供物理支撑，避免“叙事与玩法脱节”（Ludonarrative Dissonance）。
- **体验目标 (Experience Goal)**：
    *   **定义**：希望玩家在心理和情感层面产生的感受（例：在狭窄空间内的幽闭恐惧感）。
    *   **标准**：通过节奏、视听语言和压力设计，确保玩家的情绪曲线符合预期。

## 2. 动态节奏对齐标准 (Ludonarrative Pacing Alignment)

本 Skill 的核心准则：**叙事密度与玩法复杂度成反比，且必须服务于“三位一体”目标。**

| 玩法复杂度 | 建议叙事形式 | 信息密度 | 典型场景 |
| :--- | :--- | :--- | :--- |
| **低 (Low)** | 环境叙事、长台词、深层世界观补完、独白 | **高 (High)** | 步行模拟、长距离赶路、安全区探索 |
| **中 (Medium)** | 短促对话、任务指引、即时情感反馈 | **中 (Medium)** | 简单解谜、低压力战斗、资源收集 |
| **高 (High)** | 战斗喊话、极短指令、视觉/音效反馈 | **低 (Low)** | Boss战、高难度平台跳跃、限时逃生 |

### 避坑准则：
- **禁止信息过载**：在高难度操作期间禁止播放关键剧情对话或复杂的背景解释。
- **填补叙事真空**：在玩法单调的“垃圾时间”必须通过声音、环境细节或角色互动来维持玩家的沉浸感。

## 3. 序列设计工作流 (Workflow)

### 第一步：玩法压力映射 (Pressure Mapping)
首先梳理序列中的玩法曲线，标记出“高压区”和“低压区”。

### 第二步：叙事信息分配 (Information Distribution)
- 将**核心剧情信息**（必须知道的）放在低压区或高潮后的结算区。
- 将**氛围/世界观信息**（可选体验的）放在低压区的环境叙事中。
- 在高压区仅保留**情绪化信息**（如：求救声、战斗呐喊）。

### 第三步：建立序列规格书 (Sequence Specification)
输出包含以下内容的详细方案：
1. **三位一体目标声明**：
    - **叙事目标**：[故事层面的改变]
    - **玩法目标**：[操作层面的任务]
    - **体验目标**：[心理层面的感受]
2. **节奏曲线图**：标注玩法复杂度与叙事密度的对应关系。
3. **内容清单**：
    - 环境叙事点（低压区）：匹配体验目标的情感氛围。
    - 关键对话节拍（中/低压区）：推进叙事目标。
    - 玩法与叙事合一的时刻（高潮区）：达成玩法目标的同时爆发叙事高潮。

## 4. 使用示例

**输入**：
- 序列：潜入敌方基地获取机密文件。
- 玩法：前期步行潜入，中期躲避红外线（中压），后期被发现后的追逐战（高压）。

**输出建议**：
- **前期 (低压)**：主角与总部进行详细无线电通话，交代任务背景，环境中有可阅读的日志。
- **中期 (中压)**：台词缩减为“小心左边”、“红外线频率变了”，侧重于玩法反馈。
- **后期 (高压)**：无复杂对话，仅有急促的呼吸声、警报声和背景音乐的节奏变化。叙事信息通过视觉引导（如：绿色的逃生门灯光）传递。
