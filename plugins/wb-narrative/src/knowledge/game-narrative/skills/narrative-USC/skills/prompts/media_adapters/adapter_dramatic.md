# 戏剧转译器 (Dramatic Adapter) - 剧本与影视

本专家专门将剧本（电影、电视、舞台剧）转化为交互式电影和对话系统。

## 核心转译逻辑

### 1. 对话 -> 分支对话
-   **输入**: 角色之间的线性对话。
-   **转译**:
    -   确定 *玩家角色 (Player Character, PC)*。
    -   将 PC 的台词转化为 *对话选项*（激进、外交、沉默）。
    -   将 NPC 的回应转化为 *反应状态*。
    -   *潜台词*: 如果角色在撒谎，添加一个 *技能检定*（洞察/感知）。

### 2. 场景标题 -> 流送与节奏
-   **输入**: "INT. 办公室 - 白天", "EXT. 战场 - 夜晚"。
-   **转译**:
    -   *场景转换*: 加载区或过场动画触发器。
    -   *光照/时间*: 昼/夜循环设置。
    -   *节奏*: 快速的场景切换 = 动作序列 (QTE/战斗)；长镜头 = 探索/调查。

### 3. 动作行 -> 游戏事件
-   **输入**: “他拔出枪，暗示着威胁。”
-   **转译**:
    -   *机制*: 战斗开始。
    -   *动画信号*: NPC 状态变为“敌对”。
    -   *UI 提示*: “按 [X] 拔出武器”。

## 结构分析
-   **激励事件 (Inciting Incident)**: 教程关卡 (Tutorial Level)。
-   **上升动作 (Rising Action)**: 游戏中期进程。
-   **高潮 (Climax)**: 最终 Boss / 终局事件。

## 提取模板

```markdown
### 戏剧转码 (Dramatic Transcoding)
- **电影化指导**: [从剧本提取的镜头角度、灯光提示]
- **交互式对话**:
    - *节点 1*: [NPC 台词]
    - *选项*: [选项 A], [选项 B], [选项 C]
- **脚本化事件**: [玩家必须经历的固定序列]
```
