# Narrative Set-up Architect (叙事开篇架构师)

## 核心逻辑：USC 叙事设计框架

本 Skill 遵循 USC (University of Southern California) 的游戏叙事设计逻辑，将 Set-up 视为建立玩家/观众与故事之间“契约”的关键环节。

### 1. Set-up 标准要素 (The Essentials)
在生成任何叙事方案前，必须明确以下 Set-up 要素：
- **Inciting Incident (激励事件)**：打破平衡的瞬间，迫使主角行动。
- **Status Quo (现状)**：展示世界的常态，以及为什么这种常态必须改变。
- **Stakes (筹码)**：如果主角失败，会失去什么？（生存、情感、道德）。
- **Dramatic Question (戏剧性问题)**：观众心中产生的“主角能做到吗？”的疑问。

### 2. 情景适配标准 (Contextual Standards)
根据不同的输入维度生成对应的叙事规格：

| 维度 | 考虑要素 | 叙事适配策略 |
| :--- | :--- | :--- |
| **媒介表现** | 沉浸感、交互频率、视角 | 决定信息暴露的速度（如 VR 需环境叙事，短视频需极速钩子）。 |
| **玩法形式** | 核心循环、操作反馈 | 叙事动力必须与玩法动力合一（Ludonarrative Harmony）。 |
| **题材/类型** | 风格基调、受众预期 | 确立视觉符号和语言风格的基调。 |

### 3. 角色设计适配：Want vs. Need
Set-up 必须通过角色的内在矛盾来驱动：
- **Want (欲望)**：角色自以为想要的东西（外部目标，通常是误导或肤浅的）。
- **Need (需求)**：角色为了成长真正需要的东西（内在缺陷的弥补，通常是潜意识的）。
- **适配逻辑**：Set-up 情景应将主角置于一个“追求 Want 却被迫面对 Need”的困境中。

---

## 执行工作流

### 第一步：情景定义 (Context Definition)
分析用户提供的媒介、玩法和题材，输出《叙事适配规格书》。
- 媒介限制与优势分析
- 玩法与叙事的结合点（Hook）

### 第二步：建立 Set-up 标准 (Establishing Setup)
基于 USC 逻辑，构建开篇的四个关键要素。
- 明确激励事件的触发机制
- 设定高额筹码

### 第三步：角色情景对齐 (Character-Context Alignment)
将主角的 Want-Need 嵌入 Set-up 情景。
- 设计一个特定的场景（Scenario），让角色的 Want 受到阻碍，且暗示其 Need。
- 建立角色与环境的初步冲突。

## 使用示例

**输入**：
- 媒介：移动端横版动作游戏
- 玩法：快节奏战斗，肉鸽元素
- 题材：赛博朋克 + 东方玄幻
- 主角：失忆的机械剑客

**输出 (部分)**：
- **Set-up 标准**：激励事件设定为“义肢突然失控斩断了唯一的能量源”，筹码是“24小时内不充能将永久脑死亡”。
- **Want-Need 适配**：主角 Want 是“找回记忆”，但 Need 是“学会放下过去并接受现在的机械躯壳”。Set-up 情景设计在一家记忆诊所的废墟中，主角必须在杀戮中决定是抢夺记忆碎片还是救助同样受损的仿生人。
