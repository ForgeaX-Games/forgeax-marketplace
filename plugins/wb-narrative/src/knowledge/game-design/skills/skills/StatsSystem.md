# 一、系统概述

## 什么是属性系统

属性系统管理游戏中角色、敌人、物品等对象的数值属性，如生命值、攻击力、防御力等。它负责属性的存储、计算、修改和变化通知。

## 核心职责

| 职责 | 说明 |
|------|------|
| 属性存储 | 保存各种属性的基础值和当前值 |
| 属性计算 | 根据基础值和修饰器计算最终值 |
| 修饰器管理 | 添加、移除、更新属性修饰器 |
| 变化通知 | 属性变化时通知相关系统 |
| 上下限控制 | 确保属性值在合理范围内 |

## 适用游戏类型

| 游戏类型 | 复杂度 | 核心属性 |
|---------|--------|---------|
| **RPG** | 高 | HP/MP/ATK/DEF/SPD 等完整属性 |
| **ARPG** | 中高 | 战斗属性 + 暴击/闪避等 |
| **MOBA** | 中 | 攻击/防御/技能强度 |
| **Roguelike** | 中 | 基础属性 + 大量临时修饰 |
| **卡牌** | 低中 | 攻击/生命 |
| **休闲** | 低 | 简单数值 |

# 二、核心功能模块

## 1. 基础属性管理

**功能说明**：管理角色或对象的基础属性值。

| 功能 | 说明 |
|------|------|
| 属性定义 | 定义属性类型和初始值 |
| 属性获取 | 获取属性的当前值或基础值 |
| 属性设置 | 直接设置属性的基础值 |
| 属性增减 | 增加或减少属性值 |
| 批量操作 | 同时修改多个属性 |

**示例**：

> **角色创建**：创建一个战士角色，设置基础属性：HP=1000, ATK=100, DEF=50, SPD=30。
>
> **升级加点**：玩家升级后，基础攻击力从100增加到110，基础生命值从1000增加到1100。
>
> **装备加成**：装备一把武器后，攻击力属性增加50点。

---

## 2. 派生属性

**功能说明**：由基础属性计算得出的二级属性。

| 类型 | 说明 | 示例 |
|------|------|------|
| 公式派生 | 通过公式从基础属性计算 | 物理伤害 = ATK × 1.5 |
| 组合派生 | 多个属性组合计算 | 战斗力 = ATK + DEF×0.5 + HP×0.1 |
| 条件派生 | 满足条件才生效 | 低血量时攻击力×1.5 |
| 依赖派生 | 依赖其他派生属性 | 暴击期望 = 暴击率 × 暴击伤害 |

**示例**：

> **暴击伤害期望**：基础攻击力100，暴击率30%，暴击伤害200%。期望伤害 = 100 × (1 + 0.3 × (2-1)) = 130。
>
> **有效生命**：生命值1000，护甲100，护甲减伤30%。有效生命 = 1000 / (1 - 0.3) = 1428。
>
> **战斗力评分**：综合攻击、防御、生命等属性计算出一个战斗力数值，用于匹配和展示。

---

## 3. 当前值与最大值

**功能说明**：区分属性的当前值和最大值，如当前HP和最大HP。

| 功能 | 说明 |
|------|------|
| 最大值 | 属性的上限，受基础值和加成影响 |
| 当前值 | 属性的实际值，不超过最大值 |
| 消耗恢复 | 当前值的减少和恢复 |
| 溢出处理 | 最大值变化时当前值的处理 |

**示例**：

> **生命值系统**：角色最大HP=1000，当前HP=800。受到200伤害后当前HP=600。使用药水恢复300，当前HP=900。
>
> **最大值变化**：装备增加200最大HP，最大HP变为1200，当前HP保持900（可选择按比例调整）。
>
> **超量恢复**：当前HP=900，最大HP=1000，恢复200时，当前HP=1000（不超过最大值），或当前HP=1100（允许临时超量）。

---

## 4. 属性上下限

**功能说明**：限制属性值的范围，防止异常值。

| 功能 | 说明 |
|------|------|
| 下限 | 属性最小值（通常为0） |
| 上限 | 属性最大值（防止溢出） |
| 软上限 | 超过后收益递减 |
| 动态上限 | 上限可被技能/装备修改 |

**示例**：

> **护甲上限**：护甲减伤最高75%，即使护甲值很高也不能完全免疫物理伤害。
>
> **攻速上限**：攻击速度最高2.5次/秒，防止数值溢出导致的问题。
>
> **负面属性下限**：攻击力最低为1，即使受到-90%攻击力的debuff也能造成伤害。

---

## 5. 属性变化事件

**功能说明**：属性变化时发送事件通知相关系统。

| 事件 | 数据 | 用途 |
|------|------|------|
| 属性变化 | {属性名, 旧值, 新值, 变化量} | UI更新、触发效果 |
| 达到上限 | {属性名} | 满血提示、满蓝提示 |
| 达到下限 | {属性名} | 死亡判定、资源耗尽 |
| 临界触发 | {属性名, 阈值} | 低血量警告、斩杀线 |

**示例**：

> **血条更新**：HP从800变为600，发送属性变化事件。UI系统接收事件，更新血条显示。
>
> **死亡判定**：HP达到0，发送达到下限事件。战斗系统接收事件，触发死亡流程。
>
> **低血量警告**：HP低于30%最大值，发送临界触发事件。UI显示红色血条闪烁，音效提示。

# 三、属性类型设计

## 1. 常见属性分类

### 资源属性
| 属性 | 说明 | 特点 |
|------|------|------|
| 生命值 (HP) | 角色存活的核心 | 有当前值和最大值 |
| 魔法值 (MP) | 技能消耗资源 | 有当前值和最大值 |
| 体力/耐力 | 动作消耗资源 | 快速恢复 |
| 怒气/能量 | 战斗积累资源 | 战斗中积累 |
| 护盾 | 临时额外生命 | 优先消耗 |

### 攻击属性
| 属性 | 说明 | 特点 |
|------|------|------|
| 攻击力 (ATK) | 物理伤害基础 | 影响普攻和物理技能 |
| 魔法强度 (AP) | 魔法伤害基础 | 影响魔法技能 |
| 攻击速度 | 攻击频率 | 通常有上限 |
| 暴击率 | 暴击概率 | 0-100% |
| 暴击伤害 | 暴击倍率 | 通常150-300% |
| 穿透 | 无视部分防御 | 固定穿透/百分比穿透 |

### 防御属性
| 属性 | 说明 | 特点 |
|------|------|------|
| 防御力 (DEF) | 减少物理伤害 | 减伤公式计算 |
| 魔法抗性 (MR) | 减少魔法伤害 | 减伤公式计算 |
| 闪避率 | 完全躲避攻击 | 0-100%，通常有上限 |
| 格挡率 | 减少部分伤害 | 格挡时减伤50%等 |
| 韧性 | 减少控制时间 | 百分比减少 |

### 其他属性
| 属性 | 说明 | 特点 |
|------|------|------|
| 移动速度 | 角色移动快慢 | 有上下限 |
| 冷却缩减 | 减少技能CD | 通常上限40-45% |
| 生命恢复 | 每秒恢复HP | 战斗内/外可不同 |
| 经验加成 | 获得经验倍率 | 百分比加成 |
| 金币加成 | 获得金币倍率 | 百分比加成 |

---

## 2. 属性数据结构

**单属性结构**：
```
Stat {
  id: "attack"           // 属性标识
  name: "攻击力"          // 显示名称
  baseValue: 100         // 基础值
  currentValue: 150      // 当前计算值
  minValue: 0            // 最小值
  maxValue: 9999         // 最大值
  modifiers: [...]       // 修饰器列表
}
```

**资源属性结构**：
```
ResourceStat extends Stat {
  current: 800           // 当前值
  max: 1000              // 最大值
  regenRate: 5           // 每秒恢复
  regenDelay: 3          // 脱战后开始恢复延迟
}
```

# 四、修饰器系统

## 1. 修饰器类型

| 类型 | 说明 | 计算方式 | 示例 |
|------|------|---------|------|
| 固定加成 (Flat) | 直接增减固定数值 | +50 | 武器攻击力+50 |
| 百分比加成 (PercentAdd) | 基于基础值的百分比 | +20% | 力量提升20%攻击 |
| 百分比乘算 (PercentMult) | 基于当前值的乘算 | ×1.3 | 技能加成30% |
| 覆盖 (Override) | 直接设置最终值 | =500 | 固定攻击力500 |

**计算示例**：
```
基础攻击力: 100
固定加成: +50 (武器)
百分比加成: +30% (天赋)
百分比乘算: ×1.2 (Buff)

计算过程:
1. 基础值: 100
2. 固定加成: 100 + 50 = 150
3. 百分比加成: 150 × (1 + 0.3) = 195
4. 百分比乘算: 195 × 1.2 = 234

最终攻击力: 234
```

---

## 2. 修饰器优先级

**计算顺序**：
1. 收集所有修饰器
2. 按优先级排序
3. 先计算固定加成
4. 再计算百分比加成（相加后统一乘）
5. 最后计算百分比乘算（依次相乘）
6. 应用覆盖值（如果有）

**优先级示例**：
| 优先级 | 来源 | 类型 | 说明 |
|--------|------|------|------|
| 1 | 基础属性 | Base | 角色自身 |
| 2 | 装备 | Flat | 装备提供 |
| 3 | 被动技能 | Percent | 永久加成 |
| 4 | Buff | Percent | 临时加成 |
| 5 | 最终修正 | Mult | 特殊效果 |

---

## 3. 修饰器来源

| 来源 | 特点 | 示例 |
|------|------|------|
| 等级成长 | 永久，随等级增长 | 每级+5攻击力 |
| 装备 | 穿戴时生效 | 武器+100攻击 |
| 技能/天赋 | 解锁后永久 | 天赋+10%暴击 |
| Buff/Debuff | 临时，有持续时间 | 战吼+30%攻击10秒 |
| 光环 | 范围内生效 | 旗帜+20%攻击 |
| 状态 | 特定条件下 | 低血量+50%攻击 |
| 消耗品 | 使用后临时 | 药水+100攻击60秒 |

---

## 4. 时限修饰器

**功能说明**：带有持续时间的临时修饰器。

| 功能 | 说明 |
|------|------|
| 持续时间 | 修饰器生效时长 |
| 自动过期 | 时间到后自动移除 |
| 刷新时间 | 同类修饰器刷新持续时间 |
| 叠加规则 | 同类修饰器的叠加方式 |

**示例**：

> **增益Buff**：使用力量药水，获得"+50攻击力"修饰器，持续60秒。60秒后自动移除。
>
> **刷新机制**：已有力量Buff剩余30秒，再次使用药水，持续时间刷新为60秒。
>
> **叠加上限**：某Buff最多叠加5层，每层+10攻击。达到5层后再次获得只刷新时间。

# 五、属性计算流程

## 1. 完整计算流程

```
1. 获取基础值 (Base Value)
   └── 角色基础 + 等级成长

2. 收集所有修饰器
   ├── 装备修饰器
   ├── 技能修饰器
   ├── Buff修饰器
   └── 其他修饰器

3. 按类型分组
   ├── 固定加成组
   ├── 百分比加成组
   └── 百分比乘算组

4. 依次计算
   ├── 基础值 + Σ(固定加成)
   ├── × (1 + Σ(百分比加成))
   └── × Π(1 + 百分比乘算)

5. 应用上下限
   └── Clamp(最小值, 最大值)

6. 更新派生属性
   └── 重新计算依赖此属性的派生属性

7. 发送变化事件
   └── 通知UI和其他系统
```

---

## 2. 常见公式

### 伤害减免公式

**线性减免**：
```
减伤% = 护甲 / (护甲 + 常数)
示例：护甲100，常数=100
减伤% = 100 / (100 + 100) = 50%
```

**对数减免**：
```
减伤% = 1 - e^(-护甲/常数)
随护甲增加，收益递减
```

### 暴击期望

```
期望伤害 = 基础伤害 × (1 + 暴击率 × (暴击倍率 - 1))
示例：伤害100，暴击率30%，暴击伤害200%
期望 = 100 × (1 + 0.3 × 1) = 130
```

### 攻击间隔

```
攻击间隔 = 基础间隔 / (1 + 攻速加成)
示例：基础间隔1秒，攻速+50%
间隔 = 1 / 1.5 = 0.67秒
```

---

## 3. 脏标记优化

**功能说明**：只在属性被修改时重新计算，避免每帧计算所有属性。

| 功能 | 说明 |
|------|------|
| 脏标记 | 标记属性需要重新计算 |
| 延迟计算 | 获取属性值时才计算 |
| 依赖追踪 | 派生属性依赖的基础属性变化时标记 |
| 批量更新 | 多个修改后统一计算一次 |

**示例**：

> **装备更换**：玩家换装备，标记攻击力为脏。下次获取攻击力时重新计算。
>
> **批量Buff**：一次性获得多个Buff，先收集所有修改，最后统一计算一次。

# 六、各类型游戏应用

## 1. 传统RPG

### 属性设计
```
一级属性：力量、敏捷、智力、体质
二级属性：由一级属性派生
- 攻击力 = 力量 × 2 + 基础攻击
- 暴击率 = 敏捷 × 0.1%
- 魔法强度 = 智力 × 1.5
- 生命值 = 体质 × 10 + 基础生命
```

### 特点
- 属性点加点系统
- 装备提供一级或二级属性
- 属性成长曲线设计

---

## 2. ARPG / Diablo-like

### 属性设计
```
核心属性：攻击、防御、生命
特殊属性：
- 暴击率/暴击伤害
- 攻击速度
- 冷却缩减
- 元素伤害加成
- 技能等级加成
```

### 特点
- 大量随机词条
- 词条叠加上限
- 赛季机制重置

**示例**：
> 装备词条：+50攻击力, +15%暴击伤害, +2火焰技能等级
> 需要处理多种不同来源的加成叠加

---

## 3. MOBA

### 属性设计
```
基础属性：攻击力、护甲、魔抗、生命值、法力值
成长属性：每级增长值
特殊属性：攻速、暴击、穿透、冷却缩减
```

### 特点
- 属性随等级成长
- 装备提供属性
- 技能伤害有独立公式

**示例**：
> 基础攻击力60，每级成长3，当前10级
> 攻击力 = 60 + 3 × (10-1) = 87

---

## 4. Roguelike

### 属性设计
```
简单基础属性：攻击、生命、防御
大量临时修饰器：
- 词条/遗物提供各种加成
- 可能有负面属性
- 属性可能为0或负数
```

### 特点
- 修饰器数量多
- 需要处理极端情况
- 局内局外属性分离

**示例**：
> 玩家拾取遗物"玻璃大炮"：攻击力×3，最大生命值=1
> 需要支持这种极端修改

---

## 5. 卡牌游戏

### 属性设计
```
卡牌属性：攻击力、生命值
可能有：护甲、圣盾、嘲讽（标签）
```

### 特点
- 属性简单直观
- 更多靠技能效果
- 回合制数值设计
