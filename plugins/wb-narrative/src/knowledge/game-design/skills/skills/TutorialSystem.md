# 一、系统概述

## 什么是教程系统

教程系统引导新玩家学习游戏操作和玩法，通过渐进式的教学帮助玩家快速上手。

## 核心职责

| 职责 | 说明 |
|------|------|
| 教程管理 | 管理教程流程 |
| 步骤控制 | 控制教程步骤 |
| 引导展示 | 展示引导UI |
| 进度追踪 | 追踪完成进度 |
| 强制/可选 | 区分必要和可选教程 |

## 适用游戏类型

几乎所有游戏都需要教程系统。

# 三、教程设计

## 1. 教程结构

```
TutorialConfig {
  id: "basic_combat"
  name: "战斗入门"
  
  // 触发条件
  trigger: {
    type: "level",
    value: 2
  }
  
  // 是否可跳过
  skippable: false
  
  // 步骤列表
  steps: [
    {
      id: "step_1",
      type: "dialog",
      content: "让我教你如何战斗",
      next: "step_2"
    },
    {
      id: "step_2",
      type: "highlight",
      target: "attack_button",
      content: "点击这里进行攻击",
      waitFor: "button_click",
      next: "step_3"
    },
    {
      id: "step_3",
      type: "action",
      content: "攻击敌人3次",
      condition: { type: "kill_count", value: 3 },
      next: null
    }
  ]
  
  // 完成奖励
  rewards: [
    { type: "gold", amount: 100 }
  ]
}
```

# 七、功能速查表

## 教程系统方法

| 方法 | 说明 | 参数 |
|------|------|------|
| startTutorial | 开始教程 | 教程ID |
| nextStep | 下一步 | - |
| prevStep | 上一步 | - |
| skipTutorial | 跳过教程 | - |
| completeTutorial | 完成教程 | - |
| isTutorialComplete | 检查完成 | 教程ID |
| showHighlight | 显示高亮 | 目标, 提示 |
| hideHighlight | 隐藏高亮 | - |

## 教程事件

| 事件 | 数据 | 说明 |
|------|------|------|
| tutorial:started | {tutorialId} | 教程开始 |
| tutorial:step | {tutorialId, stepId} | 步骤变化 |
| tutorial:completed | {tutorialId} | 教程完成 |
| tutorial:skipped | {tutorialId} | 教程跳过 |
