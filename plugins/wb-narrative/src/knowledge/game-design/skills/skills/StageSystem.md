# 一、系统概述

## 什么是关卡系统

关卡系统管理游戏中的独立关卡单元，包括关卡解锁、进入、挑战和评价等功能。

## 核心职责

| 职责 | 说明 |
|------|------|
| 关卡管理 | 管理所有关卡 |
| 解锁控制 | 控制关卡解锁 |
| 关卡进入 | 处理关卡进入 |
| 评价记录 | 记录通关评价 |
| 进度保存 | 保存关卡进度 |

## 适用游戏类型

| 游戏类型 | 关卡特点 | 典型作品 |
|---------|---------|---------|
| **休闲** | 大量短关卡 | 糖果消消乐 |
| **塔防** | 策略关卡 | 保卫萝卜 |
| **平台** | 线性关卡 | 马里奥 |
| **解谜** | 谜题关卡 | 纪念碑谷 |
| **音游** | 歌曲关卡 | 音乐游戏 |

# 三、关卡设计

## 1. 关卡配置

```
StageConfig {
  id: "stage_1_5"
  name: "草原冒险"
  chapter: 1
  order: 5
  
  // 解锁条件
  unlockConditions: [
    { type: "stage_complete", stageId: "stage_1_4" },
    { type: "star_count", value: 10 }
  ]
  
  // 进入消耗
  entryCost: {
    stamina: 5
  }
  
  // 难度
  difficulties: ["normal", "hard", "nightmare"]
  
  // 星级条件
  starConditions: [
    { type: "complete", star: 1 },
    { type: "time", value: 120, star: 2 },
    { type: "score", value: 10000, star: 3 }
  ]
  
  // 奖励
  rewards: {
    first: [{ type: "gem", amount: 50 }],
    normal: [{ type: "gold", amount: 100 }],
    star3: [{ type: "item", id: "chest", amount: 1 }]
  }
  
  // 限制
  dailyLimit: 3,
  attemptLimit: 0     // 0=无限
}
```

# 七、功能速查表

## 关卡系统方法

| 方法 | 说明 | 参数 |
|------|------|------|
| getStages | 获取关卡列表 | 章节? |
| getStageInfo | 获取关卡信息 | 关卡ID |
| isUnlocked | 检查是否解锁 | 关卡ID |
| enterStage | 进入关卡 | 关卡ID, 难度? |
| exitStage | 退出关卡 | - |
| completeStage | 完成关卡 | 结果数据 |
| getStarCount | 获取星数 | 关卡ID |
| getBestRecord | 获取最佳记录 | 关卡ID |
| getTotalStars | 获取总星数 | 章节? |

## 关卡事件

| 事件 | 数据 | 说明 |
|------|------|------|
| stage:entered | {stageId, difficulty} | 进入关卡 |
| stage:completed | {stageId, result, stars} | 完成关卡 |
| stage:failed | {stageId, reason} | 关卡失败 |
| stage:unlocked | {stageId} | 关卡解锁 |
| stage:record | {stageId, type, value} | 刷新记录 |
| chapter:completed | {chapterId} | 完成章节 |
