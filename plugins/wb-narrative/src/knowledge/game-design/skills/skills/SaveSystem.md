# 一、系统概述

## 什么是存档系统

存档系统负责将游戏进度持久化保存，让玩家可以在退出游戏后恢复之前的状态。它管理着游戏数据的序列化、存储、读取和版本迁移。

## 核心职责

| 职责 | 说明 |
|------|------|
| 数据收集 | 从各系统收集需要保存的数据 |
| 序列化 | 将游戏对象转换为可存储格式 |
| 持久化 | 写入本地存储或云端 |
| 反序列化 | 将存储数据还原为游戏对象 |
| 数据应用 | 将读取的数据应用到各系统 |
| 版本管理 | 处理不同版本存档的兼容 |

## 适用范围

| 游戏类型 | 核心需求 |
|---------|---------|
| **RPG/冒险** | 多槽位、自动存档、详细进度 |
| **开放世界** | 大量数据、增量保存、快速加载 |
| **Roguelike** | 单存档、死亡删档、进度解锁 |
| **休闲/三消** | 云同步、关卡进度、累计奖励 |
| **竞技/联机** | 服务器存储、防作弊 |
| **AVG/视觉小说** | 大量存档位、截图预览、回溯 |

# 二、核心功能模块

## 1. 存档槽位管理

**功能说明**：管理多个存档位置，每个槽位独立保存一套完整游戏数据。

| 功能 | 说明 |
|------|------|
| 创建槽位 | 新建空白存档槽 |
| 选择槽位 | 切换当前操作的槽位 |
| 删除槽位 | 删除指定存档 |
| 复制槽位 | 复制存档到新槽位 |
| 槽位信息 | 获取槽位元数据（时间、进度等） |

**示例**：

> **RPG多存档**：游戏提供10个存档槽。玩家可以在不同槽位保存多个周目的进度，或在关键剧情选择前创建多个分支存档。
>
> **存档覆盖确认**：玩家选择已有存档的槽位保存时，弹出"是否覆盖？"确认对话框。
>
> **槽位预览**：存档选择界面显示每个槽位的截图、游戏时长、当前章节、存档时间等信息。

---

## 2. 手动存档

**功能说明**：玩家主动触发的存档操作。

| 功能 | 说明 |
|------|------|
| 快速存档 | 一键保存到指定槽位 |
| 存档界面 | 打开存档选择界面 |
| 存档命名 | 为存档添加备注 |
| 存档截图 | 保存当前画面作为预览 |
| 存档确认 | 保存成功/失败提示 |

**示例**：

> **菜单存档**：玩家打开暂停菜单，选择"保存游戏"，进入存档界面选择槽位保存。
>
> **快捷键存档**：玩家按 F5 快速保存到上次使用的槽位，屏幕角落显示"已保存"提示。
>
> **存档备注**：保存时可选择输入备注，如"准备打Boss前"、"选了A路线"。

---

## 3. 自动存档

**功能说明**：游戏自动在特定时机保存进度。

| 触发时机 | 说明 |
|---------|------|
| 检查点 | 到达特定位置自动存档 |
| 关卡开始/结束 | 进入/完成关卡时保存 |
| 重要事件 | 剧情节点、获得重要物品 |
| 定时保存 | 每隔N分钟自动保存 |
| 退出保存 | 退出游戏时自动保存 |
| 区域切换 | 进入新区域时保存 |

**示例**：

> **检查点存档**：玩家到达存档点（如篝火、传送门），自动保存进度，屏幕显示存档图标动画。
>
> **定时自动存档**：每5分钟自动保存一次，使用专门的"自动存档"槽位，不覆盖手动存档。
>
> **退出存档**：玩家关闭游戏时，自动保存当前进度到"继续"槽位。下次启动可选择"继续游戏"。

---

## 4. 读取存档

**功能说明**：从存储中读取数据并恢复游戏状态。

| 功能 | 说明 |
|------|------|
| 选择读取 | 从存档界面选择槽位读取 |
| 继续游戏 | 读取最近的存档 |
| 读取确认 | 确认丢失当前进度 |
| 加载画面 | 读取过程的加载界面 |
| 恢复状态 | 将数据应用到各系统 |

**示例**：

> **主菜单继续**：玩家在主菜单点击"继续游戏"，自动读取最近的存档并进入游戏。
>
> **读取确认**：玩家在游戏中选择读取其他存档，弹出"当前进度将丢失，是否继续？"确认。
>
> **状态恢复**：读取存档后，玩家出现在存档时的位置，背包物品、任务进度、已解锁区域等全部恢复。

---

## 5. 存档删除

**功能说明**：删除不需要的存档释放空间。

| 功能 | 说明 |
|------|------|
| 单个删除 | 删除指定槽位 |
| 批量删除 | 删除多个槽位 |
| 删除确认 | 二次确认防误删 |
| 清空全部 | 删除所有存档 |

**示例**：

> **删除确认**：玩家选择删除存档，弹出"确定删除此存档？此操作无法撤销"确认对话框。
>
> **敏感操作保护**：删除存档需要长按确认按钮3秒，或输入确认文字，防止误操作。

# 三、数据结构设计

## 1. 存档数据分类

| 分类 | 说明 | 示例 |
|------|------|------|
| 元数据 | 存档本身的信息 | 版本号、存档时间、游戏时长 |
| 玩家数据 | 玩家角色相关 | 等级、属性、位置、外观 |
| 背包数据 | 物品和装备 | 物品列表、装备槽、货币 |
| 进度数据 | 游戏进展 | 任务状态、解锁内容、成就 |
| 世界数据 | 世界状态 | NPC状态、物品位置、开关状态 |
| 设置数据 | 游戏设置 | 音量、按键绑定、显示设置 |

## 3. 可序列化接口

**功能说明**：定义统一的数据导出/导入接口，让各系统实现自己的序列化逻辑。

| 方法 | 说明 |
|------|------|
| serialize() | 将当前状态导出为可存储数据 |
| deserialize(data) | 从存储数据恢复状态 |
| getVersion() | 获取数据版本号 |
| migrate(oldData, oldVersion) | 迁移旧版本数据 |

**示例**：

> **背包系统序列化**：遍历所有物品，导出物品ID和数量。对于有特殊属性的物品（如强化装备），额外导出实例数据。
>
> **背包系统反序列化**：根据物品ID查找物品配置，创建物品实例，应用实例数据（如强化等级）。

# 四、存储方式

## 1. 本地存储

### LocalStorage（Web/移动端）
| 特点 | 说明 |
|------|------|
| 容量 | 约5-10MB |
| 速度 | 快 |
| 持久性 | 持久，清除浏览器数据会丢失 |
| 适用 | 休闲游戏、小型存档 |

**示例**：三消游戏将关卡进度、累计分数、设置保存在 LocalStorage 中。

### IndexedDB（Web）
| 特点 | 说明 |
|------|------|
| 容量 | 较大（数百MB） |
| 速度 | 较快 |
| 持久性 | 持久 |
| 适用 | 大型Web游戏 |

**示例**：Web版RPG将大量物品数据、任务数据、世界状态保存在 IndexedDB 中。

### 文件系统（桌面/移动端原生）
| 特点 | 说明 |
|------|------|
| 容量 | 仅受磁盘限制 |
| 速度 | 取决于磁盘 |
| 持久性 | 持久 |
| 适用 | 大型桌面游戏 |

**示例**：PC版开放世界游戏将存档保存为文件，每个槽位一个文件夹，包含主存档、设置、截图。

---

## 2. 云存储

| 功能 | 说明 |
|------|------|
| 云端同步 | 存档上传到服务器 |
| 多设备共享 | 在不同设备使用同一存档 |
| 备份恢复 | 设备丢失后恢复存档 |
| 冲突处理 | 处理多设备同时修改的冲突 |

**示例**：

> **手游云存档**：玩家在手机A玩到50级，云同步后在手机B登录，自动下载存档继续游戏。
>
> **冲突处理**：玩家在离线时两台设备都玩了，联网后提示"检测到多个存档，请选择保留哪个"或"合并两个存档"。

---

## 3. 服务器存储

| 功能 | 说明 |
|------|------|
| 权威存储 | 服务器为数据权威来源 |
| 防作弊 | 客户端无法篡改关键数据 |
| 实时保存 | 关键操作立即保存 |
| 定期同步 | 定时将状态同步到服务器 |

**示例**：

> **MMORPG**：玩家数据存储在服务器，客户端只是展示。获得物品、完成任务等操作由服务器验证并保存。
>
> **竞技游戏**：段位、胜负记录存储在服务器，确保公平性。

# 五、高级功能

## 1. 版本迁移

**功能说明**：当游戏更新导致存档格式变化时，自动将旧版存档转换为新版。

| 功能 | 说明 |
|------|------|
| 版本检测 | 读取存档时检测版本号 |
| 迁移脚本 | 每个版本升级对应一个迁移脚本 |
| 链式迁移 | 从旧版本逐级升级到最新版 |
| 迁移备份 | 迁移前备份原存档 |

**示例**：

> **属性系统改版**：1.0版存档中攻击力字段叫"atk"，2.0版改为"attack"。迁移脚本将"atk"的值复制到"attack"。
>
> **新增系统**：1.5版新增宠物系统，旧存档没有宠物数据。迁移时为旧存档添加空的宠物数据结构。
>
> **链式迁移**：1.0版存档读取时，依次执行 1.0→1.1、1.1→1.2、1.2→2.0 的迁移脚本，最终转换为最新格式。

---

## 2. 增量保存

**功能说明**：只保存发生变化的数据，而非完整存档。

| 功能 | 说明 |
|------|------|
| 变化追踪 | 标记哪些数据发生了变化 |
| 差异计算 | 计算与上次保存的差异 |
| 增量写入 | 只写入变化的部分 |
| 完整备份 | 定期保存完整存档 |

**示例**：

> **开放世界自动存档**：玩家移动时位置每秒都在变，但只有位置数据需要更新。增量保存只写入新位置，而非整个世界状态。
>
> **优化存档速度**：完整存档需要3秒，增量保存只需要0.1秒，适合高频自动存档。

---

## 3. 存档压缩与加密

### 压缩
| 功能 | 说明 |
|------|------|
| 数据压缩 | 减少存档文件大小 |
| 压缩算法 | GZIP、LZ4等 |
| 解压读取 | 读取时自动解压 |

**示例**：原始存档数据500KB，压缩后50KB，减少90%存储空间和上传时间。

### 加密
| 功能 | 说明 |
|------|------|
| 加密保存 | 存档数据加密后存储 |
| 解密读取 | 读取时解密 |
| 防篡改 | 校验存档完整性 |
| 混淆格式 | 非标准格式防止直接编辑 |

**示例**：

> **防作弊**：单机游戏加密存档，防止玩家直接修改金币数量。
>
> **完整性校验**：存档末尾附加校验和，读取时验证。若校验失败，提示"存档已损坏"。

---

## 4. 存档回滚

**功能说明**：保留多个历史版本的存档，允许回滚到之前的状态。

| 功能 | 说明 |
|------|------|
| 历史版本 | 保留最近N次存档 |
| 回滚选择 | 选择回滚到哪个版本 |
| 自动清理 | 超出限制时删除最旧版本 |

**示例**：

> **AVG后悔药**：玩家做出选择后发现不满意，可以回滚到选择前的存档。
>
> **意外覆盖**：玩家误操作覆盖了重要存档，可以从历史版本恢复。

---

## 5. 存档导入导出

**功能说明**：将存档导出为文件或从文件导入。

| 功能 | 说明 |
|------|------|
| 导出文件 | 将存档导出为独立文件 |
| 导入文件 | 从文件导入存档 |
| 分享存档 | 将存档分享给其他玩家 |
| 格式验证 | 验证导入文件的有效性 |

**示例**：

> **存档分享**：玩家将自己的游戏存档导出，分享到论坛。其他玩家下载后导入，可以直接体验高级角色。
>
> **设备迁移**：玩家换手机，将旧手机存档导出，通过文件传输到新手机导入。

---

## 6. 新游戏+（New Game Plus）

**功能说明**：通关后开始新游戏时，保留部分进度。

| 功能 | 说明 |
|------|------|
| 继承项目 | 定义哪些数据可以继承 |
| 重置项目 | 定义哪些数据需要重置 |
| 难度调整 | 新周目可能有难度变化 |
| 特殊奖励 | 新周目专属内容 |

**示例**：

> **继承与重置**：
> - 继承：角色等级、技能、收集进度
> - 重置：主线任务、剧情状态、地图探索
> - 新增：更高难度敌人、隐藏Boss、特殊装备

# 六、各类型游戏应用

## 1. RPG / JRPG

### 存档特点
| 特点 | 说明 |
|------|------|
| 多槽位 | 10-99个存档槽 |
| 存档点 | 特定位置才能存档 |
| 详细预览 | 截图、时长、进度 |
| 剧情存档 | 关键选择前自动存档 |

### 需要保存的数据
```
- 角色：等级、属性、位置、外观
- 队伍：队伍成员、阵型
- 背包：物品、装备、货币
- 任务：主线进度、支线状态
- 剧情：剧情标记、对话历史
- 世界：探索进度、NPC状态、开关状态
- 成就：解锁成就
- 统计：游戏时长、击杀数、死亡数
```

**示例**：

> **存档点系统**：玩家只能在城镇的旅馆或迷宫中的存档水晶处保存游戏。存档时显示当前位置、等级、游戏时间。
>
> **剧情分支存档**：进入重要剧情选择前，系统自动创建"剧情存档"，玩家日后可从此处选择不同路线。


**示例**：

>
> **货币掉落**：玩家死亡时货币掉落在死亡地点。存档记录货币位置，玩家有一次机会取回。

> **局外解锁**：玩家用累计货币解锁新角色、新装备。这些解锁信息永久保存，影响后续所有局。

# 七、安全与优化

## 1. 数据安全

| 措施 | 说明 |
|------|------|
| 加密存储 | 敏感数据加密 |
| 完整性校验 | 检测存档篡改 |
| 备份策略 | 保留多个备份 |
| 权限控制 | 限制存档访问权限 |

---

## 2. 性能优化

| 优化 | 说明 |
|------|------|
| 异步保存 | 保存操作不阻塞主线程 |
| 增量保存 | 只保存变化的数据 |
| 延迟写入 | 合并多次保存请求 |
| 压缩数据 | 减少IO量 |
| 分块加载 | 按需加载数据 |

**示例**：

> **异步保存**：自动存档在后台线程进行，玩家操作不会卡顿。
>
> **延迟写入**：1秒内的多次修改合并为一次写入，减少IO次数。

---

## 3. 错误处理

| 情况 | 处理 |
|------|------|
| 保存失败 | 重试，提示玩家 |
| 读取失败 | 尝试恢复，加载备份 |
| 存档损坏 | 提示损坏，提供备份 |
| 版本不兼容 | 尝试迁移，失败则提示 |
| 空间不足 | 提示清理空间 |

**示例**：

> **存档损坏恢复**：读取存档失败，自动尝试读取备份文件。若备份也损坏，提示玩家"存档已损坏，是否从最近自动存档恢复？"
