# 一、系统概述

## 什么是交互系统

交互系统管理玩家与游戏世界中物体的互动，如拾取物品、开门、与NPC对话、操作机关等。

## 核心职责

| 职责 | 说明 |
|------|------|
| 交互检测 | 检测可交互物体 |
| 提示显示 | 显示交互提示 |
| 交互执行 | 执行交互行为 |
| 优先级管理 | 多物体时选择优先级 |

## 适用游戏类型

| 游戏类型 | 交互重要性 | 特点 |
|---------|----------|------|
| **解谜** | 核心 | 机关交互 |
| **RPG** | 高 | NPC/物品交互 |
| **开放世界** | 高 | 大量可交互物 |
| **生存** | 高 | 采集交互 |
| **动作** | 中 | 场景交互 |

---

## 2. 交互配置

```
InteractableConfig {
  id: "treasure_chest"
  type: "container"
  
  // 提示
  prompt: "打开宝箱"
  promptKey: "F"
  
  // 条件
  conditions: [
    { type: "has_key", keyId: "chest_key" }
  ]
  
  // 交互
  interactionTime: 1.5     // 交互时间
  animation: "open_chest"   // 交互动画
  
  // 效果
  effects: [
    { type: "open_loot", lootTable: "chest_loot" }
  ]
  
  // 一次性
  oneTime: true
}
```

# 六、功能速查表

## 交互系统方法

| 方法 | 说明 | 参数 |
|------|------|------|
| detectInteractables | 检测可交互物 | 范围 |
| showPrompt | 显示提示 | 物体 |
| hidePrompt | 隐藏提示 | - |
| interact | 执行交互 | 物体 |
| canInteract | 检查能否交互 | 物体 |
| setHighlight | 设置高亮 | 物体, 开关 |

## 交互事件

| 事件 | 数据 | 说明 |
|------|------|------|
| interact:started | {target} | 开始交互 |
| interact:completed | {target, result} | 完成交互 |
| interact:cancelled | {target} | 取消交互 |
| interact:failed | {target, reason} | 交互失败 |
