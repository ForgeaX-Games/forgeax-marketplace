# 一、系统概述

## 什么是AI系统

AI系统控制非玩家角色(NPC)的行为决策，包括敌人的战斗逻辑、友方单位的辅助行为、中立NPC的日常活动等。

## 核心职责

| 职责 | 说明 |
|------|------|
| 决策制定 | 决定NPC的下一步行动 |
| 目标选择 | 选择攻击/追踪目标 |
| 路径规划 | 计算移动路径 |
| 感知处理 | 检测周围环境和敌人 |
| 行为执行 | 执行决策的具体行为 |

## 适用游戏类型

| 游戏类型 | AI复杂度 | 特点 |
|---------|---------|------|
| **开放世界** | 高 | 复杂行为树、日程系统 |
| **RTS** | 高 | 群体AI、战术决策 |
| **ARPG/ACT** | 中高 | 战斗AI、Boss模式 |
| **RPG** | 中 | 回合制决策 |
| **塔防** | 低 | 简单寻路 |
| **休闲** | 低 | 基础逻辑 |

# 二、核心功能模块

## 1. 有限状态机 (FSM)

**功能说明**：AI通过不同状态之间的切换来控制行为。

| 功能 | 说明 |
|------|------|
| 状态定义 | 定义各种行为状态 |
| 状态切换 | 满足条件时切换状态 |
| 状态更新 | 每帧执行当前状态逻辑 |
| 进入/退出 | 状态变化时的回调 |

### 常见状态
```
巡逻 (Patrol)
  └─发现敌人─▶ 追击 (Chase)
                   └─进入攻击范围─▶ 攻击 (Attack)
                                       └─敌人逃离─▶ 追击
                                       └─敌人死亡─▶ 巡逻
                   └─丢失目标─▶ 搜索 (Search)
                                  └─超时─▶ 巡逻
```

**示例**：

> **守卫AI**：
> - 巡逻状态：沿预设路径移动
> - 发现玩家：切换到追击状态，向玩家移动
> - 接近玩家：切换到攻击状态，执行攻击
> - 玩家逃离视野：切换到搜索状态，在最后位置附近搜索
> - 搜索超时：返回巡逻状态

---

## 2. 行为树 (Behavior Tree)

**功能说明**：更复杂的决策结构，支持并行和条件分支。

### 节点类型
| 类型 | 说明 | 示例 |
|------|------|------|
| 顺序 (Sequence) | 依次执行，一个失败则失败 | 检查→移动→攻击 |
| 选择 (Selector) | 依次尝试，一个成功则成功 | 攻击或逃跑或待机 |
| 并行 (Parallel) | 同时执行多个 | 移动同时攻击 |
| 条件 (Condition) | 检查条件 | 敌人在范围内？ |
| 动作 (Action) | 执行具体行为 | 移动到目标 |
| 装饰 (Decorator) | 修改子节点行为 | 重复、反转 |

**示例行为树**：
```
[选择器] 战斗行为
├── [顺序] 攻击流程
│   ├── [条件] 有目标
│   ├── [条件] 在攻击范围
│   └── [动作] 执行攻击
├── [顺序] 追击流程
│   ├── [条件] 有目标
│   ├── [条件] 不在攻击范围
│   └── [动作] 移动到目标
└── [动作] 待机
```

## 3. 感知系统

**功能说明**：AI感知周围环境的能力。

| 感知类型 | 说明 | 参数 |
|---------|------|------|
| 视觉 | 视野范围内检测 | 视野角度、视野距离 |
| 听觉 | 声音范围检测 | 听觉半径 |
| 触觉 | 接触检测 | 碰撞体 |
| 第六感 | 被攻击时感知 | 无需视线 |

**示例**：

> **视觉感知**：守卫视野90度，距离15米。玩家进入视野范围且无遮挡，被发现。
>
> **听觉感知**：玩家奔跑产生声音，10米内敌人会被吸引过来调查。
>
> **潜行机制**：蹲下移动减少声音半径50%，草丛内视觉检测无效。

---

## 4. 仇恨系统

**功能说明**：管理AI的目标优先级。

| 功能 | 说明 |
|------|------|
| 仇恨生成 | 受到伤害/治疗产生仇恨 |
| 仇恨排序 | 按仇恨值排序目标列表 |
| 仇恨衰减 | 随时间降低仇恨值 |
| 嘲讽 | 强制成为最高仇恨 |
| 仇恨转移 | 将仇恨转移给他人 |

**示例**：

> **Boss仇恨**：坦克造成1000伤害产生1000仇恨，DPS造成2000伤害产生2000仇恨。坦克使用嘲讽，仇恨值×3，成为第一。
>
> **仇恨衰减**：脱战后仇恨每秒衰减10%，30秒后完全重置。

# 三、行为模式

## 1. 巡逻行为

| 模式 | 说明 |
|------|------|
| 路点巡逻 | 按顺序访问预设路点 |
| 往返巡逻 | 在两点间往返 |
| 循环巡逻 | 路点循环 |
| 随机巡逻 | 在区域内随机移动 |
| 警戒巡逻 | 原地站岗，偶尔转向 |

**示例**：
> 守卫沿 A→B→C→D→A 路点循环巡逻，每个路点停留3秒观察。

---

## 2. 战斗行为

| 行为 | 说明 |
|------|------|
| 接近攻击 | 靠近目标后攻击 |
| 远程攻击 | 保持距离攻击 |
| 包围攻击 | 多个敌人包围目标 |
| 回避攻击 | 攻击后拉开距离 |
| 掩体战斗 | 利用掩体射击 |

**示例**：
> **近战敌人**：移动到玩家近身位置，执行攻击，攻击后短暂后退避免反击。
>
> **弓箭手**：与玩家保持10米距离，玩家靠近则后退，保持射击距离。

---

## 3. 逃跑/撤退

| 条件 | 行为 |
|------|------|
| 低血量 | 血量低于20%时逃跑 |
| 寡不敌众 | 敌人数量过多时撤退 |
| 战术撤退 | 拉开距离等技能冷却 |
| 呼叫增援 | 撤退同时呼叫援军 |

**示例**：
> 盗贼血量低于30%时尝试逃跑，使用烟雾弹后隐身逃离，15秒后恢复追击。

---

## 4. 协作行为

| 行为 | 说明 |
|------|------|
| 编队移动 | 保持阵型移动 |
| 协同攻击 | 同时攻击同一目标 |
| 角色分工 | 坦克吸引，DPS输出 |
| 援护 | 掩护队友撤退 |

**示例**：
> 三个敌人发现玩家：一个正面吸引注意，两个绕后包抄。

# 四、寻路系统

## 1. 寻路算法

| 算法 | 适用场景 | 特点 |
|------|---------|------|
| A* | 网格地图 | 最常用，性能好 |
| Dijkstra | 无权重需求 | 保证最短路径 |
| NavMesh | 3D场景 | 支持复杂地形 |
| 流场 | 大量单位 | RTS首选 |

---

## 2. 导航功能

| 功能 | 说明 |
|------|------|
| 路径计算 | 计算到目标的路径 |
| 路径跟随 | 沿路径移动 |
| 动态避障 | 避开移动障碍物 |
| 路径更新 | 目标移动时重新计算 |
| 区域成本 | 不同区域移动成本不同 |

**示例**：
> AI计算到玩家的路径，玩家移动后超过5米，重新计算路径。遇到其他NPC时自动绕开。

---

## 3. 群体移动

| 功能 | 说明 |
|------|------|
| 分离 | 避免与其他单位重叠 |
| 对齐 | 与附近单位方向一致 |
| 聚集 | 向群体中心靠拢 |
| 避障 | 群体避开障碍物 |

# 五、高级AI机制

## 1. Boss阶段系统

**功能说明**：Boss根据血量切换不同行为阶段。

| 阶段 | 血量 | 行为特点 |
|------|------|---------|
| 阶段1 | 100%-70% | 基础攻击模式 |
| 阶段2 | 70%-40% | 增加新技能 |
| 阶段3 | 40%-10% | 狂暴模式 |
| 最终阶段 | <10% | 疯狂攻击 |

**示例**：
> **龙王Boss**：
> - 阶段1：火焰吐息、尾巴横扫
> - 阶段2：召唤小龙、飞行轰炸
> - 阶段3：持续喷火、地面火焰

---

## 2. 难度调节

| 方式 | 说明 |
|------|------|
| 数值调节 | 调整AI属性 |
| 行为调节 | 调整AI策略复杂度 |
| 反应调节 | 调整AI反应速度 |
| 容错调节 | 调整AI失误率 |

**示例**：
> **简单难度**：敌人攻击前有明显预警，攻击间隔长，命中后不追击。
>
> **困难难度**：敌人快速攻击，主动追击，会协作包围玩家。

---

## 3. 日程系统

**功能说明**：NPC按照时间表执行日常活动。

| 时间 | 行为 |
|------|------|
| 6:00 | 起床 |
| 7:00 | 前往工作地点 |
| 12:00 | 午餐 |
| 18:00 | 返回家中 |
| 22:00 | 睡觉 |

**示例**：
> 村民早上去农田劳作，中午到酒馆吃饭，晚上回家休息。下雨天待在室内。

# 六、各类型游戏应用

## 1. 动作游戏 (ACT)

### AI特点
- 攻击模式多样
- 有明显攻击预警
- 群体配合

### 敌人AI示例
```
近战杂兵：
- 接近玩家
- 攻击前摇0.5秒（警告）
- 攻击后硬直1秒
- 3个以上时包围攻击

远程敌人：
- 保持距离8-12米
- 玩家靠近则后撤
- 射击后换位置
```

---

## 2. RTS

### AI特点
- 资源管理
- 战术决策
- 群体控制

### AI层次
```
战略层：决定整体策略（进攻/防守/发展）
战术层：分配部队执行任务
单位层：个体寻路和战斗
```

---

## 3. 开放世界

### AI特点
- 复杂日程
- 社会系统
- 记忆系统

### NPC行为
```
日常行为：工作、休息、社交
反应行为：对玩家行为的反应
记忆系统：记住玩家的行为
```
