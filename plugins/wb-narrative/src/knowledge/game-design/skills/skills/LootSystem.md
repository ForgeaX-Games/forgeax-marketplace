# 一、系统概述

## 什么是掉落系统

掉落系统管理游戏中物品的随机生成，如怪物掉落、宝箱开启、抽卡等。

## 核心职责

| 职责 | 说明 |
|------|------|
| 掉落计算 | 根据掉落表计算产物 |
| 概率控制 | 管理掉落概率 |
| 保底机制 | 实现保底逻辑 |
| 掉落展示 | 物品掉落动画 |

## 适用游戏类型

| 游戏类型 | 掉落复杂度 | 特点 |
|---------|----------|------|
| **Diablo-like** | 极高 | 复杂词条掉落 |
| **手游** | 高 | 抽卡系统 |
| **MMORPG** | 高 | Boss掉落 |
| **Roguelike** | 中 | 随机道具 |
| **RPG** | 中 | 怪物掉落 |

# 二、核心功能模块

## 1. 掉落触发

**功能说明**：何时触发掉落计算。

| 触发方式 | 说明 |
|---------|------|
| 击杀怪物 | 怪物死亡时掉落 |
| 开启宝箱 | 打开宝箱时 |
| 完成副本 | 通关奖励 |
| 任务奖励 | 任务完成时 |
| 抽卡/抽奖 | 消耗资源抽取 |
| 活动奖励 | 活动获得 |

**示例**：

> **击杀Boss**：Boss死亡，触发掉落计算，生成掉落物品。
>
> **开宝箱**：玩家打开宝箱，根据宝箱类型计算掉落。

---

## 2. 掉落生成

**功能说明**：根据掉落表生成物品。

| 功能 | 说明 |
|------|------|
| 物品选择 | 从掉落池选择物品 |
| 数量计算 | 计算掉落数量 |
| 品质确定 | 确定物品品质 |
| 词条生成 | 生成随机词条 |

**示例**：

> **掉落计算**：Boss掉落表包含10种物品，根据权重随机选择3个掉落。

|---------|------|

# 三、掉落表设计

## 1. 掉落表结构

```
LootTable {
  id: "boss_dragon"
  
  // 保证掉落
  guaranteed: [
    { itemId: "gold", min: 1000, max: 5000 },
    { itemId: "exp_orb", count: 1 }
  ]
  
  // 随机掉落
  random: {
    rolls: 3,                // 抽取次数
    items: [
      { itemId: "sword_rare", weight: 10 },
      { itemId: "armor_rare", weight: 10 },
      { itemId: "sword_epic", weight: 3 },
      { itemId: "armor_epic", weight: 3 },
      { itemId: "sword_legendary", weight: 1 }
    ]
  }
  
  // 条件掉落
  conditional: [
    { 
      condition: { firstKill: true },
      items: [{ itemId: "achievement_token", count: 1 }]
    }
  ]
}
```

# 四、概率计算

## 1. 权重概率

**计算公式**：
```
物品概率 = 物品权重 / 总权重

示例：
物品A权重=50，物品B权重=30，物品C权重=20
总权重 = 100
A概率 = 50%，B概率 = 30%，C概率 = 20%
```

---

## 2. 独立概率

**说明**：每个物品独立判定是否掉落。

```
物品A掉落率 = 30%
物品B掉落率 = 20%
物品C掉落率 = 10%

每个物品独立判定，可能同时掉落多个
```

---

## 3. 稀有度提升

| 提升方式 | 说明 |
|---------|------|
| 幸运值 | 玩家属性提高掉率 |
| 难度加成 | 高难度掉率更高 |
| 活动加成 | 活动期间双倍 |
| 组队加成 | 组队掉率提高 |
| 首杀加成 | 首次击杀掉率高 |

**示例**：

> **幸运加成**：基础掉率1%，幸运值+50%，实际掉率1.5%。

# 五、高级机制

## 1. 保底机制

**功能说明**：确保玩家在一定次数内获得稀有物品。

| 保底类型 | 说明 |
|---------|------|
| 硬保底 | 达到次数必出 |
| 软保底 | 逐渐提高概率 |
| 大小保底 | 区分不同保底 |

**示例**：

> **硬保底**：10连抽必出一个紫色，90抽必出一个橙色。
>
> **软保底**：从第75抽开始，每抽橙色概率+6%，90抽时概率接近100%。
>
> **大小保底**：小保底50%几率出UP角色，大保底100%出UP角色。

---

## 2. 概率递增

**功能说明**：随着尝试次数增加概率。

```
基础概率：0.6%
递增规则：从第74次开始，每次+6%

第74次：0.6 + 6 = 6.6%
第75次：6.6 + 6 = 12.6%
...
第89次：84.6 + 6 = 90.6%
第90次：保底100%
```

---

## 3. 掉落保护

| 保护类型 | 说明 |
|---------|------|
| 归属保护 | 只有击杀者可拾取 |
| 时间保护 | 一段时间后公开 |
| 分配保护 | 按规则分配给队员 |
| 防外挂 | 检测异常拾取 |

**示例**：

> **归属时间**：物品掉落后30秒内只有击杀者可拾取，30秒后对所有人开放。

---

## 4. 智能掉落

| 功能 | 说明 |
|------|------|
| 职业掉落 | 优先掉落当前职业装备 |
| 缺装掉落 | 优先掉落缺少的部位 |
| 去重掉落 | 减少重复物品 |
| 个性掉落 | 根据玩家喜好调整 |

**示例**：

> **职业加权**：战士击杀Boss，板甲掉率提高50%。

# 六、各类型游戏应用

## 1. Diablo-like

### 掉落特点
- 大量随机词条
- 高掉率普通装备
- 极低掉率暗金

### 配置
```
词条随机：2-6条
品质分布：白60%/绿25%/蓝12%/紫3%/橙0.5%
暗金掉率：0.01%
```

---

## 2. 手游抽卡

### 掉落特点
- 明确概率公示
- 保底机制完善
- UP机制

### 配置
```
R卡：85%
SR卡：10%
SSR卡：5%（UP角色50%）
保底：10抽保底SR，90抽保底SSR
```

---

## 3. MMORPG

### 掉落特点
- Boss专属掉落
- 需求roll点
- DKP系统

### 配置
```
普通怪：金币+材料
精英怪：有概率蓝装
Boss：紫装+概率橙装
```
