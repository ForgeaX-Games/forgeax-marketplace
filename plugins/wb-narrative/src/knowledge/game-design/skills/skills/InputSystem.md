# 一、系统概述

## 什么是输入系统

输入系统是游戏与玩家交互的桥梁，负责检测、处理和分发来自键盘、鼠标、手柄、触摸屏等设备的输入信号。

## 核心职责

| 职责 | 说明 |
|------|------|
| 设备检测 | 检测并管理输入设备 |
| 输入采集 | 采集原始输入信号 |
| 按键映射 | 将物理按键映射到游戏动作 |
| 输入处理 | 处理连击、长按、缓冲等 |
| 输入分发 | 将输入事件分发给对应系统 |

## 适用游戏类型

| 游戏类型 | 输入复杂度 | 特点 |
|---------|----------|------|
| **FTG** | 极高 | 帧级精度、指令输入 |
| **ACT** | 高 | 连击、缓冲、闪避 |
| **FPS** | 高 | 精准瞄准、快速切换 |
| **RPG** | 中 | 技能快捷键、交互 |
| **休闲** | 低 | 简单点击、滑动 |

# 二、核心功能模块

## 1. 按键检测

**功能说明**：检测按键的各种状态。

| 状态 | 说明 |
|------|------|
| 按下 | 按键被按下的瞬间 |
| 持续 | 按键保持按下状态 |
| 抬起 | 按键被释放的瞬间 |
| 按压时长 | 按键被按住的持续时间 |

**示例**：

> **短按跳跃**：按下跳跃键不到0.2秒松开，角色小跳。
>
> **长按跳跃**：按住跳跃键超过0.2秒，角色大跳。

---

## 2. 按键映射

**功能说明**：将物理按键绑定到游戏动作。

| 功能 | 说明 |
|------|------|
| 动作定义 | 定义游戏动作（跳跃、攻击等） |
| 按键绑定 | 将按键绑定到动作 |
| 多键绑定 | 同一动作绑定多个按键 |
| 重新绑定 | 玩家自定义按键 |
| 预设方案 | 多套按键配置切换 |

**示例**：

> **跳跃动作**：绑定空格键（键盘）、A键（手柄）、屏幕右下按钮（触摸）。

---

## 3. 输入缓冲

**功能说明**：在动作进行中预输入下一动作。

| 功能 | 说明 |
|------|------|
| 缓冲时间 | 预输入的有效时长 |
| 优先级 | 多个预输入的优先级 |
| 缓冲取消 | 取消已缓冲的输入 |
| 缓冲执行 | 当前动作结束后执行 |

**示例**：

> **连招缓冲**：玩家在攻击动作第25帧时输入下一招，系统缓冲该输入，第30帧攻击结束后立即执行。

---

## 4. 输入队列

**功能说明**：按顺序记录和处理输入。

| 功能 | 说明 |
|------|------|
| 队列记录 | 记录输入历史 |
| 指令识别 | 识别特定输入序列 |
| 队列清空 | 清空输入历史 |
| 队列回放 | 回放输入序列 |

**示例**：

> **波动拳指令**：检测到输入队列中300ms内依次出现 ↓↘→+拳，触发波动拳技能。

# 三、输入类型设计

## 1. 键盘输入

### 基础配置
```
KeyboardConfig {
  // 按键检测
  repeatDelay: 500ms       // 首次重复延迟
  repeatRate: 50ms         // 重复触发间隔
  
  // 组合键
  modifiers: ["Shift", "Ctrl", "Alt"]
}
```

### 功能列表

| 功能 | 说明 |
|------|------|
| 单键检测 | 检测单个按键 |
| 组合键 | Ctrl+S、Alt+Tab等 |
| 同时按键 | 多键同时按下 |
| 按键序列 | 连续按键输入 |

---

## 2. 鼠标输入

### 基础配置
```
MouseConfig {
  // 灵敏度
  sensitivity: 1.0
  acceleration: false
  
  // 按键
  doubleClickTime: 300ms
  dragThreshold: 5px
}
```

### 功能列表

| 功能 | 说明 |
|------|------|
| 按键点击 | 左键/右键/中键 |
| 双击检测 | 快速连点两次 |
| 拖拽检测 | 按住并移动 |
| 滚轮滚动 | 滚轮上下滚动 |
| 位置追踪 | 鼠标位置/移动量 |

---

## 3. 手柄输入

### 基础配置
```
GamepadConfig {
  // 摇杆
  deadzone: 0.15           // 死区（忽略微小偏移）
  innerDeadzone: 0.1
  outerDeadzone: 0.9
  
  // 扳机
  triggerThreshold: 0.1    // 触发阈值
  
  // 震动
  vibrationEnabled: true
}
```

### 功能列表

| 功能 | 说明 |
|------|------|
| 按键检测 | A/B/X/Y等按键 |
| 摇杆输入 | 方向和力度 |
| 扳机输入 | 压力感应（0-1） |
| 震动反馈 | 触觉反馈输出 |
| 多手柄 | 支持多个手柄 |

**示例**：

> **摇杆死区**：摇杆偏移小于15%时忽略输入，避免摇杆漂移问题。

---

## 4. 触摸输入

### 基础配置
```
TouchConfig {
  // 点击判定
  tapMaxDuration: 300ms    // 最长点击时长
  tapMaxDistance: 20px     // 最大移动距离
  
  // 长按
  longPressTime: 500ms
  
  // 滑动
  swipeMinDistance: 50px
  swipeMaxTime: 300ms
}
```

### 功能列表

| 功能 | 说明 |
|------|------|
| 点击 | 轻触屏幕 |
| 长按 | 按住超过阈值 |
| 滑动 | 快速滑动方向 |
| 拖拽 | 按住并移动 |
| 多点触控 | 多指同时操作 |
| 手势识别 | 捏合、旋转等 |

**示例**：

> **双指缩放**：检测到两指距离变化，触发缩放操作，距离增大放大，距离减小缩小。

# 五、高级机制

## 1. 指令输入

**功能说明**：检测特定按键序列触发技能。

| 指令类型 | 格式 | 示例 |
|---------|------|------|
| 方向+按键 | 方向序列+攻击 | ↓↘→+拳 |
| 蓄力释放 | 蓄力方向+释放 | ←(蓄2秒)→+拳 |
| 连打 | 快速连按 | 拳×5 |
| 同时按 | 多键同按 | 拳+脚 |

**参数配置**：
```
CommandConfig {
  inputWindow: 300ms       // 指令输入窗口
  keyInterval: 100ms       // 按键间隔容差
  chargeTime: 1500ms       // 蓄力最小时长
  simultaneousWindow: 3帧  // 同时按键容差
}
```

**示例**：

> **超必杀指令**：玩家在500ms内依次输入 ↓↘→↓↘→+重拳，触发超必杀技。

---

## 2. 连击检测

**功能说明**：检测按节奏的连续输入。

| 功能 | 说明 |
|------|------|
| 连击窗口 | 两次输入的有效间隔 |
| 连击计数 | 记录连击次数 |
| 连击派生 | 不同连击触发不同招式 |
| 连击中断 | 超时或受击中断 |

**示例**：

> **轻轻重派生**：玩家连按轻攻击2次后按重攻击，触发"轻轻重"特殊连招。

---

## 3. 输入录制

**功能说明**：记录并回放输入。

| 功能 | 说明 |
|------|------|
| 录制开始 | 开始记录输入 |
| 录制结束 | 停止记录 |
| 回放 | 按时序回放输入 |
| 帧步进 | 逐帧回放分析 |

**示例**：

> **训练模式**：录制假人的防守动作，然后练习破防连招。

---

## 4. 设备切换

**功能说明**：自动检测并切换输入设备。

| 功能 | 说明 |
|------|------|
| 自动检测 | 检测到新输入自动切换 |
| 按键提示切换 | UI按键图标自动更新 |
| 多设备共存 | 同时支持多种输入 |

**示例**：

> **即时切换**：玩家正在用手柄玩，突然按下键盘，UI立即从手柄图标切换为键盘图标。

---

## 5. 输入容错

**功能说明**：提升操作手感的容错机制。

| 机制 | 说明 | 参数 |
|------|------|------|
| 土狼时间 | 离开平台后仍可跳跃 | 80-150ms |
| 跳跃缓冲 | 落地前输入落地后执行 | 100-200ms |
| 同时按容差 | 多键同时按的帧误差 | 2-4帧 |
| 方向简化 | 斜向可识别为单方向 | - |

**示例**：

> **土狼时间**：玩家从平台边缘跑出后100ms内按跳跃仍可正常跳起，避免挫败感。

# 六、各类型游戏应用

## 1. 格斗游戏

### 输入特点
- 帧级精度检测
- 复杂指令识别
- 输入缓冲系统

### 配置
```
输入窗口：300-500ms
缓冲时长：6-12帧
同时按容差：2-4帧
蓄力时长：1.5-2秒
```

---

## 2. 动作游戏

### 输入特点
- 连击系统
- 闪避/弹反时机
- 蓄力攻击

### 配置
```
连击窗口：200-400ms
完美闪避窗口：3-6帧
弹反窗口：5-10帧
蓄力阶段：0.5/1/2秒
```

---

## 3. 平台跳跃

### 输入特点
- 跳跃手感
- 容错机制
- 空中控制

### 配置
```
土狼时间：80-150ms
跳跃缓冲：100-200ms
空中控制力：60%-100%
短/长按跳跃比：50%-100%
```

---

## 4. 射击游戏

### 输入特点
- 精准瞄准
- 快速切换
- 瞄准辅助

### 配置
```
鼠标灵敏度：1-100
摇杆死区：5%-25%
瞄准辅助：无/弱/中/强
扳机死区：0.1
```

|------|------|------|
| isPressed | 按键是否按下 | 按键/动作 |
