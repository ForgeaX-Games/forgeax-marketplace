# 一、系统概述

## 什么是剧情系统

剧情系统管理游戏的叙事内容，包括章节管理、过场动画、剧情分支和回顾功能。

## 核心职责

| 职责 | 说明 |
|------|------|
| 章节管理 | 管理剧情章节 |
| 剧情解锁 | 控制剧情进度 |
| 过场演出 | 播放过场动画 |
| 分支管理 | 管理剧情分支 |
| 剧情回顾 | 重温已看剧情 |

## 适用游戏类型

| 游戏类型 | 剧情重要性 | 特点 |
|---------|----------|------|
| **AVG** | 核心 | 剧情即玩法 |
| **RPG** | 高 | 主线剧情 |
| **开放世界** | 高 | 多线剧情 |
| **剧情向** | 核心 | 叙事为主 |

# 三、剧情设计

## 1. 章节结构

```
StoryChapter {
  id: "chapter_1"
  name: "第一章：觉醒"
  
  // 解锁条件
  unlockCondition: {
    type: "quest_complete",
    questId: "prologue"
  }
  
  // 剧情节点
  nodes: [
    { type: "cutscene", id: "awakening" },
    { type: "dialogue", id: "meet_mentor" },
    { type: "choice", id: "first_choice" }
  ]
  
  // 分支
  branches: {
    "choice_a": "chapter_2a",
    "choice_b": "chapter_2b"
  }
}
```

# 六、功能速查表

## 剧情系统方法

| 方法 | 说明 | 参数 |
|------|------|------|
| getChapters | 获取章节列表 | - |
| startChapter | 开始章节 | 章节ID |
| getCurrentChapter | 获取当前章节 | - |
| playCutscene | 播放过场 | 过场ID |
| skipCutscene | 跳过过场 | - |
| makeChoice | 做出选择 | 选项ID |
| getUnlockedEndings | 获取已解锁结局 | - |
| replayStory | 回顾剧情 | 剧情ID |

## 剧情事件

| 事件 | 数据 | 说明 |
|------|------|------|
| story:chapter_started | {chapterId} | 章节开始 |
| story:chapter_completed | {chapterId} | 章节完成 |
| story:cutscene_started | {cutsceneId} | 过场开始 |
| story:cutscene_ended | {cutsceneId} | 过场结束 |
| story:choice_made | {choiceId, option} | 做出选择 |
| story:ending_reached | {endingId} | 达成结局 |
