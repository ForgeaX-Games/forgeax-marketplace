# 一、系统概述

## 什么是场景系统

场景系统负责管理游戏中的不同"场景"或"画面"，如主菜单、游戏关卡、暂停界面、设置页面等。它提供统一的生命周期管理、过渡动画和资源加载机制。

## 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                    场景管理器 (SceneManager)                  │
│                                                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ 场景注册表  │  │  场景栈     │  │ 过渡控制器  │         │
│  │ (Registry)  │  │  (Stack)    │  │ (Transition)│         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
├─────────────────────────────────────────────────────────────┤
│                    当前活动场景 (Active Scene)               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  init() → preload() → create() → update() → destroy()│   │
│  └─────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│                    场景栈 (Scene Stack)                      │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                     │
│  │ Pause   │  │ Dialog  │  │  HUD    │  ← 可叠加场景       │
│  └─────────┘  └─────────┘  └─────────┘                     │
└─────────────────────────────────────────────────────────────┘
```

## 适用范围

| 游戏类型 | 核心需求 |
|---------|---------|
| **所有类型** | 基础场景切换 |
| **RPG/冒险** | 场景栈（暂停、对话叠加）、过渡动画 |
| **大型游戏** | 异步加载、预加载、加载进度显示 |
| **开放世界** | 无缝加载、流式加载 |
| **休闲/三消** | 简单场景切换、快速重试 |

# 二、核心功能模块

## 1. 场景注册

**功能说明**：将游戏中的所有场景注册到场景管理器，便于统一管理和调用。

| 功能 | 说明 |
|------|------|
| 场景注册 | 为每个场景分配唯一标识（key） |
| 场景配置 | 设置场景名称、是否叠加、是否暂停下层等属性 |
| 批量注册 | 一次性注册多个场景 |
| 动态注册 | 运行时动态添加新场景 |

**示例**：游戏启动时，注册"主菜单"、"游戏关卡"、"暂停菜单"、"设置"等场景。每个场景有唯一的 key，如 `mainMenu`、`game`、`pause`、`settings`。

---

## 2. 场景切换

**功能说明**：从当前场景切换到目标场景，销毁旧场景并创建新场景。

| 功能 | 说明 |
|------|------|
| 切换场景 | 完全替换当前场景 |
| 重启场景 | 重新加载当前场景 |
| 过渡动画 | 切换时播放过渡效果 |
| 数据传递 | 向目标场景传递参数 |

**示例**：

> **主菜单→游戏**：玩家点击"开始游戏"，场景从主菜单切换到游戏关卡。主菜单场景被销毁，游戏场景被创建。过渡动画为1秒淡入淡出。
>
> **关卡失败重试**：玩家在游戏结束界面点击"重试"，当前游戏场景重新加载，所有状态重置，但关卡配置保持不变。
>
> **关卡完成→下一关**：玩家通关后，切换到下一关卡场景，传递当前得分和累计道具等数据。

---

## 3. 场景栈管理

**功能说明**：支持场景叠加，新场景覆盖在当前场景之上，而非替换。适用于暂停菜单、对话框等。

| 功能 | 说明 |
|------|------|
| 推入场景 (Push) | 将新场景推入栈顶，覆盖当前场景 |
| 弹出场景 (Pop) | 移除栈顶场景，恢复下层场景 |
| 弹出到指定场景 | 连续弹出直到指定场景 |
| 弹出到底部 | 只保留栈底场景 |
| 下层暂停 | 新场景推入时暂停下层场景的更新 |
| 透明叠加 | 新场景可透明显示下层内容 |

**示例**：

> **暂停菜单**：玩家在游戏中按 ESC，暂停菜单场景被推入栈顶。游戏场景暂停更新但仍可见（半透明遮罩）。玩家点击"继续"，暂停菜单弹出，游戏恢复。
>
> **设置嵌套**：暂停菜单中点击"设置"，设置场景被推入栈顶（此时栈中有：游戏→暂停→设置）。点击返回弹出设置，回到暂停菜单。
>
> **直接返回游戏**：在设置界面点击"返回游戏"，连续弹出设置和暂停菜单，直接回到游戏场景。

---

## 4. 场景状态

**功能说明**：追踪每个场景的当前状态。

| 状态 | 说明 |
|------|------|
| 未加载 (Unloaded) | 场景尚未创建 |
| 加载中 (Loading) | 场景正在加载资源 |
| 就绪 (Ready) | 资源加载完成，等待激活 |
| 活动 (Active) | 场景正在运行，接收更新 |
| 暂停 (Paused) | 场景被覆盖，暂停更新 |
| 销毁中 (Destroying) | 场景正在清理资源 |

**示例**：游戏场景正在运行（Active）→ 玩家按暂停 → 游戏场景变为 Paused → 暂停菜单变为 Active → 玩家点击继续 → 暂停菜单销毁 → 游戏场景恢复为 Active。

# 三、场景生命周期

## 生命周期阶段

```
┌────────────────────────────────────────────────────────────┐
│                       场景生命周期                          │
└────────────────────────────────────────────────────────────┘

  注册                    激活                    销毁
    │                      │                      │
    ▼                      ▼                      ▼
┌────────┐            ┌────────┐            ┌────────┐
│UNLOADED│───start───▶│LOADING │            │DESTROY │
└────────┘            └────┬───┘            └────────┘
                          │                      ▲
                          ▼                      │
                      ┌────────┐                 │
                      │PRELOAD │ (异步资源加载)   │
                      └────┬───┘                 │
                          │                      │
                          ▼                      │
                      ┌────────┐                 │
                      │CREATE  │                 │
                      └────┬───┘                 │
                          │                      │
                          ▼                      │
                      ┌────────┐     destroy     │
              ┌──────▶│ACTIVE  │─────────────────┤
              │       └────┬───┘                 │
              │           │                      │
              │     pause │ resume               │
              │           ▼                      │
              │       ┌────────┐                 │
              └───────│PAUSED  │─────────────────┘
                      └────────┘
```

## 生命周期钩子详解

### 1. init(data)

**调用时机**：场景开始加载时，在预加载之前

**用途**：
- 接收从上一个场景传递的数据
- 初始化场景级变量
- 准备加载配置

**示例**：游戏场景接收关卡编号 `{ level: 3 }`，保存到场景变量中，后续 preload 根据关卡编号加载对应资源。

### 2. preload()

**调用时机**：init 之后，create 之前

**用途**：
- 异步加载场景所需资源（图片、音频、数据文件）
- 显示加载进度
- 可被取消（用户返回时）

**示例**：游戏场景预加载关卡地图、敌人精灵图、背景音乐。加载界面显示"加载中... 45%"。

### 3. create()

**调用时机**：preload 完成后

**用途**：
- 创建游戏对象（精灵、UI、物理体）
- 初始化游戏逻辑
- 开始播放背景音乐
- 注册事件监听

**示例**：创建玩家角色、生成敌人、布置地形、显示 HUD、播放关卡BGM。

### 4. update(deltaTime)

**调用时机**：每帧调用（场景处于 Active 状态时）

**用途**：
- 更新游戏逻辑
- 处理输入响应
- 更新动画
- 检测碰撞

**示例**：每帧更新玩家位置、检测玩家与敌人碰撞、更新 UI 数值显示。

### 5. pause()

**调用时机**：场景被新场景覆盖时

**用途**：
- 暂停游戏逻辑
- 暂停音乐和动画
- 暂停计时器

**示例**：暂停菜单打开时，游戏场景的 pause() 被调用，背景音乐暂停，敌人停止移动。

### 6. resume()

**调用时机**：覆盖场景被移除后

**用途**：
- 恢复游戏逻辑
- 恢复音乐和动画
- 恢复计时器

**示例**：暂停菜单关闭后，游戏场景的 resume() 被调用，背景音乐继续播放，游戏恢复运行。

### 7. destroy()

**调用时机**：场景被移除时

**用途**：
- 清理游戏对象
- 释放资源
- 取消事件订阅
- 停止音乐

**示例**：玩家返回主菜单时，游戏场景的 destroy() 被调用，停止背景音乐，移除所有游戏对象，释放加载的纹理。

# 四、场景过渡效果

## 1. 内置过渡类型

| 类型 | 效果 | 适用场景 |
|------|------|---------|
| 无过渡 (None) | 直接切换 | 即时切换、调试 |
| 淡入淡出 (Fade) | 黑屏过渡 | 通用，最常见 |
| 左滑 (SlideLeft) | 新场景从右侧滑入 | 前进、进入下一页 |
| 右滑 (SlideRight) | 新场景从左侧滑入 | 返回、回到上一页 |
| 上滑 (SlideUp) | 新场景从下方滑入 | 弹出菜单 |
| 下滑 (SlideDown) | 新场景从上方滑入 | 下拉菜单 |
| 放大 (ZoomIn) | 从中心放大 | 进入详情 |
| 缩小 (ZoomOut) | 向中心缩小 | 退出详情 |

**示例**：

> **主菜单→游戏**：使用淡入淡出过渡，1秒黑屏切换，营造"进入游戏世界"的感觉。
>
> **关卡选择→关卡详情**：使用放大过渡，从点击的关卡图标位置放大，展示关卡详情。
>
> **暂停菜单弹出**：使用上滑过渡，暂停菜单从屏幕底部滑入，0.3秒完成。

---

## 2. 过渡参数配置

| 参数 | 说明 | 典型值 |
|------|------|-------|
| 过渡时长 | 动画持续时间 | 300-1000ms |
| 缓动函数 | 动画速度曲线 | easeInOut、easeOut |
| 遮罩颜色 | 淡入淡出的颜色 | 黑色、白色 |
| 回调函数 | 过渡开始/结束时执行 | 播放音效等 |

**示例**：

> **场景淡入淡出**：时长800ms，颜色黑色，缓动函数 easeInOut。前400ms屏幕逐渐变黑，后400ms从黑色恢复。
>
> **快速切换**：时长200ms，无缓动，直接线性过渡，适合需要快速响应的场景。

---

## 3. 自定义过渡效果

| 效果 | 说明 | 适用场景 |
|------|------|---------|
| 圆形遮罩 | 从中心或指定点扩散/收缩 | 复古游戏、马里奥风格 |
| 像素化 | 画面逐渐像素化后消失 | 复古、像素游戏 |
| 百叶窗 | 横向或纵向百叶窗效果 | 日式RPG |
| 旋涡 | 画面旋转消失 | 魔法、奇幻风格 |
| 碎片化 | 画面碎裂成碎片 | 战斗、紧张氛围 |
| 翻页 | 像翻书一样翻页 | 故事书、绘本风格 |

**示例**：

> **马里奥风格进入水管**：圆形遮罩从玩家位置开始收缩，完全变黑后切换场景，再从中心扩散显示新场景。
>
> **像素化过渡**：旧场景逐渐像素化（像素块变大），完全模糊后切换，新场景从像素化状态逐渐清晰。

# 五、预加载系统

## 1. 基础预加载

**功能说明**：在切换到目标场景前，提前加载该场景的资源。

| 功能 | 说明 |
|------|------|
| 资源预加载 | 提前加载图片、音频、数据 |
| 加载进度 | 获取当前加载进度（0-100%） |
| 加载完成回调 | 加载完成后执行回调 |
| 加载失败处理 | 资源加载失败时的处理 |

**示例**：

> **关卡加载界面**：玩家选择关卡后，显示加载界面。进度条从0%增长到100%，同时显示加载提示文字。加载完成后自动进入游戏。
>
> **加载失败重试**：某个资源加载失败，弹出提示"加载失败，是否重试？"玩家点击重试后重新加载该资源。

---

## 2. 后台预加载

**功能说明**：在当前场景运行时，后台静默加载可能会用到的场景资源。

| 功能 | 说明 |
|------|------|
| 静默加载 | 不显示加载界面，后台加载 |
| 优先级控制 | 高优先级资源先加载 |
| 带宽限制 | 避免影响当前场景性能 |
| 加载状态查询 | 检查某场景是否已预加载完成 |

**示例**：

> **主菜单预加载第一关**：玩家在主菜单时，后台预加载第一关的资源。当玩家点击"开始游戏"时，如果预加载已完成，可以立即进入游戏无需等待。
>
> **关卡中预加载下一关**：玩家在第3关进行到一半时，后台开始预加载第4关。通关后可以无缝进入下一关。
>
> **对话中预加载战斗场景**：RPG中剧情对话即将结束时，预加载接下来的战斗场景资源。

---

## 3. 资源卸载

**功能说明**：卸载不再需要的场景资源，释放内存。

| 功能 | 说明 |
|------|------|
| 场景卸载 | 卸载指定场景的所有资源 |
| 自动卸载 | 场景销毁时自动卸载资源 |
| 共享资源保留 | 多场景共用的资源不卸载 |
| 内存警告响应 | 系统内存不足时主动卸载 |

**示例**：

> **关卡切换卸载**：从第3关切换到第4关时，卸载第3关特有的资源（如第3关专属的敌人纹理），但保留通用资源（如玩家角色纹理）。
>
> **内存不足处理**：游戏收到系统内存警告，主动卸载已预加载但未使用的场景资源。

# 六、场景间数据传递

## 1. 直接传参

**功能说明**：切换场景时直接传递数据对象。

**示例**：

> **关卡选择→游戏**：传递 `{ level: 5, difficulty: 'hard' }`，游戏场景根据参数加载第5关困难模式。
>
> **战斗结果→结算**：传递 `{ victory: true, score: 15000, time: 120, stars: 3 }`，结算场景显示胜利画面和得分。

---

## 2. 全局状态管理

**功能说明**：使用全局状态对象存储跨场景数据。

| 功能 | 说明 |
|------|------|
| 状态读取 | 任何场景都可读取全局状态 |
| 状态修改 | 修改后所有场景可见 |
| 状态持久化 | 可保存到本地存储 |
| 状态重置 | 新游戏时重置状态 |

**示例**：

> **玩家数据**：全局状态保存玩家等级、金币、背包物品。任何场景都可读取和修改这些数据。
>
> **游戏进度**：全局状态保存已解锁关卡、已获得成就。关卡选择场景读取这些数据显示解锁状态。

---

## 3. 事件系统

**功能说明**：通过事件总线在场景间通信。

**示例**：

> **成就解锁通知**：游戏场景触发成就解锁事件，成就系统接收事件并显示解锁动画，同时全局状态更新成就列表。
>
> **跨场景音乐控制**：设置场景发送"音量调整"事件，音频管理器接收事件并调整所有场景的音量。

---

# 七、各类型游戏应用

## 1. RPG / JRPG

### 典型场景结构
```
主菜单
├── 新游戏 → 开场动画 → 游戏世界
├── 继续游戏 → 存档选择 → 游戏世界
└── 设置

游戏世界
├── 大地图
├── 城镇
├── 野外
├── 迷宫
└── 战斗场景

叠加场景
├── 暂停菜单
├── 背包
├── 状态
├── 任务
├── 地图
├── 对话
└── 商店
```

### 特殊需求
| 功能 | 说明 |
|------|------|
| 场景栈深度 | 支持多层叠加（游戏→暂停→设置→按键配置） |
| 战斗过渡 | 独特的战斗开始/结束过渡效果 |
| 场景记忆 | 离开城镇后回来，NPC位置和状态保持 |
| 存档点恢复 | 读档后恢复到正确的场景和位置 |

**示例**：

> **遇敌过渡**：玩家在野外遇到敌人，屏幕出现"碎裂"效果，切换到战斗场景。战斗结束后，"拼合"效果返回野外场景，玩家位置不变。
>
> **城镇→商店**：进入商店时推入商店场景，城镇场景暂停但保留状态。离开商店后弹出，城镇恢复。

---

## 2. 动作游戏 / 平台游戏

### 典型场景结构
```
主菜单
├── 开始游戏
├── 关卡选择 → 关卡1/2/3...
└── 设置

游戏中
├── 关卡场景（可能分多个区域）
├── Boss战场景
└── 过场动画

叠加场景
├── 暂停菜单
└── 死亡/重试界面
```

### 特殊需求
| 功能 | 说明 |
|------|------|
| 快速重试 | 死亡后几乎无加载时间重试 |
| 检查点恢复 | 从最近检查点恢复而非重新加载 |
| 区域切换 | 关卡内不同区域间的无缝或快速切换 |
| 速通计时 | 场景切换时暂停/恢复计时器 |

**示例**：

> **快速重试**：玩家死亡后，不切换场景，而是将场景状态重置到检查点状态，实现秒速重试。
>
> **Boss战过渡**：进入Boss房间前，短暂黑屏，镜头推近Boss，显示Boss名字，然后战斗开始。


### 特殊需求

| 弹窗叠加 | 结算、复活确认等弹窗覆盖游戏场景 |

**示例**：

> **连续闯关**：玩家通关后，结算弹窗显示"下一关"按钮，点击后快速切换到下一关（已预加载）。

# 八、功能速查表

## 场景管理器方法

| 方法 | 说明 | 参数 |
|------|------|------|
| register | 注册场景类 | 场景类, 配置 |
| start | 切换到指定场景（替换） | 场景key, 数据, 过渡配置 |
| restart | 重启当前场景 | 数据, 过渡配置 |
| push | 推入叠加场景 | 场景key, 数据, 过渡配置 |
| pop | 弹出当前叠加场景 | 过渡配置 |
| popTo | 弹出到指定场景 | 场景key, 过渡配置 |
| popToRoot | 弹出所有叠加场景 | 过渡配置 |
| update | 每帧更新 | 时间增量 |
| renderTransition | 渲染过渡效果 | 渲染上下文 |
| getCurrentScene | 获取当前场景 | - |
| getSceneStack | 获取场景栈 | - |
| isSceneActive | 检查场景是否激活 | 场景key |
| isInTransition | 是否正在过渡 | - |
| clear | 清理所有场景 | - |

## 场景配置选项

| 选项 | 说明 | 默认值 |
|------|------|-------|
| key | 场景唯一标识 | 必填 |
| name | 场景显示名称 | 可选 |
| overlay | 是否为叠加场景 | false |
| pauseBelow | 是否暂停下层场景 | false |
| transparent | 是否透明显示下层 | false |
| persistent | 是否持久化不销毁 | false |

## 过渡配置选项

| 选项 | 说明 | 默认值 |
|------|------|-------|
| type | 过渡类型 | FADE |
| duration | 过渡时长（毫秒） | 500 |
| easing | 缓动函数 | easeInOut |
| color | 遮罩颜色 | #000000 |
| onStart | 过渡开始回调 | 无 |
| onComplete | 过渡完成回调 | 无 |

## 事件列表

| 事件 | 数据 | 说明 |
|------|------|------|
| scene:loading | {key, scene} | 场景开始加载 |
| scene:started | {key, scene} | 场景启动完成 |
| scene:pushed | {key, scene} | 场景推入栈 |
| scene:popped | {key} | 场景弹出栈 |
| scene:paused | {key} | 场景暂停 |
| scene:resumed | {key} | 场景恢复 |
