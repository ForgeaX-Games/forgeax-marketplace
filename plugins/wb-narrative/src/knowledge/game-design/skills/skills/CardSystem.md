# 一、系统概述

## 什么是卡牌系统

卡牌系统管理游戏中的卡牌收集、卡组构建和卡牌战斗，是卡牌游戏的核心系统。

## 核心职责

| 职责 | 说明 |
|------|------|
| 卡牌管理 | 管理卡牌收集 |
| 卡组构建 | 构建和保存卡组 |
| 卡牌战斗 | 卡牌对战规则 |
| 抽卡系统 | 获取新卡牌 |
| 卡牌升级 | 卡牌强化升级 |

## 适用游戏类型

| 游戏类型 | 卡牌重要性 | 特点 |
|---------|----------|------|
| **TCG/CCG** | 核心 | 纯卡牌对战 |
| **卡牌RPG** | 核心 | 卡牌+角色 |
| **Roguelike** | 高 | 局内构筑 |
| **自走棋** | 中 | 抽卡放置 |
| **手游** | 中 | 收集养成 |

# 三、卡牌设计

## 1. 卡牌属性

```
CardConfig {
  id: "fire_warrior"
  name: "烈焰战士"
  
  // 基础属性
  rarity: "rare"           // 稀有度
  cost: 3                  // 费用
  attack: 4                // 攻击力
  health: 5                // 生命值
  
  // 分类
  type: "minion"           // 类型
  race: "elemental"        // 种族
  class: "warrior"         // 职业
  
  // 效果
  abilities: [
    { type: "battlecry", effect: "deal_damage", value: 2 }
  ]
  
  // 描述
  description: "战吼：造成2点伤害"
  flavor: "他的怒火能融化一切"
}
```

# 五、战斗机制

## 1. 战斗流程

```
游戏开始
    │
    ▼
双方抽初始手牌
    │
    ▼
先手/后手确定
    │
    ▼
┌─▶ 回合开始
│       │
│       ▼
│   抽一张牌
│       │
│       ▼
│   获得费用
│       │
│       ▼
│   玩家操作（出牌/攻击）
│       │
│       ▼
│   回合结束
│       │
└───┴──判断胜负
        │
        ▼
    游戏结束
```

---

## 2. 费用系统

| 机制 | 说明 |
|------|------|
| 初始费用 | 1/0/1 等 |
| 费用增长 | 每回合+1 |
| 费用上限 | 最高10 |
| 费用恢复 | 回合开始恢复满 |
| 临时费用 | 当回合有效 |
