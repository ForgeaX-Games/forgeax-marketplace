# 一、系统概述

## 什么是制作系统

制作系统允许玩家使用材料制作新物品，是生存、建造类游戏的核心系统。

## 核心职责

| 职责 | 说明 |
|------|------|
| 配方管理 | 管理可制作的配方 |
| 材料检查 | 检查材料是否足够 |
| 制作执行 | 执行制作流程 |
| 产出生成 | 生成制作产物 |

## 适用游戏类型

| 游戏类型 | 制作复杂度 | 特点 |
|---------|----------|------|
| **生存** | 极高 | 核心玩法 |
| **建造** | 高 | 建筑制作 |
| **MMORPG** | 中高 | 生活技能 |
| **模拟经营** | 中 | 生产系统 |
| **RPG** | 低中 | 辅助功能 |

# 二、核心功能模块

## 1. 配方学习

**功能说明**：玩家获取和学习制作配方。

| 获取方式 | 说明 |
|---------|------|
| 默认配方 | 角色自带 |
| 等级解锁 | 达到等级解锁 |
| 配方书 | 使用道具学习 |
| 任务奖励 | 完成任务获得 |
| 发现解锁 | 实验发现新配方 |
| 购买配方 | 商店购买 |

**示例**：

> **配方书**：使用"铁剑配方"，学习如何制作铁剑。
>
> **发现系统**：将铁锭和木棍放入制作台，发现可以制作铁剑。

---

## 2. 材料检查

**功能说明**：检查制作所需材料。

| 功能 | 说明 |
|------|------|
| 材料列表 | 显示所需材料 |
| 持有数量 | 显示拥有的数量 |
| 缺少提示 | 高亮缺少的材料 |
| 材料来源 | 提示获取途径 |

**示例**：

> **铁剑配方**：需要铁锭×5、木棍×1。玩家有铁锭3个，显示"还需铁锭×2"。

---

## 3. 制作执行

**功能说明**：执行制作流程。

| 功能 | 说明 |
|------|------|
| 单次制作 | 制作一个 |
| 批量制作 | 制作多个 |
| 最大制作 | 制作材料允许的最大数量 |
| 制作进度 | 显示制作进度条 |
| 制作取消 | 取消当前制作 |

**示例**：

> **批量制作**：选择制作药水×50，系统检查材料足够，开始批量制作。
>
> **制作时间**：每个药水需要2秒，50个共需100秒，显示进度条。

---

## 4. 产物获取

**功能说明**：获取制作产物。

| 功能 | 说明 |
|------|------|
| 基础产出 | 获得配方产物 |
| 额外产出 | 概率获得额外物品 |
| 暴击产出 | 获得双倍产物 |
| 品质随机 | 产物品质随机 |

**示例**：

> **制作完成**：消耗材料，获得铁剑×1。
>
> **暴击制作**：触发暴击，获得铁剑×2。

# 三、配方设计

## 1. 配方结构

```
RecipeConfig {
  id: "iron_sword"
  name: "铁剑"
  category: "weapon"
  
  // 产物
  output: {
    itemId: "iron_sword",
    count: 1,
    minQuality: 1,
    maxQuality: 5
  }
  
  // 材料
  materials: [
    { itemId: "iron_ingot", count: 5 },
    { itemId: "wood_stick", count: 1 }
  ]
  
  // 制作条件
  requirements: {
    level: 10,
    skill: { blacksmith: 5 },
    station: "forge"
  }
  
  // 制作属性
  craftTime: 5           // 秒
  experienceGain: 50     // 制作经验
}
```

---

## 2. 配方分类

| 分类 | 说明 | 示例 |
|------|------|------|
| 武器 | 各类武器 | 剑、弓、杖 |
| 防具 | 各类防具 | 头盔、胸甲 |
| 消耗品 | 使用后消失 | 药水、食物 |
| 材料 | 中间材料 | 锭、布料 |
| 建筑 | 建筑物 | 墙、门、家具 |
| 工具 | 采集工具 | 镐、斧、锄 |

---

## 3. 材料类型

| 类型 | 说明 |
|------|------|
| 精确材料 | 必须是指定物品 |
| 类型材料 | 同类型任意物品 |
| 标签材料 | 带有特定标签 |
| 可选材料 | 可以替换的材料 |

**示例**：
> **木棍**：任意"木材"类型物品×2可制作。
>
> **锻造**：需要"燃料"标签物品×1，可以是煤炭或木炭。

# 四、制作流程

## 1. 基础制作流程

```
选择配方
    │
    ▼
检查配方是否已学习
    │已学习
    ▼
检查材料是否足够 ──不足──▶ 提示缺少材料
    │足够
    ▼
检查制作条件 ──不满足──▶ 提示条件不满足
    │满足
    ▼
开始制作（扣除材料）
    │
    ▼
等待制作时间
    │
    ▼
制作完成，获得产物
    │
    ▼
获得制作经验
```

---

## 2. 制作时间

| 模式 | 说明 |
|------|------|
| 即时制作 | 立即完成 |
| 计时制作 | 需要等待 |
| 离线制作 | 离线时继续 |
| 加速制作 | 付费加速 |

**示例**：

> **计时制作**：制作高级装备需要30分钟，可以挂机或做其他事。
>
> **付费加速**：使用加速道具，立即完成制作。

---

## 3. 制作队列

**功能说明**：排队制作多个物品。

| 功能 | 说明 |
|------|------|
| 队列长度 | 可排队的任务数 |
| 队列管理 | 调整顺序、取消 |
| 自动继续 | 完成后自动下一个 |
| 队列扩展 | 付费增加队列 |

**示例**：

> **制作队列**：排队制作铁剑×5、铁盔×2、铁甲×1，按顺序自动完成。

# 五、高级功能

## 1. 制作专精

**功能说明**：制作技能等级系统。

| 功能 | 说明 |
|------|------|
| 专精等级 | 制作技能等级 |
| 经验获取 | 制作获得经验 |
| 等级效果 | 解锁配方、提高成功率 |
| 专精分支 | 不同制作方向 |

**示例**：

> **锻造专精**：锻造等级提升，可以制作更高级武器，暴击率提高。

---

## 2. 品质系统

**功能说明**：制作产物有不同品质。

| 品质 | 说明 | 概率 |
|------|------|------|
| 普通 | 基础属性 | 60% |
| 优秀 | 属性+10% | 25% |
| 精良 | 属性+25% | 12% |
| 完美 | 属性+50% | 3% |

**示例**：

> **品质影响**：高等级锻造师有更高概率制作出高品质装备。

---

## 3. 制作站

**功能说明**：需要在特定设备处制作。

| 制作站 | 制作内容 |
|--------|---------|
| 工作台 | 基础物品 |
| 熔炉 | 冶炼金属 |
| 铁砧 | 武器防具 |
| 炼金台 | 药水 |
| 织布机 | 布料 |
| 厨房 | 食物 |

**示例**：

> **冶炼铁锭**：需要在熔炉旁边才能将铁矿石冶炼成铁锭。

---

## 4. 自动制作

| 功能 | 说明 |
|------|------|
| 重复制作 | 自动重复制作 |
| 链式制作 | 自动制作中间材料 |
| 智能制作 | 智能选择材料 |

**示例**：

> **链式制作**：制作铁剑需要铁锭，如果有铁矿石，自动先冶炼铁锭。

# 六、各类型游戏应用

## 1. 生存游戏

### 制作特点
- 核心玩法
- 大量配方
- 制作站系统

### 配置
```
配方数量：100+
制作站：10+种
材料：50+种
特殊：耐久度、修复
```

---

## 2. MMORPG

### 制作特点
- 生活技能
- 交易经济
- 专精系统

### 配置
```
专精分支：锻造/炼金/烹饪/裁缝
等级上限：100级
大师制作：高品质、特殊效果
```

---

## 3. 模拟经营

### 制作特点
- 生产链
- 效率优化
- 自动化

### 配置
```
生产线：多阶段制作
效率：升级提高效率
自动化：自动生产系统
```
