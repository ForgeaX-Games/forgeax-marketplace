# 一、系统概述

## 什么是战斗系统

战斗系统是游戏中处理攻击、伤害、防御等战斗相关逻辑的核心系统。它负责伤害计算、命中判定、暴击处理、特殊效果触发等所有战斗相关的数值和流程。

## 核心职责

| 职责 | 说明 |
|------|------|
| 伤害计算 | 根据攻防属性计算最终伤害 |
| 命中判定 | 判断攻击是否命中 |
| 暴击判定 | 判断是否触发暴击 |
| 伤害类型 | 区分物理/魔法/真实伤害 |
| 伤害修正 | 应用各种加成和减免 |
| 效果触发 | 触发击杀、吸血等效果 |

## 适用游戏类型

| 游戏类型 | 战斗复杂度 | 核心机制 |
|---------|----------|---------|
| **RPG** | 高 | 完整公式、多种判定 |
| **ARPG** | 中高 | 实时战斗、连击系统 |
| **ACT** | 中 | 无敌帧、连击、处决 |
| **MOBA** | 中高 | 技能伤害、穿透计算 |
| **FTG** | 高 | 帧判定、连段系统 |
| **塔防** | 低中 | 简单伤害计算 |
| **卡牌** | 低 | 回合制、固定伤害 |

# 二、核心功能模块

## 1. 攻击流程

**功能说明**：从发起攻击到造成伤害的完整流程。

```
攻击发起
    │
    ▼
命中判定 ──失败──▶ Miss处理
    │成功
    ▼
闪避判定 ──成功──▶ 闪避处理
    │失败
    ▼
格挡判定 ──成功──▶ 格挡减伤
    │失败
    ▼
暴击判定 ──成功──▶ 暴击加成
    │
    ▼
伤害计算
    │
    ▼
护盾吸收
    │
    ▼
造成伤害
    │
    ▼
后续效果（吸血、击杀等）
```

**示例**：

> **完整攻击流程**：战士A攻击法师B。命中率95%，判定命中；法师闪避率10%，判定未闪避；战士暴击率30%，判定暴击；计算暴击伤害后，法师护盾先吸收一部分，剩余伤害扣除生命值。

---

## 2. 伤害类型

**功能说明**：不同类型的伤害有不同的减免机制。

| 伤害类型 | 减免方式 | 说明 |
|---------|---------|------|
| 物理伤害 | 护甲减免 | 受物理防御影响 |
| 魔法伤害 | 魔抗减免 | 受魔法抗性影响 |
| 真实伤害 | 无减免 | 直接扣除生命值 |
| 纯粹伤害 | 无视一切 | 无视护盾和无敌 |

**示例**：

> **物理攻击**：战士普攻造成100物理伤害，目标护甲减免30%，实际造成70伤害。
>
> **魔法技能**：法师火球造成200魔法伤害，目标魔抗减免20%，实际造成160伤害。
>
> **真实伤害**：刺客处决技能造成300真实伤害，无视目标护甲魔抗，直接造成300伤害。

---

## 3. 命中与闪避

**功能说明**：判断攻击是否命中目标。

| 功能 | 说明 |
|------|------|
| 命中率 | 攻击者的命中概率 |
| 闪避率 | 防御者的闪避概率 |
| 最终命中 | 命中率 - 闪避率 |
| 必中攻击 | 某些技能无视闪避 |
| 必闪状态 | 某些状态必定闪避 |

**计算公式**：
```
实际命中率 = 基础命中率 × (1 - 闪避率 + 命中加成)
最低命中率 = 5%（保底）
最高闪避率 = 95%（上限）
```

**示例**：

> **普通攻击**：攻击者命中率100%，目标闪避率20%，实际命中率80%。
>
> **技能必中**：某技能标记为"必定命中"，无视目标闪避率。
>
> **高闪避目标**：刺客闪避率60%，但闪避上限为95%，实际闪避率60%。

---

## 4. 暴击系统

**功能说明**：判定是否触发暴击及暴击伤害计算。

| 功能 | 说明 |
|------|------|
| 暴击率 | 触发暴击的概率 |
| 暴击伤害 | 暴击时的伤害倍率 |
| 必暴机制 | 某些技能必定暴击 |
| 暴击抵抗 | 降低被暴击概率 |
| 暴击减免 | 降低受到的暴击伤害 |

**计算公式**：
```
暴击伤害 = 基础伤害 × 暴击倍率
期望伤害 = 基础伤害 × (1 + 暴击率 × (暴击倍率 - 1))
```

**示例**：

> **普通暴击**：基础伤害100，暴击率30%，暴击伤害200%。触发暴击时造成200伤害。
>
> **高暴击流派**：暴击率80%，暴击伤害350%。期望伤害 = 100 × (1 + 0.8 × 2.5) = 300。
>
> **暴击抵抗**：目标有20%暴击抵抗，攻击者30%暴击率，实际暴击率10%。

---

## 5. 格挡与护盾

### 格挡
| 功能 | 说明 |
|------|------|
| 格挡率 | 触发格挡的概率 |
| 格挡减伤 | 格挡成功时减少的伤害比例 |
| 完美格挡 | 时机精准时完全抵挡 |

**示例**：
> 目标格挡率40%，格挡减伤50%。攻击100伤害，触发格挡后实际受到50伤害。

### 护盾
| 功能 | 说明 |
|------|------|
| 护盾值 | 可吸收的伤害量 |
| 护盾类型 | 物理/魔法/全能护盾 |
| 护盾优先级 | 多个护盾的消耗顺序 |
| 护盾持续时间 | 临时护盾的有效期 |
| 护盾溢出 | 伤害超过护盾的处理 |

**示例**：
> 目标有200点护盾，受到300物理伤害。护盾吸收200，剩余100伤害扣除生命值。

# 三、伤害计算系统

## 1. 基础伤害公式

### 线性公式
```
伤害 = 攻击力 × 技能倍率 - 防御力 × 减伤系数
```
特点：简单直观，防御效果明显

### 除法公式
```
伤害 = 攻击力 × 技能倍率 × (1 - 防御 / (防御 + 常数))
```
特点：防御收益递减，无法完全免疫

### 乘法公式
```
伤害 = 攻击力 × 技能倍率 × (1 - 防御减伤率)
```
特点：百分比减伤，适合MOBA

---

## 2. 完整伤害计算流程

```
1. 基础伤害
   = 攻击力 × 技能倍率

2. 伤害加成
   × (1 + 伤害加成%)
   × (1 + 对特定目标加成%)

3. 暴击加成
   × 暴击倍率（如果暴击）

4. 穿透计算
   目标有效防御 = 防御 × (1 - 百分比穿透) - 固定穿透

5. 防御减伤
   × (1 - 有效防御 / (有效防御 + 常数))

6. 最终减伤
   × (1 - 最终减伤%)

7. 最终伤害
   = 以上计算结果（最低为1）
```

---

## 3. 穿透机制

| 类型 | 说明 | 计算 |
|------|------|------|
| 固定穿透 | 无视固定防御值 | 有效防御 = 防御 - 固定穿透 |
| 百分比穿透 | 无视百分比防御 | 有效防御 = 防御 × (1 - 穿透%) |
| 计算顺序 | 先百分比后固定 | 有效防御 = 防御 × (1 - 穿透%) - 固定穿透 |

**示例**：
> 目标防御100，攻击者有30%穿透和20固定穿透
> 有效防御 = 100 × (1 - 0.3) - 20 = 50

---

## 4. 伤害加成与减免

### 伤害加成
| 类型 | 说明 |
|------|------|
| 通用加成 | 所有伤害增加 |
| 类型加成 | 物理/魔法伤害增加 |
| 技能加成 | 特定技能伤害增加 |
| 目标加成 | 对特定目标伤害增加 |
| 条件加成 | 满足条件时伤害增加 |

**示例**：
> 技能基础伤害100
> 通用伤害+20%，对Boss+30%，低血量时+50%
> 最终伤害 = 100 × 1.2 × 1.3 × 1.5 = 234

### 伤害减免
| 类型 | 说明 |
|------|------|
| 护甲减免 | 来自防御属性 |
| 抗性减免 | 对特定伤害减免 |
| 最终减伤 | 最后阶段的减免 |
| 伤害上限 | 单次伤害不超过某值 |

# 四、战斗机制

## 1. 吸血/生命偷取

| 功能 | 说明 |
|------|------|
| 物理吸血 | 物理伤害的百分比回复 |
| 法术吸血 | 魔法伤害的百分比回复 |
| 全能吸血 | 所有伤害的百分比回复 |
| AOE衰减 | 范围伤害吸血效果降低 |
| 吸血上限 | 单次吸血不超过某值 |

**示例**：
> 攻击造成1000伤害，10%物理吸血，回复100生命。
> AOE技能命中5人，每人吸血效果33%，共回复 100 × 5 × 0.33 = 165。

---

## 2. 反伤/反弹

| 功能 | 说明 |
|------|------|
| 固定反伤 | 每次受击反弹固定伤害 |
| 百分比反伤 | 反弹受到伤害的百分比 |
| 魔法反伤 | 反弹魔法伤害 |
| 反伤类型 | 反伤是物理/魔法/真实 |

**示例**：
> 装备荆棘甲，受到攻击时反弹20点伤害+受伤值的15%。
> 受到100物理伤害，反弹 20 + 15 = 35 伤害给攻击者。

---

## 3. 处决机制

| 功能 | 说明 |
|------|------|
| 斩杀线 | 目标生命低于某百分比时处决 |
| 斩杀伤害 | 额外造成目标缺失生命的伤害 |
| 处决标记 | 满足条件可执行处决动作 |
| 处决免疫 | 某些目标免疫处决 |

**示例**：
> 技能效果：目标生命低于20%时直接击杀。
> 目标最大生命1000，当前300，触发斩杀直接死亡。

---

## 4. 连击系统

| 功能 | 说明 |
|------|------|
| 连击计数 | 连续攻击不中断的次数 |
| 连击加成 | 连击越高伤害越高 |
| 连击时间 | 保持连击的时间窗口 |
| 连击重置 | 被击中或超时重置 |
| 连击里程碑 | 达到特定连击数的奖励 |

**示例**：
> 10连击内：伤害+10%
> 30连击内：伤害+30%
> 50连击以上：伤害+50%
> 被击中连击清零

---

## 5. 无敌帧

| 功能 | 说明 |
|------|------|
| 受击无敌 | 受击后短暂无敌 |
| 翻滚无敌 | 翻滚动作期间无敌 |
| 技能无敌 | 某些技能期间无敌 |
| 无敌穿透 | 某些攻击无视无敌 |

**示例**：
> 受到伤害后获得0.5秒无敌时间，期间不会受到任何伤害。
> 翻滚动作的第3-10帧为无敌帧，共约0.2秒。

# 五、特殊战斗系统

| 元素组合 | 反应 | 效果 |
|---------|------|------|
| 水+火 | 蒸发 | 伤害×2 |

**示例**：
> 敌人身上附着水元素，使用火系技能攻击。

# 六、各类型游戏应用

## 1. 回合制RPG（最终幻想、勇者斗恶龙）

### 战斗特点

| 特点 | 说明 |
|-----|------|
| 回合分明 | 敌我双方轮流行动 |
| 策略深度 | 需要考虑属性克制、队伍配置 |
| 复杂公式 | 多种伤害修正和判定 |
| 节奏可控 | 玩家有充足思考时间 |

### 战斗流程

```
┌─────────────────────────────────────────────────────────────┐
│                       战斗开始                               │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  1. 行动顺序计算                                             │
│     - 按速度排序所有单位                                     │
│     - 速度 = 基础速度 + 装备加成 + Buff加成                   │
│     - 生成本回合行动队列                                     │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  2. 回合开始阶段                                             │
│     - 触发"回合开始"效果（中毒、灼烧等）                       │
│     - Buff/Debuff 回合数-1                                   │
│     - 检查状态（眩晕则跳过行动）                               │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  3. 选择行动（玩家控制）                                      │
│     ┌─────────┬─────────┬─────────┬─────────┐               │
│     │  攻击   │  技能   │  道具   │  防御   │               │
│     └────┬────┴────┬────┴────┬────┴────┬────┘               │
│          ▼         ▼         ▼         ▼                    │
│       普通攻击   选择技能   使用道具   防御姿态                │
│       选择目标   消耗MP     选择道具   伤害减半                │
│                  选择目标   选择目标                          │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  4. 执行行动                                                 │
│     - 播放动作动画                                           │
│     - 命中判定：命中率 vs 闪避率                              │
│     - 暴击判定：暴击率 vs 暴击抵抗                            │
│     - 伤害计算（见下方公式）                                  │
│     - 触发被动效果（反击、吸血等）                            │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  5. 回合结束阶段                                             │
│     - 触发"回合结束"效果                                     │
│     - 检查死亡单位                                           │
│     - 判断胜负条件                                           │
└─────────────────────────┬───────────────────────────────────┘
                          ▼
          ┌───────────────┴───────────────┐
          ▼                               ▼
    战斗继续（下一单位）              战斗结束（结算奖励）
```

### 伤害计算公式

```typescript
// 物理伤害
physicalDamage = (ATK × skillRate - DEF × 0.5) × elementBonus × critMultiplier

// 魔法伤害  
magicalDamage = (MATK × skillRate × (1 + INT/100)) - MDEF × 0.3

// 最终伤害
finalDamage = baseDamage 
    × (1 + damageBonus)           // 伤害加成
    × (1 - damageReduction)       // 伤害减免
    × randomFactor(0.9, 1.1)      // 随机浮动
```

### 典型场景

| 场景 | 处理方式 |
|-----|---------|
| 先手攻击 | 速度高的一方在回合开始时获得额外行动 |
| 全体攻击 | 对所有敌人造成伤害×0.7 |
| 连携技 | 多角色同时行动，伤害叠加+协力加成 |
| 属性克制 | 火→草×2.0，草→水×2.0，水→火×2.0 |
| 防御姿态 | 本回合受到伤害减半，可触发"反击" |

## 2. 动作RPG（ARPG：暗黑、仁王、艾尔登法环）

### 战斗特点

```typescript
    score += combo.variety × 50;       // 使用不同招式加分
```

```typescript
function handleInput(system: MusouComboSystem, input: 'light' | 'heavy' | 'musou'): void {
    switch (input) {
        case 'light':
            if (system.state === ComboState.Idle || 
                (system.state === ComboState.Recovery && system.comboTimer < system.comboWindow)) {
                // 继续轻攻击连段
                system.lightCount = Math.min(system.lightCount + 1, system.maxLightCount);
                system.state = ComboState.LightAttack;
                playAnimation(`light_${system.lightCount}`);
                resetComboTimer();
            }
            break;

            if (system.state === ComboState.LightAttack || 
                system.state === ComboState.Recovery ||
                system.state === ComboState.Idle) {
                // 触发C技
                const cLevel = system.lightCount > 0 ? system.lightCount : 1;
                system.state = ComboState.HeavyAttack;
                playAnimation(`charge_${cLevel}`);
                // 重置连段
                system.lightCount = 0;

            if (musouGauge >= 100) {
                // 释放无双技

                playAnimation('musou_attack');
```

```typescript
// 群体攻击检测（割草游戏核心）
function processAreaAttack(attacker: Player, attack: Attack): void {
    // 获取攻击范围内所有敌人（可能有数十上百个）
    const enemies = getEnemiesInArea(attack.hitbox);
    
    // 批量处理伤害（性能优化）
    const damageResults: DamageResult[] = [];
    
    enemies.forEach(enemy => {
        // 计算伤害
        let damage = attack.baseDamage;
        
        // 杂兵/武将区分
        if (enemy.type === 'grunt') {
            damage *= 1.5;  // 对杂兵伤害加成
            // 杂兵可能直接死亡
            if (enemy.hp <= damage) {
                enemy.die();
                attacker.musouGauge += MUSOU_GAIN_ON_KILL;
                attacker.killCount++;
            }
        } else if (enemy.type === 'officer') {
            // 武将有更多生命和防御
            damage *= (1 - enemy.defense / 100);
        }
        
        damageResults.push({ enemy, damage });
    });
    
    // 批量显示伤害数字（避免性能问题）
    showDamageNumbers(damageResults);
    
    // 更新连击数
    attacker.comboCount += enemies.length;
    updateComboUI(attacker.comboCount);
}
```

```typescript
    links: Move[];           // 中继技

    damage: number;          // 总伤害

// 连段补正（伤害递减）
function calculateComboDamage(moves: Move[]): number {
    let totalDamage = 0;
    let scaling = 1.0;
    
    moves.forEach((move, index) => {
        if (index === 0) {
            totalDamage += move.damage;  // 首击无补正
        } else {
            scaling *= 0.8;  // 每击补正20%
            totalDamage += move.damage * Math.max(scaling, 0.1);
        }
    });
    
    return Math.floor(totalDamage);
}
```

```typescript
function applyCC(target: Hero, ccType: string, duration: number) {
    const recentCC = target.recentCC.filter(cc => 
        Date.now() - cc.time < 10000
    );
    
    // 计算递减
    let reduction = 0;

        reduction += 0.35;  // 每次控制递减35%
```

```typescript
// 散布计算
interface SpreadConfig {
    baseSpread: number;        // 基础散布（站立不动）
    moveSpread: number;        // 移动散布
    jumpSpread: number;        // 跳跃散布
    crouchReduction: number;   // 蹲下减少
    adsReduction: number;      // 开镜减少
    recoilPattern: Vector2[];  // 后坐力模式
}

function calculateSpread(config: SpreadConfig, state: PlayerState): number {
    let spread = config.baseSpread;
    
    if (state.isMoving) spread += config.moveSpread;
    if (state.isJumping) spread += config.jumpSpread;
    if (state.isCrouching) spread *= (1 - config.crouchReduction);
    if (state.isADS) spread *= (1 - config.adsReduction);
    
    // 连射散布增加
    spread += state.shotsFired × 0.5;
    
    return spread;
}

// 弹道计算
function calculateBulletPath(origin: Vector3, direction: Vector3, spread: number): Vector3 {
    const randomAngle = Math.random() × Math.PI × 2;
    const randomRadius = Math.random() × spread;
    
    const offsetX = Math.cos(randomAngle) × randomRadius;
    const offsetY = Math.sin(randomAngle) × randomRadius;
    
    return direction.add(new Vector3(offsetX, offsetY, 0)).normalize();
}
```

```typescript
function towerAttack(tower: Tower, enemy: Enemy): number {
    let damage = tower.baseDamage;
    
    // 属性克制
    if (tower.damageType === 'magic' && enemy.type === 'armored') {
        damage *= 0.5;  // 魔法对重甲减半
    }
    if (tower.damageType === 'physical' && enemy.type === 'flying') {
        damage *= 0;    // 物理无法攻击飞行

    // 护甲减伤
    damage = damage × (1 - enemy.armor / (enemy.armor + 100));
    
    // 暴击
    if (Math.random() < tower.critRate) {
        damage *= tower.critMultiplier;
    }
    
    return damage;
}
```

```typescript
// 卡牌结构
interface Card {
    name: string;
    cost: number;            // 费用
    type: 'attack' | 'skill' | 'power' | 'curse';
    rarity: 'common' | 'uncommon' | 'rare';
    effects: CardEffect[];
    exhaust?: boolean;       // 使用后消耗
    ethereal?: boolean;      // 回合结束消耗
    innate?: boolean;        // 战斗开始在手牌
    retain?: boolean;        // 回合结束保留
}

// 伤害计算（杀戮尖塔风格）
function calculateCardDamage(card: Card, player: Player, enemy: Enemy): number {
    let damage = card.baseDamage;
    
    // 力量加成
    damage += player.strength;
    
    // 易伤（受到伤害+50%）
    if (enemy.hasDebuff('vulnerable')) {
        damage = Math.floor(damage × 1.5);
    }
    
    // 虚弱（造成伤害-25%）
    if (player.hasDebuff('weak')) {
        damage = Math.floor(damage × 0.75);
    }
    
    // 遗物加成
    player.relics.forEach(relic => {
        damage = relic.modifyDamage(damage, card, enemy);
    });
    
    return damage;
}

// 护盾计算
function calculateBlock(card: Card, player: Player): number {
    let block = card.baseBlock;
    
    // 敏捷加成
    block += player.dexterity;
    
    // 脆弱（获得护盾-25%）
    if (player.hasDebuff('frail')) {
        block = Math.floor(block × 0.75);
    }
    
    return block;
}
```

```typescript
// 武器伤害
function calculateWeaponDamage(weapon: Weapon, player: Player): number {
    let damage = weapon.baseDamage;
    
    // 武器等级加成
    damage += weapon.level × weapon.damagePerLevel;
    
    // 力量加成
    damage *= (1 + player.might / 100);
    
    // 暴击
    if (Math.random() < player.luck × 0.01) {
        damage *= 2;
    }
    
    return Math.floor(damage);
}
```
