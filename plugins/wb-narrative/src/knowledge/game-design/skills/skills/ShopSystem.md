# 一、系统概述

## 什么是商店系统

商店系统提供玩家购买和出售物品的功能，是游戏经济的重要组成部分。

## 核心职责

| 职责 | 说明 |
|------|------|
| 商品展示 | 展示可购买商品 |
| 购买处理 | 处理购买请求 |
| 出售处理 | 处理物品出售 |
| 库存管理 | 管理商品库存 |
| 刷新机制 | 商品刷新逻辑 |

## 适用游戏类型

| 游戏类型 | 商店复杂度 | 特点 |
|---------|----------|------|
| **手游** | 高 | 多商店、限购 |
| **RPG** | 中 | NPC商店 |
| **Roguelike** | 中 | 随机商店 |
| **MMORPG** | 高 | 多种货币 |
| **休闲** | 低 | 简单商店 |

# 二、核心功能模块

## 1. 商品浏览

**功能说明**：展示商店中的商品列表。

| 功能 | 说明 |
|------|------|
| 商品列表 | 显示可购买商品 |
| 分类筛选 | 按类型筛选 |
| 搜索功能 | 按名称搜索 |
| 排序功能 | 按价格/类型排序 |
| 详情查看 | 查看商品详情 |

**示例**：

> **商店界面**：显示武器商店的所有武器，可按类型（剑/斧/杖）筛选，按价格排序。

---

## 2. 购买功能

**功能说明**：购买商店中的商品。

| 功能 | 说明 |
|------|------|
| 单件购买 | 购买一件商品 |
| 批量购买 | 购买多件同类商品 |
| 货币检查 | 检查货币是否足够 |
| 背包检查 | 检查背包是否有空间 |
| 购买确认 | 贵重商品二次确认 |

**示例**：

> **购买药水**：选择生命药水，输入数量10，确认购买，消耗1000金币，获得10个药水。
>
> **购买限制**：VIP专属商品，非VIP玩家无法购买。

---

## 3. 出售功能

**功能说明**：向商店出售物品。

| 功能 | 说明 |
|------|------|
| 单件出售 | 出售一件物品 |
| 批量出售 | 出售多件物品 |
| 快速出售 | 一键出售指定类型 |
| 出售价格 | 显示回收价格 |
| 出售确认 | 稀有物品二次确认 |

**示例**：

> **出售装备**：将白色装备出售给商店，获得装备售价（通常为购买价的10-30%）。
>
> **快速出售**：一键出售所有白色品质装备。

---

## 4. 回购功能

**功能说明**：买回误售的物品。

| 功能 | 说明 |
|------|------|
| 回购列表 | 最近出售的物品 |
| 回购价格 | 通常等于出售价格 |
| 回购限制 | 列表容量限制 |
| 时间限制 | 下线后清空 |

**示例**：

> **误售回购**：不小心卖掉一件紫色装备，在回购页面花原价买回。

# 四、商品配置

## 1. 商品属性

```
ShopItem {
  id: "potion_hp_large"
  itemId: "potion_hp_large"   // 对应物品ID
  
  // 价格
  price: {
    currency: "gold",
    amount: 100
  }
  
  // 限购
  limitType: "daily"          // none/daily/weekly/forever
  limitCount: 10
  
  // 库存
  stockType: "unlimited"      // unlimited/limited/refresh
  stockCount: 0
  
  // 条件
  requirements: {
    level: 10,
    vip: 0,
    reputation: { faction: "alliance", level: 2 }
  }
  
  // 折扣
  discount: 0                 // 0-100，0表示无折扣
}
```

---

## 2. 限购设计

| 限购类型 | 说明 | 刷新 |
|---------|------|------|
| 无限购 | 不限制购买次数 | - |
| 每日限购 | 每日重置购买次数 | 每日0点 |
| 每周限购 | 每周重置购买次数 | 每周一0点 |
| 终身限购 | 永久只能购买一次 | 不刷新 |
| 活动限购 | 活动期间限购 | 活动结束 |

**示例**：

> **每日限购**：特价药水每日限购5个，已购买3个，今日还能买2个，明天重置。

---

## 3. 库存设计

| 库存类型 | 说明 | 补充 |
|---------|------|------|
| 无限库存 | 永远有货 | - |
| 有限库存 | 卖完为止 | 不补充 |
| 刷新库存 | 定时刷新 | 定时补充 |
| 共享库存 | 全服共享 | 抢购 |

**示例**：

> **限量商品**：传说武器全服限量100把，卖完下架。
>
> **刷新库存**：神秘商人每6小时刷新一次商品。

# 五、高级功能

## 1. 商店刷新

**功能说明**：刷新商店商品。

| 刷新类型 | 说明 |
|---------|------|
| 自动刷新 | 定时自动刷新 |
| 手动刷新 | 玩家付费刷新 |
| 购买刷新 | 购买后刷新单个 |
| 随机刷新 | 刷新后商品随机 |

**示例**：

> **Roguelike商店**：每个商人有5个随机商品，可花费100金币刷新一次，每层限刷3次。

---

## 2. 折扣系统

| 折扣类型 | 说明 |
|---------|------|
| 固定折扣 | 商品固定折扣 |
| 会员折扣 | VIP专属折扣 |
| 活动折扣 | 活动期间折扣 |
| 首购折扣 | 首次购买折扣 |
| 批量折扣 | 买多优惠 |

**示例**：

> **VIP折扣**：VIP3用户在商店购买享受9折优惠。
>
> **限时折扣**：活动期间所有商品5折。

---

## 3. 礼包系统

**功能说明**：打包多个物品销售。

| 功能 | 说明 |
|------|------|
| 礼包组合 | 多个物品打包 |
| 礼包价格 | 通常比单买便宜 |
| 限购礼包 | 只能购买一次 |
| 等级礼包 | 达到等级解锁 |

**示例**：

> **新手礼包**：包含武器×1、药水×10、金币×1000，原价200钻，现价50钻，限购1次。

---

## 4. 推荐系统

| 功能 | 说明 |
|------|------|
| 热销推荐 | 购买量最高 |
| 个性推荐 | 根据玩家喜好 |
| 新品推荐 | 新上架商品 |
| 即将下架 | 限时商品提醒 |

# 六、各类型游戏应用

## 1. 手游

### 商店特点
- 多种商店
- 付费导向
- 限时限购

### 商店配置
```
普通商店：金币购买
钻石商店：钻石购买
限时商店：限时商品
每日商店：每日刷新
首充商店：首充奖励
```

---

## 2. Roguelike

### 商店特点
- 随机商品
- 刷新机制
- 金币紧张

### 商店配置
```
商品数量：5-8个随机
刷新费用：递增
特殊商人：稀有遭遇
```

---

## 3. MMORPG

### 商店特点
- NPC商店
- 多货币
- 声望解锁

### 商店配置
```
城镇商人：基础物品
声望商人：声望物品
副本商人：副本兑换
```

# 七、功能速查表

## 商店系统方法

| 方法 | 说明 | 参数 |
|------|------|------|
| openShop | 打开商店 | 商店ID |
| getItems | 获取商品列表 | 商店ID |
| buyItem | 购买商品 | 商店ID, 商品ID, 数量 |
| sellItem | 出售物品 | 物品, 数量 |
| canBuy | 检查能否购买 | 商店ID, 商品ID |
| refreshShop | 刷新商店 | 商店ID |
| getBuybackList | 获取回购列表 | - |
| buyback | 回购物品 | 回购项索引 |

## 商店类型

| 类型 | 说明 |
|------|------|
| GENERAL | 杂货店 |
| WEAPON | 武器店 |
| ARMOR | 防具店 |
| CONSUMABLE | 消耗品店 |
| SPECIAL | 特殊商店 |
| EVENT | 活动商店 |

## 商店事件

| 事件 | 数据 | 说明 |
|------|------|------|
| shop:opened | {shopId} | 商店打开 |
| shop:closed | {shopId} | 商店关闭 |
| shop:purchased | {shopId, itemId, count, cost} | 购买商品 |
| shop:sold | {itemId, count, gain} | 出售物品 |
| shop:refreshed | {shopId} | 商店刷新 |
| shop:stock_empty | {shopId, itemId} | 库存售罄 |
