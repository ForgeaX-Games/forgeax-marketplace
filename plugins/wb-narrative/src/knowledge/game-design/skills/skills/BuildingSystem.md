# 一、系统概述

## 什么是建造系统

建造系统允许玩家在游戏世界中放置、建造和管理建筑物或结构。

## 核心职责

| 职责 | 说明 |
|------|------|
| 建筑放置 | 在世界中放置建筑 |
| 碰撞检测 | 检测放置合法性 |
| 资源消耗 | 消耗建造材料 |
| 建筑管理 | 升级、拆除、修复 |
| 功能实现 | 建筑的功能效果 |

## 适用游戏类型

| 游戏类型 | 建造重要性 | 特点 |
|---------|----------|------|
| **建造** | 核心 | 自由建造 |
| **生存** | 核心 | 基地建设 |
| **SLG** | 高 | 城市建设 |
| **模拟经营** | 高 | 设施建设 |
| **RTS** | 高 | 建筑生产 |

# 三、建筑设计

| 装饰建筑 | 美化环境 | 雕像、花园 |

---

## 2. 建筑属性

```
BuildingConfig {
  id: "house"
  name: "木屋"
  icon: "building_house.png"
  model: "house.fbx"
  
  // 尺寸
  size: { width: 2, height: 2 }
  
  // 建造
  buildTime: 60          // 秒
  materials: [
    { id: "wood", count: 100 },
    { id: "stone", count: 50 }
  ]
  
  // 解锁
  unlockLevel: 5
  
  // 效果
  effects: [
    { type: "population", value: 5 }
  ]
  
  // 升级
  maxLevel: 5
  upgrades: [...]
}
```

# 四、放置机制

## 1. 网格系统

| 功能 | 说明 |
|------|------|
| 网格对齐 | 建筑对齐网格 |
| 网格显示 | 显示放置网格 |
| 网格大小 | 网格尺寸设置 |
| 自由放置 | 不受网格限制 |

---

## 2. 碰撞检测

| 检测 | 说明 |
|------|------|
| 重叠检测 | 不能与其他建筑重叠 |
| 地形检测 | 检测地形是否可放置 |
| 边界检测 | 不能超出边界 |
| 距离检测 | 与其他建筑距离要求 |

**放置状态**：
- 绿色预览：可以放置
- 红色预览：无法放置

---

## 3. 放置规则

| 规则 | 说明 |
|------|------|
| 地形限制 | 只能放在特定地形 |
| 连接要求 | 需要连接道路/电网 |
| 距离要求 | 需要离某建筑一定距离 |
| 数量限制 | 同类建筑数量限制 |

# 七、功能速查表

## 建造系统方法

| 方法 | 说明 | 参数 |
|------|------|------|
| startPlacement | 开始放置模式 | 建筑ID |
| cancelPlacement | 取消放置 | - |
| confirmPlacement | 确认放置 | 位置, 旋转 |
| canPlace | 检测能否放置 | 位置 |
| rotatePreview | 旋转预览 | 角度 |
| upgradeBuilding | 升级建筑 | 建筑ID |
| demolishBuilding | 拆除建筑 | 建筑ID |
| moveBuilding | 移动建筑 | 建筑ID, 新位置 |

## 建筑事件

| 事件 | 数据 | 说明 |
|------|------|------|
| building:placed | {buildingId, position} | 放置建筑 |
| building:completed | {buildingId} | 建造完成 |
| building:upgraded | {buildingId, level} | 建筑升级 |
| building:demolished | {buildingId} | 建筑拆除 |
| building:damaged | {buildingId, damage} | 建筑损坏 |
