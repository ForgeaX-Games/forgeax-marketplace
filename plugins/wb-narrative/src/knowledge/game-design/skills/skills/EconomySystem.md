# 一、系统概述

## 什么是经济系统

经济系统管理游戏中的虚拟货币和经济活动，包括货币的获取、消费、交易等。

## 核心职责

| 职责 | 说明 |
|------|------|
| 货币管理 | 管理各种货币的数量 |
| 收支记录 | 记录货币流向 |
| 交易处理 | 处理玩家间交易 |
| 经济平衡 | 控制通货膨胀/紧缩 |

## 适用游戏类型

| 游戏类型 | 经济复杂度 | 特点 |
|---------|----------|------|
| **MMORPG** | 极高 | 玩家经济、拍卖行 |
| **SLG** | 高 | 多资源管理 |
| **模拟经营** | 高 | 经济模拟 |
| **手游** | 中高 | 多货币付费 |
| **RPG** | 中 | 简单金币系统 |

# 二、核心功能模块

## 1. 货币获取

**功能说明**：玩家获得货币的途径。

| 来源 | 说明 |
|------|------|
| 战斗掉落 | 击败敌人获得 |
| 任务奖励 | 完成任务获得 |
| 出售物品 | 向商店出售 |
| 玩家交易 | 玩家间交易 |
| 活动奖励 | 活动获得 |
| 充值购买 | 真实货币兑换 |

**示例**：

> **战斗掉落**：击败怪物，掉落50-100金币。
>
> **任务奖励**：完成主线任务，获得10000金币。

---

## 2. 货币消费

**功能说明**：玩家消耗货币的途径。

| 消费项 | 说明 |
|--------|------|
| 商店购买 | 购买物品 |
| 装备强化 | 强化消耗 |
| 技能学习 | 学习技能 |
| 传送费用 | 使用传送 |
| 修理费用 | 修理装备 |
| 公会贡献 | 公会捐献 |

**示例**：

> **购买物品**：购买生命药水，消耗100金币。
>
> **装备强化**：强化武器从+9到+10，消耗50000金币和10个强化石。

---

## 3. 货币查询

**功能说明**：查询当前货币数量。

| 功能 | 说明 |
|------|------|
| 当前余额 | 查询货币数量 |
| 历史记录 | 查询收支记录 |
| 统计分析 | 收入/支出统计 |

---

## 4. 货币转换

**功能说明**：不同货币之间的兑换。

| 功能 | 说明 |
|------|------|
| 固定汇率 | 固定比例兑换 |
| 浮动汇率 | 根据市场调整 |
| 兑换限制 | 每日兑换上限 |
| 单向兑换 | 只能A换B，不能B换A |

**示例**：

> **钻石兑金币**：1钻石 = 1000金币，每日限兑100次。
>
> **积分兑换**：100积分 = 1抽卡券，单向兑换。

# 三、货币设计

## 1. 货币类型

| 货币 | 获取方式 | 用途 | 特点 |
|------|---------|------|------|
| 金币 | 战斗/任务 | 基础消费 | 易获取 |
| 钻石 | 充值/活动 | 高级消费 | 稀缺 |
| 积分 | 活动/签到 | 兑换道具 | 活动限定 |
| 荣誉 | PVP | 购买PVP装备 | 特定用途 |
| 公会币 | 公会活动 | 公会商店 | 公会专用 |

---

## 2. 货币上限

| 功能 | 说明 |
|------|------|
| 持有上限 | 最多持有的数量 |
| 获取上限 | 每日/周获取上限 |
| 溢出处理 | 超过上限的处理 |

**示例**：

> **金币上限**：最多持有9999999金币，超过部分自动转入邮箱。
>
> **每日上限**：竞技场每日最多获得1000荣誉点。

---

## 3. 货币配置

```
CurrencyConfig {
  id: "gold"
  name: "金币"
  icon: "currency_gold.png"
  
  maxAmount: 99999999
  dailyEarnLimit: 0       // 0=无限制
  
  displayFormat: "{value:,}"  // 千分位显示
  displayInUI: true
  
  exchangeRate: {
    diamond: 1000         // 1钻石=1000金币
  }
}
```

# 四、交易系统

## 1. 玩家交易

**功能说明**：玩家之间直接交易物品和货币。

| 功能 | 说明 |
|------|------|
| 发起交易 | 向对方发起交易请求 |
| 放入物品 | 放入交易物品 |
| 放入货币 | 放入交易金额 |
| 锁定确认 | 双方锁定确认 |
| 完成交易 | 交换物品和货币 |
| 取消交易 | 取消当前交易 |

**示例**：

> **交易流程**：
> 1. 玩家A向玩家B发起交易
> 2. A放入装备，B放入10000金币
> 3. 双方点击锁定
> 4. 双方点击确认
> 5. 交易完成，物品和金币交换

---

## 2. 拍卖行

**功能说明**：玩家通过拍卖系统买卖物品。

| 功能 | 说明 |
|------|------|
| 上架物品 | 设置价格和时间 |
| 浏览物品 | 按条件筛选 |
| 购买物品 | 一口价购买 |
| 竞拍 | 出价竞拍 |
| 下架 | 主动下架物品 |
| 过期处理 | 到期未售的处理 |

**示例**：

> **上架装备**：设置一口价50000金币，竞拍起价20000，持续24小时。
>
> **竞拍机制**：当前最高价30000，玩家出价35000成为最高出价者。

---

## 3. 交易税

**功能说明**：交易时收取的手续费。

| 类型 | 说明 |
|------|------|
| 上架费 | 上架物品时收取 |
| 成交费 | 成功交易时收取 |
| 阶梯费率 | 根据金额不同费率 |

**示例**：

> **拍卖行手续费**：成交时收取成交额的5%作为手续费。

---

## 4. 邮寄系统

**功能说明**：通过邮件发送物品和货币。

| 功能 | 说明 |
|------|------|
| 发送邮件 | 附带物品/货币 |
| 邮费 | 发送需要费用 |
| 领取附件 | 领取邮件物品 |
| 退回机制 | 超时未领取退回 |

# 六、各类型游戏应用

## 1. MMORPG

### 经济特点
- 玩家经济为主
- 拍卖行系统
- 通胀控制

### 货币配置
```
主货币：金币（游戏内获取）
付费货币：钻石（充值获取）
绑定货币：绑定金币（不可交易）
```

---

## 2. 手游/F2P

### 经济特点
- 多货币设计
- 付费驱动
- 数值膨胀

### 货币配置
```
金币：基础货币
钻石：付费货币
体力：时间货币
各种代币：活动货币
```

---

## 3. 模拟经营

### 经济特点
- 经济模拟
- 市场波动
- 投资回报

### 货币配置
```
资金：主要货币
资源：多种资源
股票/投资：金融系统
```

# 七、功能速查表

## 经济系统方法

| 方法 | 说明 | 参数 |
|------|------|------|
| getCurrency | 获取货币数量 | 货币类型 |
| addCurrency | 增加货币 | 货币类型, 数量, 来源 |
| spendCurrency | 消费货币 | 货币类型, 数量, 用途 |
| canAfford | 检查是否足够 | 货币类型, 数量 |
| exchange | 货币兑换 | 源货币, 目标货币, 数量 |
| transfer | 转账 | 发送者, 接收者, 数量 |
| getHistory | 获取历史记录 | 货币类型, 时间范围 |

## 货币类型

| 类型 | 说明 |
|------|------|
| GOLD | 金币 |
| DIAMOND | 钻石 |
| POINT | 积分 |
| HONOR | 荣誉 |
| GUILD_COIN | 公会币 |

## 经济事件

| 事件 | 数据 | 说明 |
|------|------|------|
| economy:earned | {currency, amount, source} | 获得货币 |
| economy:spent | {currency, amount, purpose} | 消费货币 |
| economy:exchanged | {from, to, amount} | 货币兑换 |
| economy:transferred | {from, to, amount} | 转账 |
| economy:insufficient | {currency, required, current} | 货币不足 |
