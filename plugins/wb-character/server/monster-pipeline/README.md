# 怪物生成平台

一键输入怪物描述，自动生成 **8方向 × 5动画 × 4帧** 完整 2D 俯视角精灵素材包。

适用于 Hades 风格 / 俯视角 ARPG / Roguelike 等游戏的怪物批量生产。

---

## 环境要求

- **Python 3.10+**（推荐 3.11）
- **操作系统**：Windows / macOS / Linux 均可
- **网络**：需要能访问 Google Gemini API（默认）或 OpenAI API

检查 Python 版本：

```bash
python --version
```

如果没有 Python，请前往 https://www.python.org/downloads/ 下载安装。

---

## 安装步骤

### 第一步：解压

把 `monster-pipeline-v2.zip` 解压到任意目录，比如桌面。

### 第二步：安装依赖

打开终端（Windows 用 PowerShell 或 CMD，macOS/Linux 用 Terminal），进入解压后的目录：

```bash
cd monster-pipeline-v2
pip install -r requirements.txt
```

> 如果 pip 报权限错误，在前面加 `python -m pip install -r requirements.txt`。
>
> 首次安装 `rembg` 会自动下载一个 ~170MB 的去背景模型文件，需要等一会。

### 第三步：启动服务

```bash
python server.py
```

看到如下输出表示启动成功：

```
==================================================
  怪物生成平台 V2
  http://localhost:5000
==================================================
```

### 第四步：打开浏览器

在浏览器地址栏输入：

```
http://localhost:5000
```

看到深色主题的怪物生成界面即可开始使用。

> 如果同局域网的其他电脑也想访问，用你的 IP 地址替换 localhost：
> `http://你的IP:5000`

## 使用方法

### 1. 填写怪物信息

打开页面后，你会看到一个输入表单。需要填写：

| 字段 | 是否必填 | 怎么填 | 示例 |
|------|---------|-------|------|
| **怪物名称** | 必填 | 英文 ID，不要空格，会作为文件夹名 | `FireDragon` |
| **外观描述** | 必填 | 越详细角色越稳定。写清楚主要特征 | "暗红色鳞片覆盖的火龙，头部有两只金色弯角，翼展宽大，尾部燃烧着蓝色火焰" |
| **主体颜色** | 建议填 | 锁定角色主色调，提升一致性 | "深红 #8B0000" |
| **辅助颜色** | 可选 | 角色特征色 | "金色角 #FFD700" |
| **第三色** | 可选 | 额外的配色 | "蓝色火焰 #0066FF" |
| **大类 / 子类 / 种族** | 可选 | 三级分类，会自动关联形态 | 元素类 → 火系 → 火龙 |
| **画风** | 默认赛璐珞2D | 点击切换，可选像素风 | CEL_2D |
| **形态** | 默认四足兽 | 根据怪物类型选择 | 四足兽 / 人形 / 蛇形 / 虫形 / 漂浮 / 无定形 |
| **视角** | 默认60° | 俯视角角度 | 60 |

### 2. 配置 AI 模型（一般不用改）

页面底部有模型配置区域：

- **模型**：默认 `nanobanana-pro`（Google Gemini），效果最好，支持参考图输入
- **API Key**：已内置默认 Key，可直接使用。如果额度用完可替换成自己的
- **API Base**：默认留空即可。如果用 OpenAI 兼容接口可以填写

### 3. 点击生成

点击 **「一键生成怪物」** 按钮，平台会自动执行全流程：

```
生成立绘 → 5方向精灵帧生成(50张图) → 3方向镜像翻转 → 组装精灵条 → 质检打包
```

- 全程约 **8-15 分钟**（取决于 API 速度）
- 页面上会实时显示当前进度（哪个方向、哪个动画）
- **刷新页面不会丢失进度**，会自动恢复

### 4. 查看结果

生成完成后页面会显示：

- 左侧：怪物立绘
- 中间：8个方向的动画 GIF 预览
- 上方：生成统计信息

### 5. 下载素材包

点击 **「下载 ZIP 素材包」** 按钮，即可下载包含所有素材的压缩包。

## 输出素材说明

下载的 ZIP 解压后结构如下：

```
FireDragon/
│
├── hero_512.png                 ← 高精度立绘 (512×512 透明背景)
│
├── S/                           ← 南方向 (面朝屏幕下方)
│   ├── idle.png                 ← 站立动画精灵条 (4帧横排拼接)
│   ├── walk.png                 ← 行走动画精灵条
│   ├── atk.png                  ← 攻击动画精灵条
│   ├── hit.png                  ← 受击动画精灵条
│   ├── die.png                  ← 死亡动画精灵条
│   ├── idle.gif                 ← 站立动画 GIF 预览
│   ├── walk.gif / atk.gif / ... ← 其他动画 GIF
│   └── frames/                  ← 单帧 PNG
│       ├── idle_F0.png          ← 第1帧
│       ├── idle_F1.png          ← 第2帧
│       ├── idle_F2.png          ← 第3帧
│       ├── idle_F3.png          ← 第4帧
│       └── walk_F0.png ~ die_F3.png
│
├── SE/ E/ NE/ N/                ← 另外4个AI生成方向 (结构同上)
├── SW/ W/ NW/                   ← 3个镜像翻转方向 (结构同上)
│
├── monster_config.json          ← 引擎接入配置文件
└── preview.html                 ← 用浏览器打开可离线预览全部动画
```

### 方向系统

```
         N (正北/背面)
         ↑
  NW ↗       ↖ NE
W ←    怪物    → E
  SW ↘       ↙ SE
         ↓
         S (正南/正面)
```

- **AI 生成**（5方向）：S、SE、E、NE、N
- **代码镜像**（3方向）：SW = 翻转 SE、W = 翻转 E、NW = 翻转 NE

### 动画说明

| 动画名 | 帧数 | 循环 | 帧率 | 说明 |
|--------|------|------|------|------|
| idle   | 4    | 循环 | 6fps | 放松站姿，微微呼吸 |
| walk   | 4    | 循环 | 8fps | 行走步态循环 |
| atk    | 4    | 单次 | 10fps | 蓄力→攻击→收势 |
| hit    | 4    | 单次 | 12fps | 受击闪白→后仰→恢复 |
| die    | 4    | 单次 | 6fps | 踉跄→倒下→死亡 |

### 在游戏引擎中使用

**精灵条（推荐）**：每个 `<动画>.png` 是 4 帧横排拼接的透明 PNG。在引擎中设置水平切割为 4 列即可。

**单帧**：`frames/` 目录下是已拆分的独立帧，可直接导入。

**配置文件**：`monster_config.json` 包含所有动画的帧数、帧率、是否循环等元数据，可用代码直接读取。

## 常见问题

**Q: 生成速度很慢？**
A: 正常。50 张图需要逐张调用 AI API，加上去背景和组装，8-15分钟是正常范围。5 个方向是并行生成的，已经是最优速度了。

**Q: 生成出来的角色前后不一致？**
A: 平台会先生成一张高精度立绘，后续所有帧都以立绘作为参考图输入 AI，保证一致性。如果仍然不够一致，在描述里写得越详细越好（具体的颜色、身体特征、装饰物等）。

**Q: 页面刷新了怎么办？**
A: 不用担心，页面状态自动保存在浏览器中。刷新后会自动恢复进度，正在生成的任务会继续。

**Q: API Key 用完了？**
A: 在页面底部的 API Key 输入框中替换成你自己的 Google Gemini Key。获取地址：https://aistudio.google.com/app/apikey

**Q: 可以用 OpenAI 的模型吗？**
A: 可以。在模型下拉框选择 `dall-e-3`，填入 OpenAI API Key 即可。但 OpenAI 不支持参考图输入，角色一致性会略差。

**Q: 需要 VPN 才能跑吗？**
A: 不需要。只要有普通互联网就行。去背景功能有两种模式：
- **远程 API 云端**（需配置 `REMOTE_BG_API_URL` + COS 凭证，效果最好）
- **本地 rembg**（不需要任何网络，自动降级）

没有配置远程 API 或缺少 COS 凭证时会自动使用本地 rembg，完全不影响使用。要启用云端去背景，把 `cos-credentials.json` 放到 `config/` 目录并设置 `REMOTE_BG_API_URL` 环境变量即可。

**Q: 端口 5000 被占用了？**
A: 修改 `server.py` 最后一行的 `port=5000` 改成其他端口号（如 8080），重新启动。

**Q: 可以在服务器上跑吗？**
A: 可以。启动时会监听 `0.0.0.0:5000`，局域网内任何机器都可以通过 IP 访问。

## 项目结构

```
monster-pipeline-v2/
├── server.py              ← 主程序入口（启动这个）
├── requirements.txt       ← Python 依赖列表
├── README.md              ← 本文件
├── .gitignore
├── pipeline/              ← 核心管线模块
│   ├── config.py          ←   全局配置（方向/动画/阈值）
│   ├── hero_gen.py        ←   立绘生成
│   ├── prompt_gen.py      ←   AI 提示词自动生成
│   ├── image_gen.py       ←   AI 图像生成（Gemini / OpenAI）
│   ├── bg_removal.py      ←   去背景（云端 + 本地双保险）
│   ├── assembler.py       ←   精灵图拆分/清理/缩放/对齐/组装
│   ├── packager.py        ←   质检 + 配置生成 + ZIP 打包
│   └── cos_helper.py      ←   云存储上传辅助
├── static/
│   └── index.html         ← 前端页面
├── scripts/               ← 独立工具脚本
└── references/            ← 技术参考文档
```
