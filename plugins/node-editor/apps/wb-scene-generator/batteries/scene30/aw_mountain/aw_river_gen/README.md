# 山地河流生成 (aw_river_gen)

在指定地块类型（默认草地）内生成一条有机弯曲的河流，宽度根据当前地形宽阔程度自动变化。

## 算法原理

```
terrainGrid
    │
    ▼
[BFS 距离变换] → distField（每格 = 到最近非目标地块的距离）
    │               宽阔区中心值大，狭窄通道值小
    │
    ▼
[带噪声 Dijkstra] → path（从深部随机起点到远端终点的有机路径）
    │                 代价函数：偏好高距离场值格子 + 噪声扰动蜿蜒
    │
    ▼
[distField 采样 → 高斯平滑] → smoothedWidths（沿路径的平滑宽度序列）
    │
    ▼
[SDF 圆形渲染] → riverMask（二值掩码）
```

## 功能特点

1. **自适应宽度**：宽阔草地中河流自动变粗，狭窄通道自动变细，完全由地形形状驱动
2. **有机弯曲**：Dijkstra 路径代价函数叠加确定性噪声扰动，产生自然蜿蜒效果
3. **平滑过渡**：沿路径对宽度做高斯平滑，避免粗细突变
4. **地形感知**：路径严格限制在目标地块内，不会穿越山地或其他区域

## 适用情况

- 野外山地场景的草地区域河流生成
- 搭配 `grid_max_merge` 将河流掩码叠加到地块网格

## 基本使用方法

1. 将 `biome_classifier` 的 `terrainGrid` 输出连接至本节点 `terrainGrid` 输入
2. 将 `riverMask` 输出连接至下游的覆盖/合并节点（如 `grid_max_merge`）
3. 调整 `widthScale` 控制整体粗细，`noiseStrength` 控制蜿蜒程度

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| terrainGrid | grid | 必填 | 地块分类器输出的 ID 网格 |
| targetBiome | number | 3 | 河流所在地块 ID（3=草地/橙色）|
| widthScale | number | 0.55 | 宽度缩放系数（0.3=细 ~ 1.0=粗）|
| noiseStrength | number | 0.45 | 蜿蜒强度（0=直线 ~ 1=高度蜿蜒）|
| smoothSigma | number | 8 | 宽度过渡平滑半径（格子数）|
| seed | number | 42 | 随机种子 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| riverMask | grid | 河流为 1、其余为 0 的二值掩码 |

## 注意事项

1. **目标地块需足够宽阔**：若草地区域过于细碎（最大距离场值 < 3），河流宽度会很细甚至为 1 格
2. **河流只有一条**：本节点生成单条河流；如需多条，可多次连接不同种子的本节点后用 `grid_max_merge` 合并
3. **叠加到地块图**：`riverMask` 输出为 0/1，需用叠加节点（如 `grid_max_merge`）将河流 ID 写入地块网格后才能被渲染为颜色
