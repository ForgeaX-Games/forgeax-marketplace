# terrain_dome_points — 穹顶中心点生成

## 功能

在 0–1 归一化坐标空间内随机采样多个穹顶中心点，用作 `radial_height_boost` 的输入。

使用**拒绝采样**保证任意两点之间满足最小间距约束，避免多个穹顶过度重叠。当空间已满无法再放置时，会降级选择使现有点间距尽量最大的候选点。

## 输入

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `count` | number | 3 | 生成的点数量 |
| `minSpacing` | number | 0.25 | 点间最小距离（归一化，0–1） |
| `margin` | number | 0.1 | 距地图边缘的最小距离（归一化） |
| `seed` | number | 0 | 随机种子；0 使用时间戳 |

## 输出

| 端口 | 类型 | 说明 |
|------|------|------|
| `points` | array | `{x, y}[]` 点列表，坐标归一化 (0–1) |

## 使用方法

1. 将此节点连接到 `radial_height_boost` 的 `points` 输入
2. 调整 `count` 控制山体/台地数量（建议 2–5）
3. `minSpacing` 建议约为 `radius × 2`，使各穹顶有足够间隔
4. 修改 `seed` 得到不同的空间布局

## 示例参数

```json
{ "count": 3, "minSpacing": 0.28, "margin": 0.12, "seed": 42 }
```

输出示例：
```json
[
  { "x": 0.31, "y": 0.22 },
  { "x": 0.68, "y": 0.55 },
  { "x": 0.19, "y": 0.71 }
]
```
