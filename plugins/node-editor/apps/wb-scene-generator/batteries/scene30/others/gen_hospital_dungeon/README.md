# 医院地牢生成 (gen_hospital_dungeon)

使用BSP（二叉空间分割）算法生成医院风格的地牢地图，包含相互连通的矩形房间和L形走廊。

## 功能特点

1. **BSP递归分割**：将地图递归划分为子区域，每个叶节点生成一个矩形房间
2. **L形走廊连接**：相邻BSP节点之间用2格宽L形走廊连通，保证全图连通性
3. **传送门放置**：传送门位于最后一个房间中央
4. **装饰物生成**：自动在房间中放置病床、监控器、椅子，走廊中放置灯光
5. **掩码ID输出**：0=虚空边界，1=墙，2=地板，3=走廊，5=传送门

## 适用情况

- 医院、实验室、精神病院等室内地牢场景
- 需要房间分明、走廊连接的方块地图
- 恐怖游戏、Roguelike地牢类游戏场景生成

## 基本使用方法

直接连接 `outputGrid` 和 `outputNameList` 到渲染电池即可使用，`spawnX/Y` 用于放置玩家，`decors` 用于渲染装饰物。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| width | number | 72 | 网格宽度（列数），最小40 |
| height | number | 54 | 网格高度（行数），最小30 |
| seed | number | 0 | 随机种子，0使用当前时间戳 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGrid | grid | 生成的地牢网格 |
| outputNameList | array | 掩码ID与名称映射 |
| spawnX | number | 出生点X瓦片坐标 |
| spawnY | number | 出生点Y瓦片坐标 |
| portalX | number | 传送门X瓦片坐标 |
| portalY | number | 传送门Y瓦片坐标 |
| decors | array | 装饰物列表，每项含 {tx, ty, type} |

## 注意事项

1. **网格尺寸**：宽高需足够大（建议40×30以上），否则BSP可能生成极少房间
2. **种子可复现**：相同seed产生相同地图，seed=0每次随机
3. **装饰物坐标**：decors中的tx/ty为瓦片坐标，需乘以瓦片尺寸转换为像素坐标
