# 棋盘格道路-路段抖动 (chess_road_jitter)

规则网格路段在每两个交叉点之间被分为若干小节，每节加上随机垂直抖动（竖向路段左右抖，横向路段上下抖）。路网在交叉点处始终对齐，保证整体连通性。

## 功能特点

1. **保持连通性**：所有交叉点坐标固定不动，只有路段内部节点发生偏移，路网始终完全连通
2. **随机抖动可控**：`jitterAmp` 控制最大偏移格数，`segmentCount` 控制折弯密度
3. **有韵律的随机感**：路线在规则骨架上抖动，不像噪声扰动那样无规律，更有节奏感

## 适用情况

- 中世纪城镇、有机生长的历史街道
- 既要保持规则棋盘格拓扑，又想增加手绘折线感
- 轻度随机（`jitterAmp` ≤ 3）时接近标准棋盘格，重度随机（`jitterAmp` 6–8）时接近锯齿街道

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| inputGrid | grid | — | 源掩码网格，所有非零单元格均视为可用区域 |
| mainSpacing | number | 24 | 主路网格间距（单元格数），推荐 12–40 |
| subSpacing | number | 8 | 辅路网格间距（单元格数），推荐 4–16 |
| mainRoadWidth | number | 2 | 主干道宽度（单元格数），推荐 1–4 |
| subRoadWidth | number | 1 | 支路宽度（单元格数），推荐 1–2 |
| jitterAmp | number | 3 | 路段中间节点最大偏移量（格数），0 = 完全规则，推荐 0–8 |
| segmentCount | number | 4 | 每段分节数（两交叉点之间），推荐 2–8 |
| seed | number | 0 | 随机种子，0 = 当前时间 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| mainRoad | grid | 主路掩码：主路 = 1，其余 = 0 |
| subRoad | grid | 辅路掩码：辅路 = 1，其余 = 0 |
| parcels | grid | 多值地块：每块唯一ID（1, 2, 3…），非地块 = 0 |
| nameList | array | 地块名称清单：`[{id, name}]` |

## 注意事项

1. **jitterAmp 不宜超过 spacing/3**：抖动幅度过大时路线可能与相邻路段重叠，产生奇异地块
2. **segmentCount 越大性能越低**：`segmentCount` 与路段数量相乘，较大时稍慢
3. **subSpacing 需小于 mainSpacing**，否则辅路和主路位置重叠
4. **可复现结果**：指定非零 `seed` 保证每次结果一致
