# 棋盘格道路 (chess_road)

使用两级 BSP（二叉空间分割）在输入网格的所有非零区域内生成城市棋盘格道路网络，输出独立的主路掩码、辅路掩码和多值地块网格。

## 功能特点

1. **两级 BSP 分层生成**：第一级产生主路和超级块，第二级在每个超级块内再分割产生辅路和独立地块
2. **三路独立输出**：主路、辅路、地块各占一个输出端口，下游可分别处理或叠加
3. **多值地块**：每个地块拥有唯一整数 ID（1, 2, 3…），方便后续对单个地块施加建筑、绿化等操作
4. **名称清单输出**：同步输出 `nameList`，格式为 `[{id, name}]`，与 `districts` 字段兼容
5. **掩码感知**：所有道路和地块只生成在输入网格的非零区域内，不规则输入形状也能正确处理

## 适用情况

- 在已有地形掩码内生成城市街区布局
- 作为建筑生成、地块填充等下游节点的前置步骤
- 需要有机随机感的街区布局（地块大小不均）

## 基本使用方法

1. 将目标区域掩码网格连接到 `inputGrid`，网格中所有非零单元格均视为可用区域
2. 调整 `mainRoadWidth`、`mainBlockMinSize`（控制主路疏密）
3. 调整 `subRoadWidth`、`parcelMinSize`（控制地块大小）
4. 执行后各输出端口分别给出主路、辅路、地块和名称清单

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| inputGrid | grid | — | 源掩码网格，所有非零单元格均视为可用区域 |
| mainRoadWidth | number | 2 | 主干道宽度（单元格数），推荐 2–4 |
| subRoadWidth | number | 1 | 支路宽度（单元格数），推荐 1–2 |
| mainBlockMinSize | number | 20 | 主路分割后超级块的最小边长，控制主路疏密 |
| parcelMinSize | number | 8 | 支路分割后地块的最小边长，控制地块大小 |
| splitRatio | number | 0.4 | 分割比例下限（0–0.5），越大越均匀，推荐 0.35–0.45 |
| seed | number | 0 | 随机种子，0 = 当前时间 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| mainRoad | grid | 主路掩码：主路 = 1，其余 = 0 |
| subRoad | grid | 辅路掩码：辅路 = 1，其余 = 0 |
| parcels | grid | 多值地块：每块唯一ID（1, 2, 3…），非地块 = 0 |
| nameList | array | 地块名称清单：`[{id, name}]`，仅包含实际存在的地块条目 |

## 注意事项

1. **最小尺寸约束**：非零区域最小边长需大于 `mainBlockMinSize * 2 + mainRoadWidth` 才能产生任何主路分割；若区域过小，输出可能只有一个整体地块
2. **主路与辅路不重叠**：辅路只在超级块内部生长，不会覆盖主路单元格；两个输出可以安全叠加
3. **地块不含道路**：`parcels` 输出中所有主路/辅路单元格均为 0，不会与道路重叠
4. **可复现结果**：指定非零 `seed` 保证每次结果一致
