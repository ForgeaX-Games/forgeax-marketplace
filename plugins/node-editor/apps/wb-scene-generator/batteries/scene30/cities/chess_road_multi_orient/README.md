# 棋盘格道路-多朝向 (chess_road_multi_orient)

将掩码区域随机划分为若干 Voronoi 子区，每个子区随机选取一个旋转角度绘制网格路网，子区边界作为主路，形成多期规划叠加的复杂城市感。

## 功能特点

1. **多方向路网拼接**：每个子区的路网朝向不同（0/15/30/45/60/75/90°），整体像不同时期规划的城区拼接
2. **Voronoi 自然分区**：子区形状由随机 Voronoi 图决定，边界自然弯曲
3. **子区边界即主路**：不同朝向区域的交界线自动绘制为主路
4. **随机感最强**：在所有棋盘格算法中视觉冲击力最强

## 适用情况

- 多期规划叠加的复杂大城市布局
- 需要强烈视觉差异感的城市地图
- 表现不同文化区域混合的城市（如租界、古城区 + 新区）

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| inputGrid | grid | — | 源掩码网格，所有非零单元格均视为可用区域 |
| mainSpacing | number | 24 | 主路网格间距（单元格数），推荐 12–40 |
| subSpacing | number | 8 | 辅路网格间距（单元格数），推荐 4–16 |
| mainRoadWidth | number | 2 | 主干道宽度（单元格数），推荐 1–4 |
| subRoadWidth | number | 1 | 支路宽度（单元格数），推荐 1–2 |
| zoneCount | number | 4 | Voronoi 子区数量，推荐 2–8 |
| seed | number | 0 | 随机种子，0 = 当前时间 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| mainRoad | grid | 主路掩码：主路 = 1，其余 = 0 |
| subRoad | grid | 辅路掩码：辅路 = 1，其余 = 0 |
| parcels | grid | 多值地块：每块唯一ID（1, 2, 3…），非地块 = 0 |
| nameList | array | 地块名称清单：`[{id, name}]` |

## 注意事项

1. **旋转角度候选集固定**：0/15/30/45/60/75/90°，同一 seed 下每个子区的角度确定
2. **子区数量过多时**每个子区面积缩小，路网内部可能产生碎片地块
3. **subSpacing 需小于 mainSpacing**，否则辅路和主路位置重叠
4. **可复现结果**：指定非零 `seed` 保证每次结果一致
