# islands_path_generate (islands_path_generate)

在岛屿可步行区域内生成随机漫步土路，用来形成探索和到达关系。

## 功能特点

1. **轻量路径**：不依赖复杂寻路，只用随机漫步快速铺出自然小路。
2. **自动避障**：不会主动穿过水域、山体、雪地和悬崖。
3. **风格可调**：通过数量和步数控制路径疏密与长度。

## 适用情况

- 岛屿地形已经成型，需要增加人工活动痕迹时。
- 需要让后续 POI 更容易被路径串联时。
- 不适合精确道路规划或主城道路网设计。

## 基本使用方法

把 `islands_terrain_refine.outputGrid` 连接进来。生成的 `outputGrid` 一般继续传给 `islands_poi_layout`。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grid | grid | - | 输入地形网格 |
| seed | number | 0 | 随机种子 |
| pathCount | number | 6 | 路径条数 |
| stepMin | number | 20 | 单条路径最短步数 |
| stepMax | number | 60 | 单条路径最长步数 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGrid | grid | 增加土路后的地形 |
| outputNameList | array | 对应名称清单 |

## 处理规则说明

### 处理规则格式

```json
[]
```

### 处理规则参数

本电池不使用 `processRules`，路径生成仅由基础参数控制。

## 使用示例

### 输入示例

```json
{
  "grid": [[3,3,3,3,6],[3,10,3,3,6]],
  "seed": 42,
  "pathCount": 6,
  "stepMin": 20,
  "stepMax": 60
}
```

### 输入文件格式

```json
{
  "districts": [{"id":11,"name":"土路"}],
  "grid": [[3,3,3,3,6],[3,10,3,3,6]]
}
```

### 输出示例

```json
{
  "outputGrid": [[3,11,11,3,6],[3,10,11,11,6]],
  "outputNameList": [{"id":11,"name":"土路"}]
}
```

## 注意事项

1. **不是寻路图**：它更像“踩出来的小路”，不是两点之间的最优路径。
2. **尺寸较小时更密**：小地图里 `pathCount` 过大会让土路覆盖过重。
3. **已有泥地可被覆盖**：路径会把可步行泥地改写成 `土路`。
