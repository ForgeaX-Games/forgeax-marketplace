# RTS走廊道路生成 (rts_road_gen)

从 `baseGrid` 掩码中**自动识别各独立区域**，取每个区域离地图中心最近的边界格子作为"入口点"，从地图中心出发构建**树状折线走廊网络**。

## 核心思路

传统方式连接的是基地的"质心"（区域内部的面积中心），道路会穿入基地内部。  
本电池改为连接每个区域的**边界入口点**（靠近地图中心的边界格子），道路更自然地从外部接入基地边缘。

### 入口点提取流程

```
baseGrid 掩码
   ↓
BFS 连通分量标记 → 识别所有独立区域（region 1, 2, 3...）
   ↓
每个区域 → 扫描所有边界格子（有空地邻居的格子）
   ↓
取离地图中心最近的边界格子 → 该区域的"入口点"
   ↓
按区域大小降序排列，取前 maxRegions 个
```

### 节点树结构

```
         中心 (出发点)
           |
     ←拐点← | →拐点→
           |
       分叉节点 J           ← enableBranch=true 时生成
         /       \
   ←拐点←         →拐点→
       /               \
 入口点(终点A)         侧点(终点B)
```

### 入口点 vs 质心

| | 质心（旧） | 边界入口点（新） |
|--|--|--|
| 位置 | 区域面积的中心 | 区域中最靠近地图中心的边界格子 |
| 含义 | 基地内部坐标 | 基地面向中心的"门口" |
| 道路走向 | 穿入基地内部 | 接到基地边缘，更自然 |

## 拐点生成原理

拐点不是曲线，而是在 A→B 连线的等分位置上，**沿垂直方向随机偏移**出来的折叠节点：

```
A ──────── 拐点 ──────── B    (两段直线，仅H/V/45°方向)
             ↑
       垂直 A-B 方向偏移
       大小 = waypointOffset × 线段长度
```

`waypointsPerLeg=1` 时路径为 V 形折线；`=2` 时为 W 形；`=0` 时为直线。

## 适用情况

- SC2/Warcraft 风格 RTS 地图：需要折线走廊连接各角落基地
- 配合 `rts_quad_symmetry` + `rts_base_shape_gen` 组成完整 RTS 地图管线
- 掩码中有 4~8 个独立区域时效果最佳

## 基本使用方法

```
[rts_quad_symmetry]
    └── fullGrid    ──► [rts_road_gen].baseGrid
                        （无需再接 baseCenters，电池自动识别区域）

[rts_road_gen]
    ├── roadGrid    ──► （走廊层，与 baseGrid 合并得到完整平台层）
    └── centerGrid  ──► （中心枢纽层）
```

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| baseGrid | grid | — | 含基地平台区域的地图掩码（1=平台，0=空地），电池自动识别各连通区域 |
| maxRegions | number | 8 | 从掩码中提取的最大区域数（即终点数量），按区域大小降序取前N个 |
| roadWidth | number | 5 | 走廊膨胀半径（格），实际宽度约为 2×roadWidth+1 |
| centerRadius | number | 8 | 地图中心枢纽圆形区域的半径（格） |
| enableBranch | bool | true | true=每臂分叉（J→入口点 + J→侧点）；false=星形直连各入口点 |
| waypointsPerLeg | number | 1 | 每条路径段插入的拐点数量（0=直线，1=一次折弯，2=两次折弯，最大3） |
| waypointOffset | number | 0.3 | 拐点垂直偏移幅度，以线段长度比例表示（0=无偏移，0.5=强折弯） |
| junctionDist | number | 0.6 | 分叉节点在中心→入口点连线上的位置（0=靠近中心，1=靠近入口点） |
| branchSpread | number | 0.5 | 侧支终点垂直偏移量，以「J到入口点距离」为单位（0=无偏移，1=等距偏移） |
| diagFirst | bool | true | true=先走对角线再直行；false=先直行再对角线，影响段内折角位置 |
| seed | number | 0 | 随机种子，0 使用当前时间 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| roadGrid | grid | 走廊与中心枢纽的合并掩码（1=走廊/枢纽，0=空） |
| centerGrid | grid | 仅中心圆形枢纽区域的掩码 |

## 参数调节建议

| 效果 | 调整方式 |
|------|----------|
| 走廊更宽 | 增大 `roadWidth`（推荐 4~8） |
| 折线弯折更明显 | 增大 `waypointOffset`（0.3~0.5） |
| 路径更弯曲（多折） | 增大 `waypointsPerLeg`（2~3） |
| 路径全部为直线 | 设 `waypointsPerLeg=0` |
| 分叉点更靠近基地 | 增大 `junctionDist`（0.6~0.8） |
| 关闭分叉（各自独立） | 设 `enableBranch=false` |
| 提取更多/更少终点 | 调整 `maxRegions`（与掩码中区域数量匹配） |

## 注意事项

1. **无需 baseCenters**：v4 起电池直接从 `baseGrid` 自动提取入口点，管线中不需要再连接 `rts_quad_symmetry.baseCenters`
2. **区域数量**：若掩码中区域数 < `maxRegions`，则以实际区域数为准；若 > `maxRegions`，取面积最大的前N个
3. **拐点随机性**：不同 `seed` 下走廊形态各异；`seed=0` 每次生成不同结果
4. **侧支越界**：`branchSpread` 过大时侧支终点可能超出地图边界，走廊仍可生成但会在边界截断
