# 边界区域分布 (ring_zone_scatter)

沿网格四周边界均匀放置 N 个锚点，每个区域从边界向内竞争膨胀，直接生成紧贴场地边缘的有机区域。

## 功能特点

1. **真正贴边**：锚点直接落在网格上/下/左/右边界像素，区域从边界向内生长
2. **均匀分布**：按周长等分放置 N 个锚点，分布在四条边上
3. **向内深度控制**：`maxDepth` 限制区域向内扩展的最大格数
4. **有机形状**：多子种子竞争 BFS 膨胀，形成自然凹凸边缘
5. **区域独立**：跨区域斥力保证各区域之间有间隙，不粘连
6. **任意数量**：支持任意 N 个区域，4 个角落、6 个均匀分布等均可

## 适用情况

- RTS/MOBA 地图需要在场地四周边界放置多个基地区域
- 地图边缘的营地、哨站、资源点等分布
- 需要"守卫四边"布局的任何地形生成

## 基本使用方法

1. 将 `rect_grid` 接入 `grid` 端口（仅读取尺寸）
2. 设置 `zoneCount`（区域数量，默认 4）
3. 执行后将 `baseGrid` 接入 `rts_road_gen` 生成走廊道路

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grid | grid | — | 基础网格，仅用于读取宽高 |
| zoneCount | number | 4 | 沿四边均匀分布的区域数量 |
| maxDepth | number | 0 | 向内最大膨胀深度（格），0=自动（约 min(宽,高)×20%） |
| subSeeds | number | 3 | 每区域子种子数，越多形状越丰富，建议 2~5 |
| gapWidth | number | 3 | 区域间最小斥力间隙（格），建议 2~6 |
| growProb | number | 0.88 | BFS 膨胀基础概率（0~1） |
| noiseAmp | number | 0.15 | 空间噪声幅度，越大边缘越有机 |
| angleOffset | number | 45 | 起始角度偏移（度），控制第一个区域落在哪条边 |
| seed | number | 0 | 随机种子，0=每次随机 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| baseGrid | grid | 所有区域合并平台掩码（1=平台，0=空地） |
| regionGrid | grid | 各区域独立 ID（1-based），空地为 0 |
| zoneCenters | array | 各区域质心坐标 [{x, y, id}] |

## 注意事项

1. **maxDepth**：设置过小区域会很浅，设置过大可能让区域延伸到地图中央
2. **gapWidth**：若区域互相粘连，增大此值；若区域太小，减小此值
3. **angleOffset**：调整此值可以让区域落在角落处（45°）或边中段（0°）
