# RTS基地泊松分布生成 (rts_base_shape_poisson)

用泊松盘采样在 mask 内均匀分布锚点，再以多子种子竞争 BFS 膨胀生成有机基地 blob。

## 功能特点

1. **泊松盘锚点分布**：使用 Bridson 快速泊松盘算法，保证所有锚点彼此间距 ≥ `minDist`，分布均匀无扎堆，适合需要散布在整个区域的场景
2. **多叶有机 blob**：每个锚点衍生 `subSeeds` 个子种子并融合膨胀，形成多叶圆润的有机形态
3. **区域间斥力**：不同区域通过 `gapWidth` 斥力保持间隙，不会融合
4. **全 mask 约束**：所有锚点、子种子和膨胀均严格限定在输入 mask 的非零区域内
5. **自动后处理**：每个区域保留最大连通块，并清除 < 8 格的碎片

## 与 rts_base_shape_gen 的区别

| 特性 | rts_base_shape_gen | rts_base_shape_poisson |
|------|--------------------|-----------------------|
| 锚点放置策略 | L 形分区（角落、上边、左边） | 全 mask 泊松盘均匀采样 |
| 锚点间距保证 | 无（依赖分区隔离） | ✅ 保证最小间距 `minDist` |
| 区域数量控制 | `numSeeds` 固定 | `maxRegions` 上限，实际数量由 mask 面积决定 |
| 适用场景 | RTS 角落基地（SC2 风格） | 散布全图的独立据点、战略要地 |

## 适用情况

- 需要在整个地图内均匀分布多个基地/据点
- 开放地形中随机生成独立平台区域
- 需要精确控制各区域最小间距，避免区域扎堆
- 非对称地图中的有机地形生成

## 基本使用方法

1. 将输入 `grid` 连接到可用区域掩码（非零为可生长区域）
2. 调整 `maxRegions` 设置期望的区域数量上限
3. 调整 `minDist` 控制区域之间的最小间距（越大分布越稀疏）
4. 调整 `maxRadius` 和 `subSeeds` 控制每个 blob 的大小和形态
5. 输出 `baseGrid` 可直接作为平台掩码使用，`regionGrid` 可用于区分不同区域

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grid | grid | 必填 | 输入掩码，非零格子为可生长范围 |
| maxRegions | number | 6 | 泊松盘最多放置的锚点数，实际数量受 minDist 和 mask 面积限制 |
| minDist | number | 0 | 锚点最小间距（格），0 = 自动计算为 min(宽,高)×0.25 |
| subSeeds | number | 4 | 每区域子种子数，越多叶瓣越丰富（建议 3~6） |
| maxRadius | number | 16 | 每个子种子最大膨胀半径（格），建议 12~22 |
| radiusVariance | number | 0.25 | 子种子半径随机差异比例（0~0.8） |
| gapWidth | number | 3 | 不同区域间最小间隙（格），建议 2~5 |
| growProb | number | 0.88 | 基础生长概率（0~1） |
| noiseAmp | number | 0.12 | 空间噪声强度，越大边缘越凹凸有机（0~0.5） |
| subSpacing | number | 6 | 同区域子种子间距（格），控制叶瓣分散程度（建议 4~10） |
| seed | number | 0 | 随机种子，0=每次使用时间戳（结果不同） |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| baseGrid | grid | 所有区域合并的平台掩码：1=平台，0=空地 |
| regionGrid | grid | 各独立区域的 1-based ID，空地为 0 |

## 注意事项

1. **实际区域数可能少于 maxRegions**：当 mask 面积不足以容纳 `maxRegions` 个间距为 `minDist` 的锚点时，实际采样到的锚点会更少
2. **minDist 与 maxRadius 的关系**：`minDist` 应大于 `maxRadius`，否则相邻区域膨胀后会因斥力产生大量空白，建议 `minDist ≈ maxRadius * 2`
3. **小 mask 场景**：mask 面积较小时适当减小 `minDist` 或 `maxRegions`，否则可能只采样到 1~2 个锚点
4. **与四重对称配合**：此电池设计为在单象限掩码内使用，再由 `rts_quad_symmetry` 进行四重对称；或直接在全图掩码内使用生成非对称地图
