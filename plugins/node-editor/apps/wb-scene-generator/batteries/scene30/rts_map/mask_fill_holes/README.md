# 掩码空洞填充 (mask_fill_holes)

对二值掩码中的内部空洞进行填充，消除基地 blob 内部因随机生长产生的零散空缺点。

## 功能特点

1. **边界 BFS 标记**：从地图四条边出发，BFS 遍历所有与外部连通的空地（0 格），将其标记为"外部"
2. **自动填洞**：剩余未被标记的 0 格——即被非零格（平台）完全包围的区域——统一置为 1，彻底填实空洞
3. **无大小限制**：填充所有内部空洞，无论空洞面积大小（不像 `grid_mask_clean` 受 `minHoleSize` 限制）
4. **原值保留**：非零格（>0）的原始数值完整保留，不会被覆盖

## 适用情况

- `rts_base_shape_gen` 的 `baseGrid` 输出中，基地 blob 内部存在随机生长留下的空缺点
- `rts_quad_symmetry.fullGrid` 四重对称后仍残留的内部孔洞
- 任何需要将二值掩码"实心化"的场景（平台内部无空洞）

## 基本使用方法

```
[rts_base_shape_gen] / [rts_quad_symmetry]
    └── baseGrid / fullGrid  ──►  [mask_fill_holes].grid

[mask_fill_holes]
    └── filledGrid  ──►  [rts_road_gen].baseGrid
                          或其他下游节点
```

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grid | grid | — | 待填充的二值掩码网格（1=平台，0=空地） |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| filledGrid | grid | 内部空洞已填充的二值网格，外部空地不受影响 |

## 注意事项

1. **不影响外部空地**：只填充被平台完全包围的内部 0 格；外部空地（与地图边界连通）保持不变
2. **多值掩码兼容**：若输入网格含有 >1 的多种值（如 regionGrid），所有非零值均视为"平台"参与填洞判断，但原始非零值不被改写
3. **边界格子**：地图四条边上的 0 格一定被视为外部，不会被填充
