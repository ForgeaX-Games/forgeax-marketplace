# 家具组位置放置器 (furniture_group_position_stamp)

将家具组 mask 按语义位置（0-8）自动计算锚点并盖印到室内空间中，mask 非零值直接对应组内子组件的相对编号，输出编号整体平移到旧列表之后。

## 位置定义

```
5 ─── 1 ─── 6
│           │
4     0     2
│           │
8 ─── 3 ─── 7
```

| 编号 | 名称 | 行对齐 | 列对齐 |
|------|------|--------|--------|
| 0 | 居中 | 家具本体中心 对齐 室内中心行 | 家具本体中心 对齐 室内中心列 |
| 1 | 贴上中 | 家具本体最上行 贴上墙 | 家具本体中心 对齐 室内中心列 |
| 2 | 贴右中 | 家具本体中心 对齐 室内中心行 | 家具本体最右列 贴右墙 |
| 3 | 贴下中 | 家具本体最下行 贴下墙 | 家具本体中心 对齐 室内中心列 |
| 4 | 贴左中 | 家具本体中心 对齐 室内中心行 | 家具本体最左列 贴左墙 |
| 5 | 左上角 | 家具本体最上行 贴上墙 | 家具本体最左列 贴左墙 |
| 6 | 右上角 | 家具本体最上行 贴上墙 | 家具本体最右列 贴右墙 |
| 7 | 右下角 | 家具本体最下行 贴下墙 | 家具本体最右列 贴右墙 |
| 8 | 左下角 | 家具本体最下行 贴下墙 | 家具本体最左列 贴左墙 |

## 编号规则

```
n = oldFurnitureIndex 最大 rank

mask[r][c] = v（v >= 1）→ maskA 写入 n + v
groupIndex 中 {rank: r, name: "书桌_桌"} → 输出 {rank: n + r, name: "书桌_桌"}
```

## 与 furniture_position_stamp 的区别

| 对比项 | furniture_position_stamp | furniture_group_position_stamp |
|--------|--------------------------|-------------------------------|
| 家具描述输入 | 单个家具名称（string） | 家具组编号列表（array） |
| mask 非零值含义 | 1=主体，>1=子组件+10 | 直接作为组内相对编号 |
| 输出编号 | n+1，子组件 n+11 | n + mask值，严格一一对应 |

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| furnitureMask | grid | — | 家具组 mask；非零值=组内子组件编号，0=过道 |
| groupIndex | array | — | 家具组编号列表，每项含 rank（与 mask 值对应）、name、isGroup |
| position | number | 0 | 位置编号 0-8，见上方表格 |
| roomGrid | grid | — | 室内空间网格（1=可用，0=墙） |
| maskA | grid | — | 家具实体占用网格 |
| maskB | grid | — | 过道预留网格 |
| oldFurnitureIndex | array | — | 旧家具编号列表，无时传 `[]` |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| newMaskA | grid | 写入家具组后的实体占用网格，值 = n + mask值 |
| newMaskB | grid | 写入过道后的过道预留网格 |
| furnitureIndex | array | 所有编号列表（旧+新），新条目 rank = n + 组内相对 rank |
| placementFailed | bool | 碰撞检测不通过时为 true |
| failReason | string | 失败原因描述，成功时为空字符串 |

## 注意事项

1. **groupIndex 的 rank 必须与 mask 非零值一一对应**
2. **位置仅确定初始锚点，不保证合法**：碰撞失败时原样返回
3. **与其他放置器串联**：输出可直接作为下一个放置器的输入
