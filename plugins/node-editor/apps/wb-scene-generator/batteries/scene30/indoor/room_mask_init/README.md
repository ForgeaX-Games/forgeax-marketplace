# 房间掩码初始化 (room_mask_init)

根据室内空间网格和门位置网格，初始化家具布局流程所需的两张掩码：maskA（家具实体占用）和 maskB（门口过道预留）。

## 功能特点

1. **maskA 全零初始化**：输出一张与输入网格同尺寸的全零网格，作为家具实体占用层的起点；后续家具放置算法将向其中写入 rank 编号，当前仅为1，将来支持多值。
2. **maskB 门口预留**：自动识别门格的四个相邻格（上下左右），将其在 maskB 中标为1，确保家具放置时不会堵塞门口通道。
3. **尺寸校验**：严格检查两张输入网格的行列数是否一致，不一致时返回错误信息。

## 适用情况

- 家具自动布局流程的第一步：在运行 `furniture_placer` 之前初始化房间状态
- 需要将室内空间网格和门位置网格分离管理的场景（对应原始 `layout_mask` 中1和2的拆分）

## 基本使用方法

1. 将**室内空间网格**（房间可用格为1，墙/外部为0）连接到 `roomGrid` 端口
2. 将**门位置网格**（门格为1，其余为0）连接到 `doorGrid` 端口；两张网格尺寸必须相同
3. 执行节点，得到初始化完成的 `maskA`（全零）和 `maskB`（门口已预留）
4. 将 `maskA` 和 `maskB` 传入后续的家具放置节点

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| roomGrid | grid | — | 室内空间网格，1=可用格，0=墙/其他 |
| doorGrid | grid | — | 门位置网格，1=门格，0=其他；尺寸须与 roomGrid 完全相同 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| maskA | grid | 家具实体占用掩码，初始全零；家具放置后写入 rank 编号（现为1，将来为多值） |
| maskB | grid | 过道预留掩码，门格四邻格标为1；始终为0/1二值网格 |

## 注意事项

1. **网格尺寸必须一致**：`roomGrid` 和 `doorGrid` 的行列数必须完全相同，否则节点返回错误。
2. **maskB 写入规则**：门格自身不写入 maskB，只将其上下左右四个相邻格标为1（与 `init_result.py` 行为完全一致）；若相邻格超出网格边界则忽略。
3. **maskA 的未来扩展**：maskA 当前全零，后续家具放置算法会向其中写入家具 rank 编号，设计上已预留为多值网格（而非仅0/1）。
4. **maskB 始终二值**：maskB 仅表示通道占用，永远只有0和1，不会扩展为多值。
