# 家具列表解析 (furniture_list_split)

将 LLM 电池输出的原始 JSON 字符串解析为家具对象数组，每个元素为一条家具信息。

## 功能特点

1. **自动去除 Markdown 包裹**：兼容 LLM 返回带有 ` ```json ` 前缀的情况，自动剥离后再解析
2. **直接输出数组**：将 `furniture_list` 字段中的每个家具对象原样输出为数组元素

## 适用情况

- LLM 电池输出家具 JSON 后，需要将家具列表传入后续处理节点时

## 基本使用方法

将 LLM 电池的 `result` 输出端口连接到本电池的 `result` 输入端口，执行后从 `list` 输出端口获取家具对象数组。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| result | string | — | LLM 电池输出的原始 JSON 字符串，必须包含 `furniture_list` 数组字段 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| list | array | `furniture_list` 中每个家具对象组成的数组 |

## 注意事项

1. **输入格式**：`result` 必须是合法 JSON 字符串，且顶层包含 `furniture_list` 数组，否则返回 `error` 字段。
2. **Markdown 兼容**：若 LLM 返回内容以 ` ```json ` 开头，会自动剥离后再解析。
