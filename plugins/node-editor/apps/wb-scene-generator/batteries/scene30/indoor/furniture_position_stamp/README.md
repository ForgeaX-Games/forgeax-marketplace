# 家具位置放置器 (furniture_position_stamp)

将家具 mask 按语义位置（0-8）自动计算锚点并盖印到室内空间中，无需手动输入坐标。

## 位置定义

位置基于**室内可用区包围盒**（roomGrid 中所有值为 1 的格子的最小外接矩形）计算。

```
5 ─── 1 ─── 6
│           │
4     0     2
│           │
8 ─── 3 ─── 7
```

| 编号 | 名称 | 行对齐 | 列对齐 |
|------|------|--------|--------|
| 0 | 居中 | 家具本体中心 对齐 室内中心行 | 家具本体中心 对齐 室内中心列 |
| 1 | 贴上中 | 家具本体最上行 贴上墙（minR） | 家具本体中心 对齐 室内中心列 |
| 2 | 贴右中 | 家具本体中心 对齐 室内中心行 | 家具本体最右列 贴右墙（maxC） |
| 3 | 贴下中 | 家具本体最下行 贴下墙（maxR） | 家具本体中心 对齐 室内中心列 |
| 4 | 贴左中 | 家具本体中心 对齐 室内中心行 | 家具本体最左列 贴左墙（minC） |
| 5 | 左上角 | 家具本体最上行 贴上墙 | 家具本体最左列 贴左墙 |
| 6 | 右上角 | 家具本体最上行 贴上墙 | 家具本体最右列 贴右墙 |
| 7 | 右下角 | 家具本体最下行 贴下墙 | 家具本体最右列 贴右墙 |
| 8 | 左下角 | 家具本体最下行 贴下墙 | 家具本体最左列 贴左墙 |

**中心取整**：行列中心坐标若为非整数，统一向下取整（`Math.floor`）。

**"本体"的定义**：mask 中值 >= 1 的格子（不含过道的 0 格）。家具 mask 周边的 0 圈不影响对齐计算。

## 功能特点

1. **语义化位置**：用 0-8 九个编号描述所有常用放置位置，无需手算坐标
2. **碰撞检测**：放置前检查本体格和过道格是否与已有家具冲突，失败时原样返回
3. **自动编号**：新家具 rank = 旧列表最大 rank + 1，isGroup 固定为 true

## 适用情况

- 配合 `grid_furniture_gen` 生成 mask 后，按位置快速放置
- 需要对齐到房间某个方位的固定家具（如上墙书柜、中央地毯等）

## 基本使用方法

1. 用 `grid_furniture_gen` 或其他方式准备家具 mask
2. 连接 roomGrid、maskA、maskB
3. 连接 oldFurnitureIndex（无旧家具时传 `[]`）
4. 设置 position（0-8）和 furnitureName
5. 执行，得到更新后的网格和家具清单

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| furnitureMask | grid | — | 家具 mask 网格；1=主体，>1=子组件，0=过道 |
| furnitureName | string | 自定义家具 | 家具名称，写入家具清单 |
| position | number | 0 | 位置编号 0-8，见上方表格 |
| roomGrid | grid | — | 室内空间网格（1=可用，0=墙） |
| maskA | grid | — | 家具实体占用网格 |
| maskB | grid | — | 过道预留网格 |
| oldFurnitureIndex | array | — | 旧家具编号列表，无时传 `[]` |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| newMaskA | grid | 写入家具后的实体占用网格 |
| newMaskB | grid | 写入过道后的过道预留网格 |
| furnitureIndex | array | 所有家具编号列表（旧+新），每项含 rank、name、isGroup |
| placementFailed | bool | 碰撞检测不通过时为 true，输出为原始网格和清单 |
| failReason | string | 失败原因描述，成功时为空字符串 |

## 注意事项

1. **isGroup 固定为 true**：所有通过本电池放置的家具均标记为组合家具类型
2. **位置仅确定初始锚点，不保证合法**：若目标位置已有家具，碰撞检测会失败并原样返回
3. **与其他放置器串联**：输出 maskA/maskB 可作为下一个放置器的输入，支持链式放置
