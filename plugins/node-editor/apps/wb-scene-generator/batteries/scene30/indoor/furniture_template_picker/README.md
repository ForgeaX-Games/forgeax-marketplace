# 家具模板选择器 (furniture_template_picker)

按编号（1–15）选取预生成的通用游戏空间家具模板，输出可直接接入家具推理流程的 JSON 数据。

## 支持的空间类型

| 编号 | 房间 | 尺寸 | | 编号 | 房间 | 尺寸 |
|------|------|------|-|------|------|------|
| 1 | 卧室 | 10×10 | | 9 | 牢房 | 6×6 |
| 2 | 书房 | 10×10 | | 10 | 祭坛室 | 12×12 |
| 3 | 厨房 | 10×8 | | 11 | 炼金室 | 8×8 |
| 4 | 餐厅 | 12×10 | | 12 | 商店 | 12×10 |
| 5 | 酒馆大厅 | 16×14 | | 13 | 武器库 | 12×10 |
| 6 | 守卫室 | 8×8 | | 14 | 议事厅 | 14×12 |
| 7 | 宝库 | 10×10 | | 15 | 储藏室 | 8×8 |
| 8 | 铁匠铺 | 10×10 | | | | |

## 功能特点

1. **零 AI 调用**：直接输出预生成模板，无需每次调用大语言模型
2. **双路输出**：`result` 可替代 AI 文本推理直接接 `furniture_list_split`；`room`/`room_size` 可接提示词模板驱动 AI 推理
3. **简单数字选择**：`index` 端口设置 1-15 即可切换空间类型

## 接入方式

### 方式一：绕过 AI，直接用预生成模板

```
[家具模板选择器] → result → [家具推理结果转列表] → list → [家具拆分] → ...
```

### 方式二：为 AI 推理提供房间信息

```
[家具模板选择器] → room      → [提示词模板填充] → prompt → [AI 文本推理]
                → room_size →
```

## 输入参数

| 参数名 | 类型   | 默认值 | 说明 |
|--------|--------|--------|------|
| index  | number | 1      | 房间编号，1-15，见上方空间类型表 |

## 输出参数

| 参数名    | 类型   | 说明 |
|-----------|--------|------|
| result    | string | 完整家具模板 JSON 字符串，与 AI 推理输出格式一致 |
| room      | string | 房间中文名称 |
| room_size | string | 房间尺寸，如 `10x10` |

## 注意事项

1. `result` 输出格式与 `AI 文本推理` 节点的 `result` 端口完全兼容，可直接接 `furniture_list_split`
2. 模板数据内嵌于电池，不依赖外部文件，热重载后立即生效
