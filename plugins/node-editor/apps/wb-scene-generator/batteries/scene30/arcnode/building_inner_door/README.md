# 建筑内墙开门 (building_inner_door)

在建筑内墙上随机开 2~4 格宽的门洞，使用最小生成树算法保证每个房间都可以互相到达。

## 功能特点

1. **全房间连通保证**：使用 Kruskal 最小生成树算法，确保开完门后所有房间形成连通图，不存在孤立房间
2. **只在内墙开门**：识别并区分外墙与内墙，只对内墙（两侧均为建筑内部房间的墙段）开门
3. **随机门洞位置**：每段内墙上，门洞位置在中间区域随机选取，两端各保留至少 1 格墙柱
4. **随机门洞宽度**：门宽在 2~4 格之间随机选取（受墙段长度限制）
5. **可复现结果**：通过随机种子控制，相同种子产生相同结果

## 适用情况

- 配合 `building_inner_wall` 电池使用：先用内墙生成房间，再用此电池开门
- 需要全连通建筑平面图的场景（RPG 室内地图、建筑平面生成等）
- 不适用于外墙开门（外墙开门请使用 `building_door` 电池）

## 基本使用方法

1. 将 `building_inner_wall` 的输出网格与建筑外轮廓网格合并（或直接使用同时包含外墙+内墙的网格）
2. 将合并后的网格连接到本电池的 `inputGrid` 端口
3. 设置随机种子（可选），运行即得到开好门洞的网格

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| inputGrid | grid | - | 建筑轮廓及内墙掩码，非零值为墙体（需同时包含外墙和内墙） |
| seed | number | 0 | 随机种子，0 表示使用当前时间戳 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGrid | grid | 与输入等尺寸，内墙上开好 2~4 格门洞，门洞处值为 0 |

## 算法说明

### 房间识别

电池对输入网格中所有值为 `0` 的格子做 BFS 连通分量标记，每个连通区域视为一个独立的「房间」（建筑外部也会被识别为一个区域，但通过「室内判断」过滤掉）。

### 内墙识别

对于每个非零墙格，检查其上下（水平墙）或左右（垂直墙）两侧：
- 两侧都是值为 `0` 的空地
- 两侧的空地属于不同连通分量（即分隔了两个不同房间）
- 两侧空地的邻格中至少有一个墙格（说明它们是被围合的室内空间，而非建筑外部）

满足以上条件的连续墙格组成一段「内墙段」，是开门的候选。

### 最小生成树保证连通

用 Kruskal 算法：打乱所有内墙段，依次尝试——如果一段墙两侧的房间还未连通，则在该墙段上开门并合并两个房间。这样开的门数最少，且保证所有房间全部连通。

## 注意事项

1. **输入须包含外墙**：内墙识别依赖「室内判断」，需要外墙存在才能正确区分室内/室外空地
2. **墙段至少需要 4 格**：开一个 2 格门洞并两端留 1 格墙柱，最短墙段需要 4 格；更短的墙段会尽力开最小门洞
3. **只保证最少连通**：算法开门数量是连通所有房间的最小值（房间数-1），若需要更多门（增加路径丰富度），可考虑多次运行或后处理
4. **与 building_inner_wall 搭配**：建议工作流为：`building_profile` → `building_inner_wall` → 网格合并 → `building_inner_door`
