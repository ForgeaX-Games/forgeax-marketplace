# 建筑开门 (building_door)

在墙体网格中随机打开指定数量和宽度的门洞，门洞仅生成在有效墙段（两侧具有贯通空间）上。

## 功能特点

1. **有效性检测**：只在真正的"通透墙段"上开门——门两侧至少有一侧是空格，确保门洞连通两个区域而非悬空在实体内部。
2. **无重叠保证**：多个门洞不会在物理位置上重叠，各自独立。
3. **可控宽度**：支持任意正整数门宽，沿墙方向连续清除指定数量的格子。
4. **可复现随机**：提供随机种子参数，相同种子输出相同结果，便于调试和复现。

## 适用情况

- 为 `building_inner_wall` 等生成的内墙网格添加门洞，打通相邻房间
- 为建筑外墙添加出入口
- 不适用于需要精确控制门位置的场景（该电池为随机放置）

## 基本使用方法

1. 将墙体掩码网格（如建筑内墙生成的 `outputGrid`）连接到 `wallGrid` 输入
2. 设置 `doorCount`（门的数量）和 `doorWidth`（门宽格子数）
3. 设置 `seed`（固定随机种子可复现结果，0 则每次随机）
4. `outputGrid` 输出门洞已打开的网格，可直接与原始建筑占地网格合并使用

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| wallGrid | grid | — | 墙体掩码网格，非零值视为墙体 |
| doorCount | number | 4 | 要打开的门洞总数 |
| doorWidth | number | 2 | 每个门洞沿墙方向的宽度（格子数），最小为 1 |
| seed | number | 0 | 随机种子，0 使用当前时间戳 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGrid | grid | 与输入等尺寸，门洞处=0，其余保持原值 |

## 注意事项

1. **候选不足**：若墙体网格中符合条件的有效墙段数量少于 `doorCount`，实际生成的门数会少于请求数，不会报错。
2. **门宽与墙体关系**：`doorWidth` 应小于单段墙体的长度，否则整段墙都会被清除；建议门宽不超过墙段长度的一半。
3. **输入为内墙时**：`building_inner_wall` 输出的内墙格子值为 1，直接连接即可；若输入的墙体值不为 1 也能正常处理（只要非零即视为墙）。
4. **输出不含建筑占地信息**：该电池只修改墙体格子，输出网格不包含建筑底面，如需合并请配合掩码合并电池使用。
