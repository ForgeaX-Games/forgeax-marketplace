# 建筑开窗 (building_window)

在墙体网格中随机或均匀打开指定数量和宽度的窗洞。

## 功能特点

1. **双侧校验**：窗洞必须开在两侧均有空格的墙体上，确保窗口连通内外或相邻房间
2. **随机 / 均匀两种模式**：`randomEnable=true` 随机分布，`false` 按位置均匀间隔分配
3. **可复现**：提供固定随机种子可得到完全一致的窗洞位置
4. **不重叠保证**：同一位置不会被多个窗洞重复清零

## 适用情况

- 建筑平面图自动化开窗，生成带窗户的房间轮廓
- 与 `building_door` 配合，先开门再开窗
- 需要均匀窗户分布的场景（关闭随机模式）

## 基本使用方法

1. 将 `building_inner_wall` 或其他墙体电池输出的墙体网格连接到 `wallGrid`
2. 设置 `windowCount`（窗数）和 `windowWidth`（窗宽）
3. `randomEnable=true` 时设置 `seed` 控制随机结果；`randomEnable=false` 时忽略种子
4. `outputGrid` 中窗洞位置值为 0，其余保持原墙体值

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| wallGrid | grid | — | 墙体掩码网格，非零值视为墙体 |
| windowCount | number | 4 | 要打开的窗洞总数 |
| windowWidth | number | 1 | 每个窗洞沿墙方向的宽度（格子数），最小为 1 |
| randomEnable | bool | true | true=随机选位置；false=均匀间距分配 |
| seed | number | 0 | 随机种子，0 使用当前时间戳 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGrid | grid | 与输入等尺寸，窗洞处=0，其余保持原值 |

## 注意事项

1. **窗 vs 门的区别**：窗洞要求所在墙段**两侧**均有空格（内外都通），门只需一侧；若候选位置不足，实际开窗数量可能小于 `windowCount`
2. **窗宽过大**：`windowWidth` 大于墙段长度时该段不会被列为候选，请根据墙体厚度合理设置
3. **均匀模式下种子无效**：`randomEnable=false` 时结果完全由候选集位置决定，`seed` 不起作用
4. **不修改原始网格**：电池内部深拷贝输入，`wallGrid` 不会被修改
