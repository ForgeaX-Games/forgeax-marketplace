# 洞穴连通修复 (cave_connect)

对元胞自动机生成的洞穴网格进行连通性修复，确保所有孤立洞穴空间都与主洞穴相通。

## 功能特点

1. **自动识别孤立区域**：BFS 标记所有"洞穴空间"（值=2）的连通区域，找出孤立的小空间
2. **最短路径凿通**：从最大洞穴（主洞穴）出发，BFS 穿越墙壁找到最近的孤立区域，沿最短路径凿通隧道
3. **可调隧道宽度**：通过 `tunnelRadius` 控制隧道粗细，圆形膨胀让隧道边缘更自然
4. **小区域清理**：`minRegionSize` 可将过小的孤立区域直接填墙删除，避免在噪点区域凿隧道
5. **迭代直到全通**：每次连通一个区域后重新标记，循环直到全图只剩一个洞穴连通分量

## 适用情况

- `cellular_automata` 生成的洞穴地形中存在多个孤立的封闭空间
- Warcraft / 地下城 / Roguelike 类场景，要求所有房间/洞穴可达
- 任何需要保证洞穴全连通的地图后处理步骤

## 基本使用方法

```
[cellular_automata]
    └── caveGrids[i]  ──►  [cave_connect].caveGrid

[cave_connect]
    └── connectedGrid  ──►  下游渲染/导出节点
```

> `caveGrids` 是数组，需先用拆分节点取出单张网格，再接入本电池。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| caveGrid | grid | — | 来自 cellular_automata 的单张洞穴网格（0=蒙版外，1=墙，2=洞穴空间） |
| tunnelRadius | number | 1 | 凿通隧道时的圆形膨胀半径：0=单格，1=约3格宽，2=约5格宽 |
| minRegionSize | number | 0 | 小于此面积的孤立区域直接填墙；0=连通所有区域 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| connectedGrid | grid | 全连通洞穴网格，值编码与输入相同（0/1/2） |

## 算法原理

```
while 存在多个洞穴连通区域:
    1. 标记所有区域，按面积降序排列
    2. 面积 < minRegionSize 的区域 → 填墙，重新循环
    3. 从最大区域（主洞穴）所有格子同时出发 BFS
       穿越墙格（1），跳过蒙版外格（0）
    4. BFS 第一次碰到孤立洞穴格（2） → 记录路径
    5. 沿路径回溯，每格执行圆形膨胀（半径=tunnelRadius）
    6. 重新标记，继续下一轮
```

## 注意事项

1. **输入为单张网格**：`cellular_automata.caveGrids` 输出列表，需先取出单张再接入本电池
2. **蒙版外区域不穿越**：隧道只在蒙版内（值≠0）凿通，不会跑出蒙版边界
3. **隧道宽度建议**：`tunnelRadius=1`（约3格宽）通常视觉效果最自然；设为 0 会生成单格宽的细隧道
4. **性能**：大地图（256×256+）含大量孤立区域时每轮 BFS 较耗时，建议配合 `minRegionSize` 提前清理噪点
