# 精准选点 (point_select)

从源网格中精准提取指定坐标处的点，输出仅保留这些坐标的掩码值，其余格子为 0。

## 功能特点

1. **精准坐标选取**：通过显式列出 `[row, col]` 坐标，精确控制要保留的点位置。
2. **掩码值保留**：输出格子的值直接来自输入网格在该坐标处的值，不做任何变换。
3. **多点支持**：一次可传入任意数量的坐标，批量提取。
4. **越界安全**：超出网格范围的坐标会被静默忽略，不抛出错误。

## 适用情况

- 需要从一张掩码图中提取特定位置的点，用于后续处理（如点生矩形、点生圆）。
- 将外部逻辑计算出的坐标列表转换为网格中的点集。
- 与其他算法配合，先选点再扩展区域。

## 基本使用方法

1. 将源网格连接到「输入网格」端口。
2. 将坐标列表连接到「点列表」端口，格式为 `[[row, col], ...]`。
3. 执行后，「输出网格」中仅指定坐标处有值，其余为 0。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grid | grid | — | 源网格，提供各坐标处的掩码值 |
| points | array | — | 坐标列表，格式为 `[[row, col], ...]` |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGrid | grid | 与输入网格同尺寸，仅指定坐标处保留原掩码值，其余格子为 0 |

## 坐标格式说明

`points` 端口接受以下两种格式：

**JSON 数组（推荐，适合上游电池直接连线）：**
```json
[[1, 2], [3, 4], [5, 6]]
```

**字符串化 JSON（适合文本面板输入）：**
```
"[[1,2],[3,4],[5,6]]"
```

坐标为 `[row, col]`（行优先），行列均从 0 开始计数。

## 注意事项

1. **坐标为行列顺序**：`[row, col]`，即先行后列，与数学 `(x, y)` 方向相反。
2. **越界坐标忽略**：若坐标超出网格尺寸，该点会被静默跳过，不影响其他点的提取。
3. **源网格为零的坐标**：若源网格在指定坐标处的值本身为 0，该点在输出中也为 0，与未选取无区别。
4. **输出不修改源网格**：输出是全新的网格，输入网格不会被修改。
