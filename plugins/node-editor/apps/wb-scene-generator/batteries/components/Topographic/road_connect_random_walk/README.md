# 道路连接随机游走 (road_connect_random_walk)

从 POI 网格中提取指定值的格子作为连接点，用 Prim 最小生成树 + A* 随机游走方式全部连通，自动绕开障碍物网格。

## 功能特点

1. **网格驱动点位**：输入 POI 网格和一个填充值，自动提取所有匹配格子的坐标作为连接点，无需手动传坐标列表。
2. **全连通保证**：使用 Prim 最小生成树，以最少路线数（N-1条）连通所有 N 个点，不产生冗余回路。
3. **智能绕障**：每段路线用 A\* 寻路（曼哈顿启发），自动绕开障碍物，起终点本身始终可达。
4. **横平竖直**：只允许上下左右移动，生成参考图风格的田园式正交道路网。
5. **可调路宽**：支持通过 `roadWidth` 参数膨胀路径，生成 1~N 格宽的道路。

## 适用情况

- 田园/城镇场景中，将点位网格（如泊松采样结果、建筑中心点）连通成路网
- 需要绕开湖泊、山体、建筑等障碍物的道路规划
- 配合 `泊松采样` 或其他点位生成电池使用

## 基本使用方法

1. 准备 POI 网格（如用泊松采样生成，目标点格子值为某个固定值）。
2. 设置 `点位填充值` 为 POI 网格中点的值。
3. 如有障碍物，将障碍物 grid 接入 `障碍物网格`。
4. 调整 `道路宽度`（推荐 2~3），`道路值` 可设置为对应层标识。
5. 输出 `道路网格` 可直接叠加到地形 grid 上。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| poiGrid | grid | — | POI 网格，提取值=poiValue 的格子作为连接点 |
| poiValue | number | 1 | 要提取的 POI 格子值 |
| obstacleGrid | grid | — | 障碍物网格，非零格不可通行（可选） |
| roadWidth | number | 1 | 道路宽度（格），通过膨胀实现 |
| roadValue | number | 1 | 道路写入的数值 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGrid | grid | 道路网格，道路格=roadValue，其余=0 |

## 注意事项

1. **坐标系**：格子列索引对应 x，行索引对应 y。
2. **障碍物优先级**：A\* 不会穿过障碍物，但若两点之间完全被障碍隔断，该段路线会被跳过（不报错）。
3. **路宽膨胀不覆盖障碍**：膨胀时跳过障碍物格，保证障碍完整性。
4. **POI 点数量**：N 个点需要 N-1 次 A\* 搜索，大地图且点数较多时可能略慢。
