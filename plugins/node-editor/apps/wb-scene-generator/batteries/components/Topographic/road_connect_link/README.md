# 道路连接连连看 (road_connect_link)

从 POI 网格中提取连接点，用连连看式折线路径（最多 2 次转弯）全部连通，在障碍物间寻找干净的正交折线通道。

## 功能特点

1. **连连看路径**：两点之间优先寻找最多 `maxTurns` 次转弯的干净折线（直线→L形→Z/S/U形），路线整洁不随机。
2. **A\* 兜底**：当连连看路径因障碍物过密而找不到时，自动降级为 A\* 寻路，保证连通性。
3. **全连通保证**：使用 Prim 最小生成树，以最少路线数（N-1条）连通所有 N 个点。
4. **横平竖直**：只允许上下左右移动，生成正交风格的道路网。
5. **可调路宽**：支持通过 `roadWidth` 参数膨胀路径，生成 1~N 格宽的道路。

## 适用情况

- 需要路线整洁、转弯清晰的田园/城镇道路规划（如棋盘格小镇、农场）
- 障碍物较稀疏、点位之间有明确折线通道的场景
- 与「道路连接随机游走」对比使用时，连连看版本路线更规整，随机游走版本更自然

## 基本使用方法

1. 准备 POI 网格（目标点格子值为固定值，如 1）。
2. 设置 `点位填充值` 为 POI 网格中点的值。
3. 如有障碍物，将障碍物 grid 接入 `障碍物网格`。
4. 调整 `最大转弯数`（默认 2，即标准连连看规则）。
5. 调整 `道路宽度` 和 `道路值`。
6. 输出 `道路网格` 可直接叠加到地形 grid 上。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| poiGrid | grid | — | POI 网格，提取值=poiValue 的格子作为连接点 |
| poiValue | number | 1 | 要提取的 POI 格子值 |
| obstacleGrid | grid | — | 障碍物网格，非零格不可通行（可选） |
| roadWidth | number | 1 | 道路宽度（格），通过膨胀实现 |
| roadValue | number | 1 | 道路写入的数值 |
| maxTurns | number | 2 | 连连看路径最大转弯次数（0=直线，1=L形，2=Z/S形） |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGrid | grid | 道路网格，道路格=roadValue，其余=0 |

## 连连看路径说明

### 0 转弯（直线）
两点在同一行或同一列，且中间无障碍物时，直接连线。

### 1 转弯（L 形）
经过一个拐点，分两段直线连通。拐点有两个候选：
- `(r1, c2)`：先横向到目标列，再纵向到目标行
- `(r2, c1)`：先纵向到目标行，再横向到目标列

### 2 转弯（Z / S / U 形）
在地图范围内扫描所有可能的中继行或中继列，形成三段折线：
- 经中继行：竖→横→竖
- 经中继列：横→竖→横

### A\* 兜底
当以上所有连连看路径均被障碍物阻断时，自动使用 A\* 算法寻找任意可达路径，保证不断连。

## 注意事项

1. **整洁 vs 自然**：连连看路径比随机游走更规整，但在障碍密集区域可能回退到 A\*（路线风格会略有变化）。
2. **转弯数调节**：`maxTurns=0` 只连同行/同列的点对（其他点对全靠 A\*）；`maxTurns=1` 适合棋盘格场景；`maxTurns=2` 是最常用的标准连连看规则。
3. **路宽膨胀不覆盖障碍**：膨胀时跳过障碍物格，保证障碍完整性。
4. **POI 点数量**：N 个点需要 N-1 次路径搜索，大地图且点数较多时，2转弯扫描量为 O(rows+cols)，性能良好。
