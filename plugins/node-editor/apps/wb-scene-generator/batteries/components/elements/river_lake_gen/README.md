# 河流湖泊生成 (river_lake_gen)

在输入网格上程序化生成河流和湖泊系统，自动划分河岸/浅水/中水/深水四层深度带，并可在水域中随机放置自定义物品。

## 功能特点

1. **四种路径算法**：直线（straight）、蜿蜒（meandering）、分支（branching）、随机漫步（random），覆盖不同风格需求。
2. **可变河道宽度**：沿河道按正弦曲线变化（河口窄→中段宽→河口窄），最小/最大宽度可独立配置。
3. **四层深度分区**：BFS 距离场 + Perlin 噪声让河岸/浅水/中水/深水边界自然不规则，避免同心圆感。
4. **湖泊生成**：椭圆形基础叠加 Perlin 噪声扰动，产生有机形态湖泊。
5. **水中物品散布**：在浅水/中水区域随机放置自定义物品，各物品独占唯一掩码ID，保持间距不重叠。
6. **掩码 ID 自动顺延**：所有水域 ID 从输入网格最大值 +1 开始，不与已有掩码冲突。

## 适用情况

- 野外地图、大陆地图上生成河流水系
- 在已有地形网格上叠加水域层
- 需要河岸/浅水/深水分层渲染的场景（如 RPG 可涉水判断）

## 基本使用方法

1. 将上游地形电池的 `outputGrid` 接入 `输入网格`
2. 选择生成算法和河流宽度范围
3. 执行后 `水域网格` 即含完整水域掩码，`名称清单` 含所有 ID 映射

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grid | grid | — | 基准输入网格 |
| inputNameList | array | [] | 输入网格名称清单 [{id, name}]，与水域清单合并输出 |
| riverCount | number | 2 | 生成的河流条数 |
| algorithm | string | meandering | 路径算法：straight / meandering / branching / random |
| minWidth | number | 2 | 河道最小宽度（格） |
| maxWidth | number | 6 | 河道最大宽度（格，河流中段处） |
| lakeCount | number | 0 | 生成湖泊数量，0 = 不生成 |
| waterItems | array | [] | 水中物品名称列表，如 ["荷叶","浮萍"] |
| seed | number | 0 | 随机种子，0 = 每次不同 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| waterGrid | grid | 叠加水域掩码的输出网格 |
| nameList | array | [{id, name}] 清单，输入清单在前，水域名称在后 |

## 算法说明

| 算法 | 特点 |
|------|------|
| straight | 从起点直线插值到终点，适合运河/人工水道 |
| meandering | 正弦曲线偏移，产生自然蜿蜒感（默认推荐） |
| branching | 主路 + 随机支流，适合三角洲/复杂水系 |
| random | 70% 朝目标偏向漫步，形态最不规则 |

## ID 分配规则

设输入网格最大值为 `M`，则：

| 区域 | ID |
|------|-----|
| 河岸 | M+1 |
| 浅水 | M+2 |
| 中水 | M+3 |
| 深水 | M+4 |
| 物品0 | M+5 |
| 物品1 | M+6 |
| … | … |

## 注意事项

1. **河流穿越整个地图**：起点和终点随机选在网格边缘的对侧，无法指定具体位置；如需指定路径，请改用 `river_spline` 电池。
2. **branching 算法支流**：每条主流以 15% 概率产生支流，支流宽度略小于主流。
3. **湖泊位置**：尽量避开已有水域，但极端情况下可能与河流重叠。
4. **waterItems 格式**：传入字符串数组 `["荷叶","浮萍"]`，每种物品约占浅水/中水格子总数的 5%，格间距自动计算。
5. **性能**：超大网格（>512×512）+ 高 riverCount 时计算量上升，建议 riverCount ≤ 10。
