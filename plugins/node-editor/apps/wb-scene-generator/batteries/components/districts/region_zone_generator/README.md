# 区域分区生成器 (Region Zone Generator)

在基准掩码网格（或网格列表）上，根据区域列表（名称、面积权重、九宫格方位）生成若干不重叠的不规则区域，每个区域输出独立的单值网格。

## 功能特点

1. **方位驱动布局**：每个区域可指定 1-9 的九宫格方位，种子点将被定位到对应位置的最近可用像素
2. **面积加权 Voronoi**：面积权重大的区域自动获得更多像素，实现比例控制
3. **Lloyd 松弛**：通过迭代质心更新使区域分布更均匀，减少极端形状
4. **四种边界风格**：`organic`（自然有机）、`smooth`（更圆润）、`rectilinear`（矩形化）、`voronoi`（原始 Voronoi 直边）
5. **单值网格列表输出**：每个区域独立输出一张只含该区域 ID 和 0 的网格，便于下游独立处理
6. **支持网格列表输入**：`baseGrid` 可以是单个网格或网格列表，多个网格的可用区域取并集

## 适用情况

- 在已有地形或边界掩码上划分功能区块（森林、湖泊、城镇、荒野等）
- 需要按比例、按方位控制区域分布的场景
- 作为 bubble_layout 管线的轻量替代方案
- 搭配 `label_smoother`、`network_path_finder` 等电池进一步加工

## 基本使用方法

1. 连接上游电池（如 `bsp_rect_gen`、`random_rect_zone_gen`）提供基准掩码网格或网格列表
2. 在 `regions` 参数中填写区域描述（`[[名称, 面积权重, 方位], ...]` 格式）
3. 选择 `boundaryStyle` 风格
4. 执行，`outputGridList` 即为各分区独立网格列表

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| baseGrid | array | — | 基准掩码网格或网格列表，非零像素为可用区域 |
| regions | array | — | 区域描述数组，见下方格式 |
| boundaryStyle | string | "organic" | 边界风格：organic / smooth / rectilinear / voronoi |
| relaxIterations | number | 5 | Lloyd 松弛迭代次数（推荐 3-8） |
| smoothIterations | number | 5 | 边界平滑 CA 迭代次数（仅 organic/smooth 风格） |
| seed | number | 0 | 随机种子，0=每次不同 |

### regions 格式

```json
[
  ["森林", 3, 1],
  ["湖泊", 2, 5],
  ["城镇", 2, 9],
  ["荒野", 4, 3]
]
```

- 第 1 项：区域名称（字符串）
- 第 2 项：相对面积权重（正整数）；值越大，分配的像素越多
- 第 3 项：九宫格方位（1-9）
  ```
  1 | 2 | 3
  ---------
  4 | 5 | 6
  ---------
  7 | 8 | 9
  ```

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGridList | array | 单值网格列表，每个区域一张，只含该区域 ID 和 0，顺序与 regions 一致 |
| nameList | array | 名称清单 `[{id, name}]` |

## 边界风格说明

| 风格 | 特点 |
|------|------|
| organic | 细胞自动机平滑，阈值较高（4/8），自然有机感，推荐默认 |
| smooth | 更强平滑（3/8 阈值 + 双倍迭代），边界更圆润 |
| rectilinear | 曼哈顿距离 Voronoi，边界呈 45°/水平/垂直直线，类矩形风格 |
| voronoi | 不做后处理，保留原始 Voronoi 直线边界 |

## 使用示例

### 输入示例

```json
{
  "regions": [
    ["北方森林", 4, 2],
    ["中央城市", 3, 5],
    ["南部沙漠", 3, 8],
    ["东部山地", 2, 6]
  ],
  "boundaryStyle": "organic",
  "relaxIterations": 5,
  "smoothIterations": 6,
  "seed": 42
}
```

### 输出示例

```json
{
  "outputGridList": [
    [[0,0,2,2,...], ...],
    [[0,3,0,0,...], ...],
    [[0,0,0,4,...], ...],
    [[5,0,0,0,...], ...]
  ],
  "nameList": [
    { "id": 2, "name": "北方森林" },
    { "id": 3, "name": "中央城市" },
    { "id": 4, "name": "南部沙漠" },
    { "id": 5, "name": "东部山地" }
  ]
}
```
（假设基准网格最大 ID 为 1，新 ID 从 2 开始）

## 注意事项

1. **基准网格非零区域**：只有 baseGrid 值 ≠ 0 的像素才参与分区；值为 0 的像素（如边界/障碍）输出为 0
2. **网格列表输入**：传入多个基准网格时，可用区域取所有网格的并集，尺寸需保持一致
3. **区域数量限制**：建议区域数量 ≤ 20；数量过多时 Lloyd 松弛效率下降
4. **面积权重仅影响比例**：不保证精确面积，实际分配受掩码形状影响
5. **同方位冲突**：多个区域指定相同方位时，通过随机扰动区分种子点位置
6. **Lloyd 迭代与 seed**：相同 seed + 相同参数可复现结果；seed=0 时使用当前时间戳
