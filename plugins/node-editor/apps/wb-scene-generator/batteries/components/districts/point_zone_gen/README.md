# 点生区域 (point_zone_gen)

从给定点位按目标面积生长出形状自然的有机区域，每个区域输出独立单值网格；高度信息透传到名称清单的 `height` 字段，由渲染器自动作立面化处理。

## 功能特点

1. **点位定面积生长**：每个区域只需指定种子点 `(x, y)` 与目标面积 `area`，电池自动在网格内生长出恰好 `area` 个 4-连通格子的连通区域。
2. **有机轮廓**：以"欧氏距离 + FBM 噪声"作为优先级 BFS 的扩散权重，再叠加闭合高斯样条平滑（移植自「区域嵌套」），得到自然平滑且无尖刺的边缘。
3. **高度透传**：`height` 字段不影响平面网格内容，仅写入名称清单 `height`。当 `height > 0` 时渲染器自动将该值视作立面高度（cliff/elevation）渲染。
4. **ID 顺延**：起始 ID 取所有输入网格最大值 + 1，每个区域分配独立递增 ID，下游可直接合并不会冲突。
5. **极简接口**：仅暴露 `seed` 一个调参点，其余参数（侵蚀强度、样条算法、平滑度等）内置合理默认值，开箱即用。

## 适用情况

- 在已有底图上按 AI/规划脚本输出的"点位 + 面积 + 高度"列表生成多个分区（如村庄房址、城堡塔楼、山丘簇、岛屿群）。
- 需要每个区域有自然边界、不能用矩形或正圆描述的小型地块。
- 上层流水线已经计算好"应该在哪里有多大"的区域规划，希望直接落地为可视化网格。

不适用：

- 需要严格不重叠的紧密拼接分区 → 使用「区域分区生成器」（Voronoi 加权松弛）。
- 仅有现成掩码、希望对其做有机化嵌套侵蚀 → 使用「区域嵌套」。

## 基本使用方法

1. 上游接入一张底图网格（决定输出尺寸；若网格中已有 ID，新区域 ID 会从最大值 + 1 顺延）。
2. 把"区域定义"端口接入一个 JSON 字符串，每项格式 `[x, y, area, height]`。
3. 设置随机种子（0 = 每次随机），即可输出一张张单值区域网格 + 名称清单。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grids | array | — | 输入网格或网格列表，仅用于确定输出尺寸与起始 ID |
| regions | string | `[]` | 区域定义 JSON 字符串，每项为 `[x, y, area, height]` |
| seed | number | 0 | 随机种子，0 = 当前时间 |

### regions 字段说明

`regions` 是一个 JSON 字符串（也可直连数组）。每一项是长度为 4 的数组：

| 下标 | 字段 | 类型 | 说明 |
|------|------|------|------|
| `[0]` | x | number | 种子点列坐标（0 = 最左列） |
| `[1]` | y | number | 种子点行坐标（0 = 最上行） |
| `[2]` | area | number | 目标面积（格数），实际生长结果会经样条平滑后近似该值 |
| `[3]` | height | number | 立面高度，仅写入名称清单 `height` 字段；`>0` 时渲染器自动作 cliff 立面渲染 |

示例：

```json
[
  [12, 8, 80, 0],
  [30, 22, 120, 2],
  [50, 14, 60, 4]
]
```

含义：在 `(12,8)` 生长一个 ~80 格的平面区域（高度 0），在 `(30,22)` 生长一个 ~120 格、立面高 2 的区域，在 `(50,14)` 生长一个 ~60 格、立面高 4 的区域。

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGridList | array | 单值网格列表，与输入区域列表一一对应；每张网格只含本区域的 ID 与 0 |
| outputNameList | array | 名称清单 `[{id, name: '区域 N', type: 'tile', height}, ...]`，顺序与输出网格列表一致 |

## 注意事项

1. **种子点超出网格自动钳制**：若 `(x, y)` 在网格外，会自动钳制到最近的边界格点，仍能生长出区域。
2. **面积过大时实际可能小于目标**：当目标面积超过从种子点 4-连通可达的格子总数时，BFS 会用尽候选格停止，输出区域不足 `area`。
3. **ID 顺延规则**：取所有输入网格的最大值 + 1 作为第一个区域 ID，第 N 个区域 ID 为 `baseID + N - 1`；与「区域分区生成器」一致。
4. **height 不参与平面绘制**：高度只是写入名称清单，每张输出网格内的格值仍是单一区域 ID（或 0）；立面化效果完全由渲染器根据 `height` 字段叠加渲染。
5. **不同区域单独输出**：每个区域是独立的单值网格，区域之间允许在画面上重叠（由渲染层 z-order 决定显示），无需互斥放置。
