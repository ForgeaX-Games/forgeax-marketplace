# 自然装饰填充 (natural_decoration)

接收网格列表，按装饰物清单顺序多轮填充自然装饰物，填充值自动从各网格最大值+1起递增分配，各轮在前一轮剩余区域内填充，天然避免重叠；输出对应的填充后网格列表与名称清单列表。

## 功能特点

1. **列表批量处理**：输入为网格列表，输出为对应的填充后网格列表与名称清单列表，一次处理多个网格
2. **多装饰物清单**：通过 `decorations` 数组一次性配置多种装饰物，每种独立指定名称和密度，按顺序依次填充
3. **自动填充值分配**：无需手动指定 `fillValue`，电池从 `max(输入网格)+1` 起自动为每种装饰物递增分配唯一 ID
4. **五种填充算法**：随机散点、簇状聚集、边缘优先、哈希噪声、泊松盘采样，覆盖从粗放到精细的各类需求
5. **目标区域自动检测**：若 `targetValue` 在网格中找不到对应格子，自动检测网格中出现频率最高的非零值作为填充目标
6. **可复现随机**：设置固定 `seed` 后每次生成结果相同，便于调试与版本管理

## 适用情况

- 批量处理多个地形/建筑区域，点缀树木、灌木、花朵、石块等多种自然元素
- 需要对多种装饰物分别控制密度和分布风格的关卡编辑工作流
- 随机地图生成后的后处理装饰阶段，需要同步获取各装饰物 ID 映射
- 不适用于需要精确手动摆放单个装饰物的场景

## 基本使用方法

1. 将上游网格（array）连接到 `grid` 端口（兼容旧端口键 `gridList`）；可传单个二维网格或网格数组
2. 设置 `targetValue` 为可填充区域的网格值（例如草地=1）
3. 配置 `decorations` 装饰物清单，格式见下方说明
4. 选择 `algorithm` 决定分布风格
5. 可选：设置 `seed` 固定随机结果
6. 从 `outputGridList` 取单值装饰网格平铺列表（每种装饰物一张），从 `nameListList` 取与每个输入网格对应的 ID→名称映射列表

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grid | array | — | 输入网格：单个 `number[][]` 或网格数组；非零区域为填充目标（旧参数名 `gridList` 仍兼容） |
| targetValue | number | 1 | 可填充区域的整数值；若该值在某网格中不存在，自动检测该网格最高频非零值 |
| decorations | array / string | — | 装饰物清单（见下方）；也可传字符串，会先转成列表（JSON 数组文本，或 `树木:40,花草` 这类逗号分隔） |
| algorithm | string | "random" | 填充算法（见下方说明） |
| seed | number | 0 | 随机种子，0=每次结果不同 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGridList | array | **多个单值网格**的平铺列表：每个输入按装饰物顺序各输出一张（仅该种类 id 或 0），长度约为「输入数 × 装饰物种数」 |
| nameListList | array | 与**每个输入网格**一一对应；每项为该输入下各装饰物的 `{ id, name, type }`，`type` 固定为 `asset` |

## 装饰物清单格式

简化（推荐，单键对象：键=名称，值=密度 0–100）：

```json
[
  { "树木": 40 },
  { "灌木": 20 },
  { "石块": 10 }
]
```

兼容旧写法：

```json
[
  { "name": "树木", "density": 40 },
  { "name": "灌木", "density": 20 }
]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| 键名 / name | string | 装饰物名称 |
| 值 / density | number | 填充密度（0=不填充，100=最密） |

填充顺序即为数组顺序：第一种装饰物在所有 `targetValue` 格中填充，第二种在第一种填充后剩余的格子中继续填充，依此类推。

## 算法说明

| 算法值 | 名称 | 效果描述 |
|--------|------|----------|
| `random` | 均匀随机 | 每个目标格按 density/100 的概率独立随机填充，分布均匀无规律 |
| `cluster` | 簇状聚集 | 随机选取若干簇心，以衰减概率向周围扩散，形成自然聚团效果 |
| `edge` | 边缘优先 | 优先填充靠近区域边界的格子，中央较稀疏，适合林缘/墙根等场景 |
| `noise` | 哈希噪声 | 基于坐标哈希生成连续噪声场，装饰呈连片分布，结果固定（受 seed 影响）|
| `poisson` | 泊松盘采样 | 保证相邻装饰物之间有最小间距，分布最均匀，适合树木等需要间距的元素 |

### 密度对各算法的影响

- **random / noise**：密度直接映射为填充概率，线性关系
- **cluster**：密度影响簇的目标总数，高密度时簇会合并连片
- **edge**：边缘格密度最高可达 density×2，内部格密度约为 density×0.3
- **poisson**：密度越高，最小间距越小，填充越密（最小间距从 8 格线性减到约 2 格）

## 注意事项

1. **列表输入/输出**：`grid` 接收单个网格或网格数组；`outputGridList` 为所有输入上、按装饰物顺序展开的单值网格列表；`nameListList` 仍与输入网格个数一一对应。
2. **无需指定 fillValue**：填充值由电池自动分配，从 `max(输入网格)+1` 起，每种装饰物递增一个 ID
3. **noise 算法结果由坐标+seed 决定**：即使 seed=0 也会使用基于当前时间的种子，设置固定 seed 才能复现
4. **poisson 算法在高密度时性能稍慢**：目标区域超过 10 万格时建议改用 random 或 cluster
5. **只修改 targetValue 区域**：其他值的格子原样保留，可安全串联多个装饰电池分层填充不同区域
6. **decorations 为空时**：直接返回原网格列表和仅含剩余地面的单条 nameList 列表，不报错
