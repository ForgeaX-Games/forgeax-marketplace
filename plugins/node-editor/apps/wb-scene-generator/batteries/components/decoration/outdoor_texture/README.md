# 室外纹理 (outdoor_texture)

接收掩码网格列表（或单个掩码网格），使用 Whittaker 生物群系模型（温度 + 湿度双轴）与多层分形噪声，生成室外地面纹理分布，同类型纹理跨网格 OR 合并，输出最多 8 张**单值网格列表**与名称清单。

## 功能特点

1. **列表批量处理**：输入支持单网格或网格列表，同一纹理类型的结果 OR 合并为一张单值网格
2. **Whittaker 双参数模型**：以温度和湿度为坐标轴，每种纹理在二维空间中占据特定区域，用户通过两个滑块即可切换整个场景的气候风格
3. **多层分形噪声（FBM）**：4 层倍频叠加产生自然的空间变化，避免均匀分布
4. **域扭曲（Domain Warping）**：用低频噪声扭曲采样坐标，使纹理区域边界呈有机曲线而非方块
5. **伪海拔场**：低频噪声模拟地形起伏，高处偏碎石/雪地、低处偏湿草/苔藓
6. **边缘距离感知**：BFS 计算每个格子到掩码边界的距离，边缘更容易出现泥土/碎石等过渡纹理
7. **邻域一致性平滑**：两轮 8 邻域多数票平滑消除孤立像素，形成连贯的纹理片区
8. **生态过渡修正**：自动处理不合理的纹理邻接（如沙地直接挨着湿草），在过渡带插入中间类型

## 适用情况

- 需要为室外场景地面增加视觉多样性（草原、荒野、沼泽、沙漠边缘、雪原等）
- 输入为地面区域掩码或掩码列表（非零代表有效地面）
- 通过温度/湿度两个参数快速调整整体气候风格
- 输出单值网格列表，可直接接入渲染或 `fill_sort` 等下游节点

## 基本使用方法

1. 将地面掩码网格或网格列表连接到 `gridList` 端口
2. 拖动 `temperature` 滑块调整温度（左冷右热）
3. 拖动 `moisture` 滑块调整湿度（左干右湿）
4. 执行后，`outputGridList` 中每张网格仅含 0 与一种纹理 id，`nameList` 与之逐项对应

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| gridList | array | - | 掩码网格或网格列表（自动识别），非零单元格为有效地面；多个网格同类型 OR 合并 |
| temperature | number | 0.5 | 全局温度偏置 (0-1)：0=极寒，0.5=温和，1=酷热 |
| moisture | number | 0.5 | 全局湿度偏置 (0-1)：0=极干，0.5=适中，1=极湿 |
| seed | number | 0 | 随机种子，0 使用当前时间戳 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGridList | array | **单值网格列表**（最多 8 张）：每种纹理类型一张，仅含 0 与该 id；按 id 1→8 有序；与 nameList 一一对应 |
| nameList | array | `[{id, name, type:"tile"}]`，与 `outputGridList` 等长。`name` 由 **8 套主题**（草地变体 / 泥地变体 / 碎石 …）中随机选一整套，同一次运行内名称同属一种材质变体，不与其它套混用 |

## Whittaker 生物群系图

| 值 | 名称 | 温度中心 | 湿度中心 | 环境倾向 | 位置偏好 |
|----|------|---------|---------|---------|---------|
| 1 | 草地 | 0.50 | 0.55 | 温和 + 适中湿度 | 均匀分布 |
| 2 | 泥土 | 0.62 | 0.28 | 偏暖 + 偏干 | 边缘地带 |
| 3 | 碎石 | 0.35 | 0.15 | 偏冷 + 干燥 | 边缘 + 高地 |
| 4 | 沙地 | 0.82 | 0.12 | 高温 + 极干 | 轻微边缘 |
| 5 | 湿草 | 0.42 | 0.78 | 偏冷 + 潮湿 | 内部低洼 |
| 6 | 落叶 | 0.30 | 0.48 | 冷凉 + 适中湿度 | 轻微边缘 |
| 7 | 苔藓 | 0.18 | 0.88 | 寒冷 + 极湿 | 边缘低洼 |
| 8 | 雪地 | 0.08 | 0.45 | 极寒 + 适中湿度 | 高地 |

## 典型参数组合

| 场景 | temperature | moisture | 主要纹理 |
|------|------------|---------|---------|
| 温带草原 | 0.5 | 0.5 | 草地为主，散布泥土 |
| 干旱荒漠 | 0.8 | 0.1 | 沙地为主，碎石点缀 |
| 寒冷沼泽 | 0.2 | 0.9 | 苔藓、湿草为主 |
| 秋季落叶林 | 0.3 | 0.5 | 落叶、草地交替 |
| 热带湿地 | 0.7 | 0.8 | 草地、湿草混合 |
| 高山碎石坡 | 0.25 | 0.15 | 碎石为主，偶见苔藓 |
| 冰原雪地 | 0.05 | 0.4 | 雪地为主，碎石散布 |
| 雪线过渡 | 0.15 | 0.5 | 雪地、落叶、苔藓交替 |

## 注意事项

1. **与 v1.x 的差异**：v2.0 输入端口由 `grid`（单网格）改为 `gridList`（支持列表），输出由单张多值网格改为单值网格列表；旧连线中 `grid` 字段仍可兼容识别。
2. **输入约定**：网格中所有非零格均视为有效地面，零格在各类单值层上保持 0。
3. **温度/湿度语义**：这两个参数控制全局气候偏置，局部仍有噪声变化，不会出现完全均一的结果。
4. **seed 复现**：调试时建议固定 seed 确认效果，生产时改回 0 获得随机变化。
5. **过渡修正**：电池内部会自动修正不合理的邻接（如沙地直接挨着湿草会被泥土过渡），无需外部处理。
