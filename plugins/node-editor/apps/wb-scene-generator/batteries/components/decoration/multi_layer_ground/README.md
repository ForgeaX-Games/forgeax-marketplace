# 多层地面 (multi_layer_grass)

在基准网格的目标区域内生成多层 Perlin 噪声草地，每层通过二值化过滤后依次叠加合并，输出带名称清单的草地网格。

## 功能特点

1. **多层叠加**：可配置草地层数（1~N），每层使用不同随机种子独立生成，形成丰富的草地变体
2. **FastNoiseLite Perlin 噪声**：采用与内置 perlin_noise 电池完全相同的算法（梯度噪声 + FBm 分形），草地斑块平滑、边缘连续有机
3. **二值化控制**：通过阈值参数直接控制各层草地的覆盖面积（阈值越高覆盖越稀疏）
4. **基准网格感知**：只在目标区域内生成草地，基准网格其余区域（边界、道路等）保持原值不变
5. **名称清单输出**：自动生成各草地层的 ID 与名称映射，可直接供下游渲染或逻辑使用

## 适用情况

- 需要在已有地块上叠加多种草地类型（深草、浅草、花草等）
- 场景中草地区域已由上游电池确定，只需填充草地细节
- 希望用单个电池替代「噪声 → 二值化 → 掩码应用 → 合并」的多节点管线

## 基本使用方法

1. 将上游地块/地形网格连接到 **基准网格** 端口
2. 设置 **目标区域值**（-1 自动取基准网格最大值，即上游写入的最后一层掩码 ID）
3. 调整 **草地层数**、**二值化过滤值**、**噪声缩放** 等参数
4. 输出的 **nameList** 可接入文本面板查看，或连接到渲染器图层标签

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| baseGrid | grid | — | 基准网格，目标区域将被草地覆盖 |
| targetValue | number | -1 | 要覆盖的区域掩码值；-1 自动取网格最大值 |
| layerCount | number | 4 | 草地层数，每层独立 ID |
| threshold | number | 0.6 | 二值化阈值（0~1），值越高草地面积越小 |
| frequency | number | 0.02 | 噪声频率，值越小斑块越大越平滑（与 perlin_noise 电池保持一致） |
| octaves | number | 5 | 噪声倍频数，值越大细节越丰富 |
| seed | number | 0 | 随机种子，0 使用当前时间戳 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGrid | grid | 合并所有草地层后的网格 |
| nameList | array | 各草地层 ID 与名称映射，格式 `[{id, name}]` |

## 草地层 ID 规则

- 草地层 ID 从 `targetValue + 1` 开始顺延
- 例如 targetValue=1，layerCount=4 → 草地层 ID 分别为 2、3、4、5
- 各层名称为「草地层1」、「草地层2」…

## 注意事项

1. **层间覆盖**：后生成的层会覆盖先生成的层，层数越多最终占主导的是编号最大的草地层
2. **阈值建议**：0.5~0.7 的阈值通常产生自然比例的草地斑块；低于 0.4 会覆盖过多，高于 0.8 过于稀疏
3. **目标区域为空时**：若基准网格中不存在 targetValue 对应的格子，输出与输入完全相同
4. **ID 顺延**：如果上游管线有多层掩码，务必将 targetValue 设为正确的目标层 ID，避免覆盖其他层
