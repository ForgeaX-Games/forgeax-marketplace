# 清单预览 (Name List Panel)

将 `[{id, name, ...}]` 格式的名称清单格式化展示，每个条目单独一行，去除多余换行，方便观察和调试。

## 功能特点

1. **紧凑展示**：每个清单条目单独一行，保持 JSON 合法性的同时消除嵌套展开
2. **兼容多种输入**：支持数组对象、JSON 字符串等多种输入形式
3. **样式仿文本面板**：与文本面板一致的渲染风格，无缝融入已有工作流

## 适用情况

- 需要快速查看名称清单内容时
- 调试 ALG 电池输出的 `outputNameList` 时
- 多个 id/name 条目多而杂，需要逐条对比时

## 基本使用方法

将上游电池的 `outputNameList` 连接到本电池的「名称清单」输入端口，即可在面板中看到每条目单行展示的清单。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| input | any | null | 名称清单数组，格式为 `[{id, name, ...}]` |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| output | string | 格式化后的文本，每个条目占一行 |

## 使用示例

### 输入

```json
[
  {"id": 1, "name": "基准地块1"},
  {"id": 2, "name": "基准地块1_普通地板"},
  {"id": 3, "name": "基准地块1_青苔地板"}
]
```

### 输出（面板显示）

```
[
  {"id":1,"name":"基准地块1"},
  {"id":2,"name":"基准地块1_普通地板"},
  {"id":3,"name":"基准地块1_青苔地板"}
]
```

## 注意事项

1. **输入为空时**：面板显示空字符串，不报错
2. **输入为字符串时**：会尝试 JSON.parse，解析成功后同样格式化展示
3. **非数组输入**：直接 JSON.stringify 输出，不做特殊格式化
