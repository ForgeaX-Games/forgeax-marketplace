# 搜索 (Search)

在名称清单列表中搜索包含指定关键词的条目，整组过滤输出匹配的网格与名称条目。

## 功能特点

1. **整组过滤**：网格列表与名称清单列表一一对应，只保留名称匹配的那些对，不匹配的整组丢弃。
2. **多格式关键词**：搜索内容支持字符串、字符串列表、名称清单（取 name）、JSON字符串化数组。
3. **OR 匹配**：多个关键词任意匹配一个即保留。
4. **空关键词透传**：搜索内容为空时全部保留，不过滤。

## 适用情况

- 输入 N 个网格 + N 条名称，搜索后只输出匹配的 M 个网格 + M 条名称（M ≤ N）
- 典型场景：70个建筑地块网格 + 70条名称，搜索"大门"后输出8个网格 + 8条名称

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| gridList | any | null | 网格列表（`number[][][]`），与名称清单按索引对应 |
| nameListList | any | null | 名称清单列表（`NameEntry[]`），每条与同索引网格对应 |
| mode | string | "包含" | 搜索模式，目前仅支持「包含」 |
| searchContent | any | "" | 搜索关键词，支持多种格式 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGridList | array | 过滤后的网格列表，只保留匹配条目对应的网格 |
| outputNameListList | array | 过滤后的名称列表，只保留匹配条目 |

## 示例

输入 70 个网格 + 70 条名称，搜索"大门"，其中 8 条名称包含"大门"：

- `outputGridList` 长度：8
- `outputNameListList` 长度：8

## 注意事项

1. gridList 与 nameListList 长度不一致时，取较短一方的长度处理。
2. 搜索内容为空时不过滤，原样输出全部条目。
