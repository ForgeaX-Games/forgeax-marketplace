# 渲染预处理 (Render Preprocess)

按照渲染顺序对网格列表和名称清单进行重排与累积合并，使每个 tile 的网格包含其前序所有 tile 的区域，为渲染管线提供标准化的层级输入。

## 功能特点

1. **tile/asset 区分**：自动识别名称清单中的 `type` 字段，只对 tile 类型进行顺序合并，asset 类型直接排到末尾。
2. **累积合并（Accumulate OR）**：越靠后的 tile，其网格会包含前面所有 tile 的区域（二值 OR 操作），模拟渲染时"底层叠加"的语义。
3. **平级分组 `[]`**：渲染顺序中用 `[]` 括起来的 tile 互相独立（不互相合并），但都会继承前面的累积，且对后续 tile 来说，`[]` 内所有 tile 的并集会向后传递。
4. **自动重排**：按 renderOrder 顺序输出 tile，未出现在 renderOrder 中的 tile 跟随其后，asset 排在最末尾。
5. **详细报告**：输出 `detail` 字段，说明每一步的处理情况（忽略原因、名称匹配、最终顺序等）。

## 适用情况

- 需要将多层 tile 网格按渲染层级顺序叠加的场景（如地形渲染、地图叠加）
- 管线中 `format_factory` 之后，实际渲染之前的预处理步骤
- 需要将 tile 和 asset 分离排序的工作流

## 基本使用方法

1. 将 `format_factory` 的 `outputGridList` 和 `outputNameList` 接入本电池
2. 在 `renderOrder` 参数中填写期望的渲染顺序 JSON 字符串
3. 本电池输出的 `outputGridList` 和 `outputNameList` 可直接接入渲染节点

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| inputGridList | array | - | 输入的 01 网格列表，与名称清单一一对应 |
| nameList | array | - | 名称清单，格式为 `[{id, name, type}]` |
| renderOrder | string | `"[]"` | 渲染顺序 JSON 字符串，见下方格式说明 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGridList | array | 重排并累积合并后的网格列表 |
| outputNameList | array | 重排后的名称清单，格式 `[{id, name, type}]` |
| detail | string | 处理过程报告 |

## renderOrder 格式说明

`renderOrder` 是一个 JSON 字符串，表示渲染顺序列表：

```json
["浅水", "中水", "深水", "沙滩", ["草地", "浓草地"], "深色草地", ["土路", "乡间道路"], "山地", "雪山"]
```

### 格式规则

- **字符串**：单个 tile 名称，独占一个顺序位
- **数组 `[]`**：括号内的 tile 为**平级关系**，互不合并，但共同继承前面的累积

### 累积合并示意

以 `["浅水", "中水", "深水"]` 为例：

| 输出 tile | 实际网格内容 |
|-----------|-------------|
| 浅水 | 浅水区域（原始） |
| 中水 | 中水区域 OR 浅水区域 |
| 深水 | 深水区域 OR 中水区域 OR 浅水区域 |

以 `["浅水", ["草地", "浓草地"], "山地"]` 为例：

| 输出 tile | 实际网格内容 |
|-----------|-------------|
| 浅水 | 浅水区域（原始） |
| 草地 | 草地区域 OR 浅水区域 |
| 浓草地 | 浓草地区域 OR 浅水区域（不含草地） |
| 山地 | 山地区域 OR 浓草地区域 OR 草地区域 OR 浅水区域 |

## 使用示例

### 输入示例

```json
{
  "renderOrder": "[\"浅水\",\"中水\",[\"草地\",\"浓草地\"],\"山地\"]"
}
```

### 名称清单示例

```json
[
  {"id": 1, "name": "浅水", "type": "tile"},
  {"id": 2, "name": "中水", "type": "tile"},
  {"id": 3, "name": "草地", "type": "tile"},
  {"id": 4, "name": "浓草地", "type": "tile"},
  {"id": 5, "name": "山地", "type": "tile"},
  {"id": 6, "name": "树木", "type": "asset"}
]
```

### 输出顺序

```
浅水 → 中水 → 草地 → 浓草地 → 山地 → 树木（asset 排末尾）
```

## 注意事项

1. **对应关系**：`inputGridList[i]` 必须与 `nameList[i]` 一一对应，顺序不能错乱（通常 `format_factory` 已保证这一点）。
2. **类型字段**：名称清单中的 `type` 字段需为 `"tile"` 或 `"asset"`，空字符串或未填写的条目会被视为 tile 处理。
3. **名称匹配**：`renderOrder` 中的名称必须与名称清单中的 `name` 字段完全一致（区分大小写）；不存在的名称会被忽略并在 `detail` 中报告。
4. **asset 忽略**：即使 `renderOrder` 中列出了 asset 类型的名称，也会被自动忽略，asset 始终排在末尾。
5. **未出现的 tile**：名称清单中有但 `renderOrder` 中没有提及的 tile，会排在有序 tile 之后、asset 之前，保持原有相对顺序。
