# 透传 (Pass Through)

将多个动态输入值逐个原样输出，不做任何转换、校验或重编号。

## 功能特点

1. **动态输入端口**：默认提供 `input_0`、`input_1`，连接后会自动追加 `input_2`、`input_3` 等槽位。
2. **成对原样输出**：每个 `input_i` 会输出到对应的 `output_i`，不进行深拷贝、格式化或重编号。
3. **保持数据形状**：端口使用 `tree` access，尽量保留上游 wire 的原始结构。
4. **调试占位**：适合作为临时中转节点，用于保持管线结构或观察数据探针。

## 适用情况

- 需要在管线中临时占位，但不改变数据内容
- 需要把一组上游输出分别直接转接到下游
- 调试复杂连线时，需要一个语义明确的直通节点

## 基本使用方法

将任意上游输出连接到 `input_0`、`input_1` 等动态输入端口，再从对应的 `output_0`、`output_1` 连接到下游节点即可。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| input_0 | any | null | 第0个任意输入值 |
| input_1 | any | null | 第1个任意输入值，连接后自动追加新输入槽位 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| output_0 | any | 与 `input_0` 完全相同的值 |
| output_1 | any | 与 `input_1` 完全相同的值 |

## 使用示例

### 输入示例

```json
{
  "input_0": {
    "grid": [[1, 0], [0, 1]],
    "nameList": [{"id": 1, "name": "地块"}]
  },
  "input_1": "说明文本"
}
```

### 输出示例

```json
{
  "output_0": {
    "grid": [[1, 0], [0, 1]],
    "nameList": [{"id": 1, "name": "地块"}]
  },
  "output_1": "说明文本"
}
```

## 注意事项

1. **不做校验**：本电池不会检查输入格式是否符合下游要求。
2. **不做拷贝**：输出保持输入值本身的结构，适合作为纯直通节点。
3. **动态端口成对命名**：`input_i` 对应 `output_i`，下游请连接同编号输出端口。
