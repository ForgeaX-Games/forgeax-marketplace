# image_preview — 图像预览

## 功能

在画布上**直接预览** image 端口上的图像，并把同一个图像 alias / data URL **原样透传**到下游，方便：

- 在图像处理链路中插入一个观察点，确认上游图像真的加载到了
- 挂在任何 AI 生图节点（图生图、文生图）输出端口后，第一时间看到生成结果
- 作为「探针式」可视化中转节点，连线不会被打断，删除节点也不影响数据流

## 输入

| 名称 | 类型 | 说明 |
|------|------|------|
| `image` | `image` | 上游图像：可以是资产库中的 alias，也可以是 `data:image/...;base64,...` data URL |

## 输出

| 名称 | 类型 | 说明 |
|------|------|------|
| `image` | `image` | 与输入完全一致的图像（透传） |

## 渲染说明

- 后端 `index.ts` 只做透传，不做任何编码 / 解码。
- 真正的预览发生在编辑器前端 **ImagePreviewNode**：
  - 通过 `nodeOutputs[上游节点][上游端口]` 拿到 image 字符串
  - 若以 `data:` 开头 → 直接作为 `<img>` 的 src
  - 否则视为 alias → 通过 `/api/v1/library/serve/:alias` 获取图像
- 未连接上游时，节点展示空状态提示。

## 使用方法

1. 从电池库（`special/texture` 分类）拖入画布
2. 把任何输出 `image` 类型的端口连到该节点的 `image` 输入
3. 节点中央立刻显示对应图像缩略图
4. 如需继续传递给下游，直接从节点右侧 `image` 输出端口接出即可

## 典型用法

```
image_reader → image_preview
              (确认本地图加载成功)

AI image_gen → image_preview              (生成结果即时可视化)
              ↘ asset_export → [资产库]
```

## 与 image_reader 的区别

| 维度 | image_reader | image_preview |
|------|--------------|----------------|
| 输入 | 无 | `image`（必接） |
| 行为 | 双击选择本地文件 → 上传资产库 → 输出 alias | 接收上游 image → 显示 → 透传 |
| 用途 | 引入新图像 | 观察 / 调试 已有图像数据流 |
