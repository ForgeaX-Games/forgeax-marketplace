# 文本解析 (text_parse)

将文本字符串自动解析为单个数值或数值列表。保留原始嵌套结构，支持多种常见文本格式，自动检测分隔符。

## 功能特点

1. **智能格式识别**：自动识别 JSON 数组、分隔值、单个数字等格式
2. **保留嵌套结构**：`[[1,2],[3,4]]` 原样保留为嵌套列表，不会展平
3. **自动分隔符检测**：未指定分隔符时，按出现频率自动选择最合适的分隔符
4. **双输出互补**：单个数值走 `value`，多个数值走 `list`，下游电池按需取用
5. **容错解析**：忽略无法解析的文本片段，尽可能提取有效数值

## 输出规则

- **单个数值**（如 `"42"`）：`value = 42`，`list = []`
- **多个数值**（如 `"[1,2,3]"`）：`value = 1`（首个），`list = [1,2,3]`
- **嵌套结构**（如 `"[[1,2],[3,4]]"`）：`value = 1`（首个），`list = [[1,2],[3,4]]`

## 支持的输入格式

| 格式 | 示例 | value | list |
|------|------|-------|------|
| 单个数字 | `"42"` | 42 | [] |
| 小数/科学计数 | `"3.14"` | 3.14 | [] |
| JSON 一维数组 | `"[1, 2, 3]"` | 1 | [1, 2, 3] |
| JSON 嵌套数组 | `"[[1,2],[3,4]]"` | 1 | [[1,2],[3,4]] |
| 逗号分隔 | `"1, 2, 3, 4"` | 1 | [1, 2, 3, 4] |
| 分号分隔 | `"1; 2; 3"` | 1 | [1, 2, 3] |
| 空格分隔 | `"1 2 3 4"` | 1 | [1, 2, 3, 4] |
| 换行分隔 | `"1\n2\n3"` | 1 | [1, 2, 3] |
| 空文本 | `""` | 0 | [] |

## 解析优先级

1. **JSON 解析**：首先尝试 `JSON.parse`，保留任意深度的嵌套结构
2. **单数字**：若整段文本是一个合法数字，直接输出到 `value`
3. **分隔符拆分**：按检测到的分隔符拆分，逐段解析为数字，忽略非数字片段

### 分隔符自动检测优先级

逗号 `,` > 分号 `;` > 制表符 `\t` > 换行 `\n` > 空格 ` `

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| text | string | "" | 待解析的文本 |
| delimiter | string | "" | 分隔符，留空自动检测 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| value | number | 单个数值（列表时取首个，无效时为 0） |
| list | array | 数值列表，保留原始嵌套结构（单值时为空数组） |

## 注意事项

1. **结构保留**：JSON 嵌套数组会原样保留，如 `"[[[1]],[[2]]]"` → `[[[1]],[[2]]]`
2. **单值不重复**：当文本只有一个数字时，仅 `value` 有值，`list` 为空，避免冗余
3. **非数字被跳过**：`"1, hello, 3"` → list=[1, 3]
4. **与其他电池组合**：`list` 可作为 Voronoi 地形的种子点等参数；`value` 可作为数值参数
