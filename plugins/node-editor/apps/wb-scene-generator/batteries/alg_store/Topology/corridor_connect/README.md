# 走廊连接 (corridor_connect)

对已有房间网格构建最小生成树（Kruskal 算法），然后用 L 型或 Z 型走廊连接所有房间。可选添加额外连线形成环路，增加地图的探索性。

## 功能特点

1. **自动提取房间**：从输入网格中扫描所有正整数 ID 的房间，计算各房间几何中心
2. **MST 保证全连通**：基于房间中心距离构建最小生成树，确保所有房间可达
3. **L/Z 型走廊**：L 型为两段正交直线，Z 型为三段折线；可选随机混合
4. **额外连线**：在 MST 基础上按比例随机添加额外边，形成环路
5. **非破坏性叠加**：走廊仅覆盖空地（值为 0 的格子），不会覆盖已有房间

## 适用情况

- 配合 `poisson_room_placement` 或 `bsp_room_generator` 的输出，添加走廊连接
- 任何需要将分散的矩形区域相互连通的场景
- 地牢/关卡后处理，生成可行走的通道网络

## 基本使用方法

1. 将包含房间的网格连接到 `房间网格` 端口（0=空地，正整数=房间ID）
2. 设置 `走廊宽度`、`走廊填充值`
3. 设置 `额外连线比例` 控制环路数量
4. 从下拉框选择 `走廊形状`（L / Z / random）
5. 从 `连接后网格` 获取完整地图，从 `走廊网格` 获取走廊遮罩

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grid | grid | — | 输入房间网格：0=空地，正整数=房间ID |
| corridorWidth | number | 1 | 走廊宽度（1~5 格） |
| corridorValue | number | -1 | 走廊格子的填充值；-1 表示自动使用最大房间ID+1 |
| extraEdgeRatio | number | 0.15 | MST 之外额外连线的比例（0~1） |
| shape | string | random | 走廊形状：L=L型，Z=Z型，random=随机混合 |
| seed | number | 0 | 随机种子，0 使用默认种子 |

### 参数说明

**走廊填充值 (corridorValue)**：决定走廊格子在输出网格中的数值。设为 -1 时自动计算为最大房间ID+1，确保走廊值与所有房间 ID 不冲突。也可手动指定（如设为 -1 表示墙壁镂空，或设为 999 作为特殊标记）。

**额外连线比例 (extraEdgeRatio)**：0 表示只保留 MST（树状结构，无环路），1 表示所有可能的房间对都连接。推荐 0.1~0.3 可产生少量环路，增加地图探索性。

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| grid | grid | 包含房间和走廊的完整网格 |
| corridorGrid | grid | 走廊遮罩网格：0=非走廊，1=走廊 |
| connections | array | 连接对数组 [{from, to}]，from/to 为房间ID |
| numConnections | number | 走廊连接总数 |

## 注意事项

1. **输入要求**：输入网格必须包含至少 2 个不同的正整数房间 ID，否则无走廊可连
2. **非破坏叠加**：走廊只填充值为 0 的空地格子，穿过房间区域时保留房间原值
3. **走廊交叉**：多条走廊可能在空地处交叉，交叉点值相同（均为走廊填充值）
4. **搭配使用**：推荐将 `poisson_room_placement` 或 `bsp_room_generator` 的 grid 输出直接连接到本电池的 grid 输入
