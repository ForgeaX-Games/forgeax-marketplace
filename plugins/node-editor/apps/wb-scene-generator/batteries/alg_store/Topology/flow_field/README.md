# 流场 (flow_field)

构造从全图朝最近目标点汇聚的代价图与方向图，是 RTS 群体寻路、玩家自动导航等场景的核心数据结构。

## 功能特点

1. **多目标共享**：所有目标共用一个流场，每格自动指向最近目标，非常适合多个出口/集结点。
2. **代价感知**：可选 `useCellWeight=true` 把网格数值作为通过代价（噪声地形/沼泽减速等）。
3. **三类输出**：代价图（热力可视）、方向编码图（1~8 离散方向）、向量场（连续 [vx,vy]，可直接驱动 agent 移动）。
4. **8 方向支持**：默认开启对角，效果更平滑。

## 适用情况

- RTS 群体寻路：所有单位"流向集合点"
- 玩家迷雾里的自动指引箭头
- AI Agent 朝向最近资源点移动
- 风/水流模拟可视化

## 基本使用方法

1. 接入二值/多值网格作为可通行性 + 代价图。
2. 在 `targets` 输入目标点列表。
3. 想要更平滑的视觉，渲染 `vectorField`；想要程序化驱动 agent，用 `directionGrid` 查表。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grid | grid | — | 可通行性 / 代价网格 |
| targets | number rank 2 | [] | 目标点 `[[x,y],...]`（也兼容旧字符串 "x,y; x,y"） |
| obstacleValue | number | 0 | 视为墙的网格值 |
| diagonal | boolean | true | 是否允许 8 方向 |
| useCellWeight | boolean | false | 是否使用网格数值作为通过代价 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| costGrid | grid | 累计代价图，不可达 = -1 |
| directionGrid | grid | 方向编码 1~8（0=已在目标 / 不可达） |
| vectorField | array | [h][w] 单位向量数组 [vx,vy] |

## 注意事项

1. **方向编码顺序**：1=右、2=右下、3=下、4=左下、5=左、6=左上、7=上、8=右上（8 方向时）；4 方向时 1=右、2=下、3=左、4=上。
2. **vectorField 数据量大**：是 array 而非 grid，渲染器不直接显示；如需可视化，自行做缩放采样后输出 grid。
3. **目标全部不可达时**：所有格子 cost=-1、direction=0。
