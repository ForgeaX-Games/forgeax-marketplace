# 黏菌算法 (alg_slime_mold)

模拟真实黏菌（Physarum polycephalum）的觅食行为，在网格掩码内生成有机的网状通道结构。

算法结构基于 Jones (2010) 模型 / slime-sim-webgpu 参考实现的改进（3×3 块传感器、先移动后感知、圆盘沉积、改进转向决策）。沉积/衰减模型采用累加沉积 + 乘法衰减 + 部分扩散，适合小网格二值输出场景。

## 功能特点

1. **仿生网络**：基于 Physarum polycephalum 黏菌的真实生物行为建模，产生自然、有机的路径网络
2. **3×3 块传感器**：每个智能体在三个方向（前方、左前、右前）各采样 3×3 邻域（共 9 格）信息素总和，提供更鲁棒的感知
3. **先移动后感知**：智能体先沿当前方向移动（Motor），再感知并转向（Sensory），最后沉积轨迹（Deposit），符合经典 Physarum 两阶段模型
4. **圆盘沉积**：轨迹以可配置半径的圆盘足迹累加沉积，路径宽度可调
5. **部分扩散 + 乘法衰减**：每步 5% 浓度向邻居均值扩散，其余保留原位，再乘以衰减率；路径上的格子衰减慢（邻居也有高浓度），孤立格子衰减快，天然产生路径与背景的强对比
6. **可配置步长**：智能体每步移动距离可调，影响轨迹密度
7. **分散食物源**：少量食物源（3~8）通过最远点采样均匀分布，引导网络在它们之间形成连接
8. **随机扰动**：直行时加入小幅 wander 噪声，打破对称性，产生自然分叉

## 适用情况

- 生成有机风格的洞穴通道、地下河道网络
- 城市道路网络或自然小路的程序化生成
- 生物/细胞风格游戏的地图地形
- 与其他算法（BSP、细胞自动机）组合使用，产生混合风格
- 不适用于需要严格几何规则的场景（如城市正交网格）

## 基本使用方法

1. 连接一个网格（如矩形网格或已有地图掩码）到 `grid` 输入端口
2. 调整 `agentCount`（建议 300~800）和 `steps`（建议 150~300）
3. 用 `trailThreshold` 控制输出路径的密度（值越大路径越细）
4. 设置 `seed=0` 获得随机结果，或指定固定值重复相同布局

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grid | grid | - | 输入掩码，非零区域为仿真域 |
| agentCount | number | 500 | 智能体数量（10~5000） |
| steps | number | 200 | 仿真步数（10~800） |
| sensorAngle | number | 45 | 传感器张角（度，5~90） |
| sensorDistance | number | 8 | 传感器前瞻距离（格，1~20） |
| turnSpeed | number | 45 | 每步最大转向角度（度，1~90） |
| stepSize | number | 0.5 | 每步移动距离（格，0.1~3） |
| decayRate | number | 0.99 | 轨迹每步保留比例（0.8~0.999） |
| depositAmount | number | 1.0 | 每步累加沉积轨迹量（0.1~10） |
| depositRadius | number | 1.5 | 沉积圆盘半径（格，0.5~5） |
| trailThreshold | number | 0.15 | 路径标记阈值，相对于最大轨迹（0.01~0.99） |
| seed | number | 0 | 随机种子，0=自动随机 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| grid | grid | 输出网格，路径区域=1，其余=0 |

## 参数调节指南

### 路径密度
- 更密集：降低 `trailThreshold`（如 0.08）或增加 `agentCount`
- 更稀疏：提高 `trailThreshold`（如 0.4）或减少 `agentCount`

### 路径宽度
- 更宽：增大 `depositRadius`（如 2.5~3.0）
- 更细：减小 `depositRadius`（如 0.5，退化为单点沉积）

### 路径平滑度
- 更平滑：增大 `sensorDistance`（如 10~15）和减小 `turnSpeed`（如 20~30）
- 更曲折：减小 `sensorDistance`（如 2~3）和增大 `turnSpeed`（如 60~90）

### 网络特征
- 增大 `decayRate`（接近 1）：轨迹持久，路径更粗、更多分支
- 减小 `decayRate`（如 0.9）：轨迹快速消散，只剩主干通道

### 移动步长
- 较小 `stepSize`（0.3~0.5）：轨迹更密集，细节更多
- 较大 `stepSize`（1.0~2.0）：轨迹更稀疏，覆盖更广

### 传感器张角
- 较小 `sensorAngle`（20~30°）：智能体更敏锐地追踪现有轨迹，网络更收敛
- 较大 `sensorAngle`（50~70°）：智能体更容易发现侧面轨迹，网络更分散

## 使用示例

### 输入

```json
{
  "grid": [[0,0,...],[0,1,1,...],...]，
  "agentCount": 500,
  "steps": 200,
  "sensorAngle": 45,
  "sensorDistance": 8,
  "turnSpeed": 45,
  "stepSize": 0.5,
  "decayRate": 0.99,
  "depositAmount": 1.0,
  "depositRadius": 1.5,
  "trailThreshold": 0.15,
  "seed": 42
}
```

### 输出

```json
{
  "grid": [[0,0,...],[0,1,0,1,...],...]
}
```

## 注意事项

1. **性能**：`agentCount × steps` 决定计算量，建议不超过 200,000（如 500 × 400），超大参数可能导致计算缓慢
2. **初始状态**：智能体随机分布在整个掩码区域内，食物源通过最远点采样均匀散布
3. **随机性**：`seed=0` 时每次结果不同；设置固定 seed 可复现相同结果
4. **掩码要求**：输入网格的非零区域即为有效仿真域，建议使用较大的连通区域（至少 20×20）以获得好的效果
5. **阈值敏感性**：`trailThreshold` 对输出路径密度影响较大，建议从 0.1~0.2 开始调试
