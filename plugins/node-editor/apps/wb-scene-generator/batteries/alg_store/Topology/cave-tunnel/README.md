# 洞穴隧道 (cave_tunnel)

在岩石区域内生成分叉洞穴隧道。基于 3D Perlin 噪声方向偏移、宽度衰减和递归分支，产生自然的树状洞穴网络。

## 典型应用场景

- **地下洞穴系统**：在岩石区域内生成蜿蜒的隧道网络，模拟自然洞穴
- **矿洞/地牢走廊**：作为地牢生成的基础结构，主通道向深处延伸并分叉
- **根系/河道生成**：树状分支结构也可用于模拟树根、河流支流等自然形态
- **地形侵蚀效果**：在已有地形上"挖掘"出通道，增加地形细节

## 内部常量与默认值

| 常量 | 值 | 对应参数 |
|------|-----|-----------|
| WIDTH_CURVE | [(100,2.0), (300,4.0), (3000,5.5)] | tunnelWidth=0 时自动使用 |
| WIDTH_PER_CELL | 0.034 | widthDecay 默认 0.034 |
| MIN_WIDTH | 1.4 | minWidth 默认 1.4 |
| DIR_CHANGE_SPEED | 8.0° | dirChangeSpeed 默认 8 |
| DIR_NOISE_FREQ | 0.00205 (3D Perlin, 4 octave) | 完整移植的 PerlinNoise3D |
| BRANCH_CHANCE | 0.1 | branchChance 默认 0.1 |
| BRANCH_AFTER | 15 | branchAfter 默认 15 |
| MIN_ROCK_GROUP | 300→20 | minGroupSize 默认 20（适配小网格） |
| OPEN_PER_10K | 5.8 | 内部使用相同公式 |
| CLOSED_PER_10K | 2.5 | 内部使用相同公式 |
| MAX_OPEN / MAX_CLOSED | 3 / 1 | maxOpenTunnels / maxClosedTunnels |

**为适配 50×50 网格的修改**：
- `minGroupSize` 默认降至 20（小网格的岩石组面积较小）
- 分支距离阈值按组面积缩放：`max(5, min(18, floor(sqrt(面积) × 0.4)))`
- 面积不足以产生隧道时（按密度公式为 0 条），保证至少 1 条开放隧道

## 算法原理

### 1. 连通区域识别（Flood Fill）

对输入网格进行四连通 Flood Fill，识别所有连通的非零区域（岩石组）。面积低于 `minGroupSize` 的小区域被跳过。

### 2. 隧道宽度（WIDTH_CURVE）

隧道初始宽度根据岩石组面积自动缩放：

| 岩石组面积 | 隧道宽度 |
|-----------|---------|
| ≤100 | 2.0 |
| 300 | 4.0 |
| ≥3000 | 5.5 |
| 中间值 | 线性插值 |

设置 `tunnelWidth > 0` 可以覆盖此自动缩放。

### 3. 隧道类型与数量

| 类型 | 起点 | 方向 | 数量公式 |
|------|------|------|----------|
| **开放隧道** | 岩石组边缘（3像素边距） | 向岩石内部最深处 | `min(maxOpen, round(面积 × random(0.9,1.1) × 5.8 / 10000))` |
| **封闭隧道** | 岩石组内部 | 随机方向 | `min(maxClose, round(面积 × random(0.9,1.1) × 2.5 / 10000))` |

### 4. 挖掘过程

每一步执行：

1. **封闭隧道碰撞检测**：检查挖掘范围是否会接触到已有隧道或非岩石区域
2. **分支判定**（步数 ≥ branchAfter 且 width > minWidth + 0.4）
3. **圆形挖掘**：以当前位置为圆心、`floor(width/2)` 为半径挖掘
4. **碰撞检测**：接触到非本隧道的已挖区域时终止
5. **前进**：沿当前方向以 0.5 像素步长前进到下一个整数格（使用 `Math.floor` 取整，与原版 `int()` 一致）
6. **方向偏移**：`direction += PerlinNoise3D(dist×60, startX×200, startY×200) × dirChangeSpeed`
7. **宽度衰减**：`width -= widthDecay`

### 5. 3D Perlin 噪声

使用经典 3D Perlin 噪声实现：
- 12 梯度向量的经典 3D Perlin 梯度噪声
- 5 阶 fade 曲线：$6t^5 - 15t^4 + 10t^3$
- 4 个 octave，lacunarity=2.0，persistence=0.5
- 频率 0.00205

噪声输入 `(dist×60, startX×200, startY×200)` 确保：
- 沿隧道进展方向平滑变化（约每 8 步一个周期）
- 不同起点的隧道有完全不同的蜿蜒模式

### 6. 递归分支

分支条件（全部满足才触发）：
- 已行进 ≥ `branchAfter` 步（默认 15）
- 当前宽度 > `minWidth + 0.4`（即 > 1.8）
- 随机概率 < `branchChance`（默认 10%）

分支方向偏离主隧道 40°~90°，测试 6 个随机候选方向，选择能深入岩石最远的。分支距离阈值按组面积缩放（小组 5 格，大组 18 格）。

分支宽度 = 主隧道当前宽度 - random(0.2, 0.4)。分支共享同一"隧道标签"（不触发碰撞终止）。每条隧道最多分支一次左、一次右。

### 7. 宽度衰减

$$w_{n+1} = w_n - \text{widthDecay}$$

默认参数下（widthDecay=0.034, minWidth=1.4）：
- 宽度 4.0 的隧道最多延伸约 76 步
- 宽度 5.5 的隧道最多延伸约 121 步
- 实际长度受限于岩石区域大小

## 输入参数

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| grid | grid | — | — | 输入网格掩码，非零=岩石，0=背景 |
| maxOpenTunnels | number | 3 | 0–10 | 每组开放隧道数量上限 |
| maxClosedTunnels | number | 1 | 0–5 | 每组封闭隧道数量上限 |
| minGroupSize | number | 20 | 1–5000 | 岩石组最小面积 |
| tunnelWidth | number | 0 | 0–20 | 隧道宽度，0=自动缩放 |
| widthDecay | number | 0.034 | 0.001–1 | 每步宽度减少量 |
| minWidth | number | 1.4 | 0.5–10 | 低于此宽度停止挖掘 |
| branchChance | number | 0.1 | 0–1 | 每步分支概率 |
| branchAfter | number | 15 | 0–100 | 允许分支的最小步数 |
| dirChangeSpeed | number | 8 | 0–45 | 每步最大方向变化（度） |
| seed | number | 0 | — | 随机种子，0=默认 |

### 参数调节指南

| 效果 | 调节方法 |
|------|----------|
| 更长的隧道 | 减小 `widthDecay`（如 0.02）或设置更大的 `tunnelWidth` |
| 更短的隧道 | 增大 `widthDecay`（如 0.06） |
| 更多分支 | 增大 `branchChance`（如 0.2）、减小 `branchAfter`（如 8） |
| 更少分支 | 减小 `branchChance`（如 0.05）、增大 `branchAfter`（如 25） |
| 更蜿蜒 | 增大 `dirChangeSpeed`（如 15） |
| 更笔直 | 减小 `dirChangeSpeed`（如 3） |
| 更宽的隧道 | 设置 `tunnelWidth`（如 6），或保持 0 让大岩石组自动获得更宽隧道 |
| 更多隧道 | 增大 `maxOpenTunnels` / `maxClosedTunnels` |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| grid | grid | 隧道网格，1=隧道，0=非隧道 |

## 注意事项

1. **输入为掩码**：只在非零区域内挖掘，隧道不会超出岩石范围
2. **多个岩石组**：如果输入网格有多个不相连的非零区域，每个区域独立生成隧道
3. **自动宽度缩放**：tunnelWidth=0 时，宽度根据 WIDTH_CURVE 随岩石组面积自动调整（小组窄、大组宽）
4. **隧道数量自适应**：按密度公式（5.8/万 开放 + 2.5/万 封闭）计算。小组（面积 < ~1700）按公式为 0 条时自动补为 1 条
5. **碰撞终止**：隧道挖掘到其他隧道时自动终止，避免过度重叠
6. **确定性**：相同 `seed` 和输入产生相同结果
7. **与其他电池组合**：输出可通过 blend 电池与原始地形叠加；也可作为 mask 用于下游电池
