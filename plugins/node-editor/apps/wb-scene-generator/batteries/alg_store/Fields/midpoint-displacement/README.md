# 中点位移地形 (midpoint_displacement)

Midpoint Displacement 算法电池，用于生成随机分形地形高度图。与 Diamond-Square 算法类似但实现方式不同：边缘中点仅由该边两个端点的平均值加随机扰动决定，因此地形特征略有不同，具有更明显的轴向结构感。

## 功能特点

- 2D 中点位移分形地形生成
- 网格边长为 2^power + 1，保证递归对称性
- 支持粗糙度控制：影响每层迭代的扰动衰减速率
- 可配置初始角点高度和扰动幅度
- 确定性随机种子，相同参数产生相同结果
- 输出归一化到 0~1

## 与 Diamond-Square 的区别

| 特性 | Diamond-Square | Midpoint Displacement |
|------|---------------|----------------------|
| 边缘中点计算 | 使用 4 个邻居的平均 | 仅使用 2 个端点的平均 |
| 视觉特征 | 更均匀的分形纹理 | 略带轴向结构感 |
| 计算复杂度 | 相同 | 相同 |

## 输入参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| power | number | 7 | 网格幂次，边长 = 2^power + 1（范围 2~10） |
| roughness | number | 0.5 | 粗糙度（0~1），值越大地形越崎岖 |
| initHeight | number | 0.5 | 四个角点的初始高度（0~1） |
| spread | number | 1.0 | 初始扰动幅度，每层按粗糙度衰减 |
| seed | number | 0 | 随机种子，0 使用默认种子 |

## 输出参数

| 参数 | 类型 | 说明 |
|------|------|------|
| grid | grid | 高度图网格，值域 0~1（连续浮点值，渲染器以单色透明度渐变显示） |

## 算法流程

1. 初始化 (2^power + 1) × (2^power + 1) 网格，设置四角高度
2. 每层迭代：
   - **水平边中点**：取该边两端点的平均值 + 随机扰动
   - **垂直边中点**：取该边两端点的平均值 + 随机扰动
   - **中心点**：取四个边中点的平均值 + 随机扰动
3. 每层迭代后扰动幅度按 `2^(-roughness)` 衰减
4. 所有点计算完成后归一化到 0~1

## 注意事项

- **渲染显示**：输出为连续浮点值网格（0~1），渲染器自动识别为连续值网格，使用单色 + 透明度渐变显示（值越大越不透明），而非多色分类渲染
- `power` 范围 2~10，过大的值（如 10 → 1025×1025）会显著增加内存和计算时间
- 使用确定性 LCG 伪随机数生成器，保证相同种子输出一致
- 与 Diamond-Square 使用相同的 LCG 实现，可直接对比两种算法在相同种子下的差异
