# Value 噪声 (value_noise)

基于 FastNoiseLite 的 Value 噪声生成器，是最简单的格点噪声算法。每个整数格点分配一个随机值，格点之间通过 Hermite 插值平滑过渡。比 Perlin 噪声计算更快，但会产生略微可见的网格痕迹。

## 功能特点

1. **格点随机值噪声**：每个格点一个随机标量，使用三次 Hermite 插值（而非 Perlin 的五次插值），计算成本更低
2. **4 种分形模式**：单层 / FBm / 脊状 / 乒乓，满足不同地形和纹理需求
3. **完整参数控制**：频率、八度数、间隙度、增益、偏移、种子全部可调
4. **确定性输出**：相同参数和种子始终产生相同结果

## 与 Perlin 噪声的区别

| 特性 | Value 噪声 | Perlin 噪声 |
|------|-----------|-------------|
| 格点值 | 随机标量 | 随机梯度向量 |
| 插值方式 | 三次 Hermite (3-2t) | 五次 Hermite (6t⁵-15t⁴+10t³) |
| 计算速度 | 更快 | 较慢 |
| 网格痕迹 | 略可见 | 几乎不可见 |
| 视觉特征 | 柔和的色块过渡 | 平滑连续的梯度 |

## 适用情况

- 对性能要求高、对网格痕迹不敏感的场景
- 生成像素化或色块风格的纹理
- 作为其它电池的快速输入掩码
- PCG 管线中不需要极致平滑度的环节

## 基本使用方法

1. 设置 `width` 和 `height` 确定输出网格尺寸
2. 调整 `frequency` 控制噪声缩放（值越大细节越密）
3. 选择 `fractalType` 和 `octaves` 控制细节层次
4. `grid` 输出值域为 0~1，可直接用于渲染或后续处理

## 输入参数

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| width | number | 128 | 1–1024 | 输出网格宽度 |
| height | number | 128 | 1–1024 | 输出网格高度 |
| frequency | number | 0.02 | - | 噪声采样频率 |
| fractalType | string | "FBm" | 下拉选择 | 分形叠加方式（见下方说明） |
| octaves | number | 4 | 1–8 | 分形叠加层数 |
| lacunarity | number | 2.0 | - | 每层频率倍增系数 |
| gain | number | 0.5 | - | 每层振幅衰减系数 |
| offsetX | number | 0 | - | X 方向采样偏移 |
| offsetY | number | 0 | - | Y 方向采样偏移 |
| seed | number | 1337 | - | 随机种子 |

### fractalType 分形类型

| 值 | 名称 | 效果 |
|----|------|------|
| `None` | 无分形 | 单层 Value 噪声，最简单 |
| `FBm` | 分形布朗运动 | 多层叠加，自然地形效果，最常用 |
| `Ridged` | 脊状 | 取噪声绝对值的反转，产生山脊/裂缝效果 |
| `PingPong` | 乒乓 | 噪声在 0~1 间来回折叠，产生条纹/波浪纹理 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| grid | grid | 噪声网格，值域 0~1 |

## 注意事项

1. **值域映射**：原始 Value 噪声输出约 -1~1，电池内部自动线性映射到 0~1
2. **渲染显示**：输出为连续浮点值网格，渲染器自动识别为连续值网格，使用单色 + 透明度渐变显示
3. **频率选择**：对 128×128 网格，frequency=0.02 约产生 2~3 个噪声周期；frequency=0.05 约 6~7 个周期
4. **分形性能**：octaves 越多计算量线性增长，128×128 网格 8 层仍可即时完成
5. **网格痕迹**：单层模式下可能出现轻微的格栅感，使用分形叠加可有效减弱
6. **算法来源**：核心算法移植自 FastNoiseLite v1.1.1（MIT License, Jordan Peck 2023）
