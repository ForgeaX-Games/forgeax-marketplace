# 波函数坍缩 (wfc_tile_solver)

基于波函数坍缩（Wave Function Collapse）算法的瓦片地图组装器。输入一组 NxN 瓦片模版及其邻接规则，自动生成满足所有约束的拼接地图。

## 功能特点

1. **标准 WFC 流程**：最小熵选择 → 加权随机坍缩 → 弧一致性约束传播，忠实实现经典 WFC 算法
2. **邻近优先启发**：坍缩时优先选择已坍缩邻居旁的格子，使地图从中心向外生长，减少矛盾
3. **自动重试**：遇到矛盾自动更换种子重试，在 maxRetries 次内寻找合法解
4. **模版拼接输出**：WFC 求解完成后自动将模版拼接为完整的输出网格
5. **内置地牢 Demo**：当 templates 和 adjacency 均未输入时，自动使用 16 种 Diablo 风格地牢瓦片（7×7，0=墙壁 1=通道），涵盖 N/S/E/W 四方向的全部 2^4=16 种开口组合，邻接规则为边缘匹配（相邻瓦片在共享边上开口状态一致）

## 适用情况

- 地牢/迷宫地图的程序化生成
- 基于瓦片规则的城市或室内布局组装
- 任何需要满足局部约束的网格拼接问题

## 基本使用方法

**快速体验**：不连接 templates 和 adjacency 输入，直接设置 rows/cols 即可生成地牢地图（使用内置 16 种瓦片 Demo）

**自定义模版**：
1. 准备若干 NxN 瓦片模版（如 7x7 的地牢瓦片），每个为一个二维数字网格
2. 为每个模版定义四方向的邻接规则：`{"N": [...], "S": [...], "E": [...], "W": [...]}`
3. 设置输出网格的行列数
4. 执行后从 `grid` 端口获取拼接好的完整地图

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| templates | array | 内置 Demo | 模版列表，每个元素为 NxN 二维数字网格，所有模版尺寸须一致。不填则使用 16 种地牢瓦片 |
| adjacency | array | 内置 Demo | 邻接规则列表，与模版等长，每个元素为 `{N:[...],S:[...],E:[...],W:[...]}`。不填则自动匹配 |
| rows | number | 8 | WFC 网格行数（1~64） |
| cols | number | 8 | WFC 网格列数（1~64） |
| weights | array | — | 模版权重列表，与模版等长，值越大越常出现，不填则等权 |
| maxRetries | number | 50 | 矛盾时最大重试次数（1~200） |
| seed | number | 0 | 随机种子，0 使用默认 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| grid | grid | 拼接后的完整网格，尺寸为 (rows × 模版高度) × (cols × 模版宽度) |

## 邻接规则说明

邻接规则定义了每个模版在四个方向上可以与哪些模版相邻。索引从 0 开始，对应 templates 列表中的位置。

**示例**：3 个模版的邻接规则

```json
[
  {"N": [0, 1], "S": [0, 2], "E": [1], "W": [2]},
  {"N": [0], "S": [1, 2], "E": [0, 2], "W": [0]},
  {"N": [2], "S": [0], "E": [0, 1], "W": [1]}
]
```

含义：模版 0 的北侧可以放模版 0 或 1，南侧可以放模版 0 或 2，以此类推。

**对称性要求**：如果模版 A 的 `N` 包含模版 B，那么模版 B 的 `S` 也应包含模版 A，否则约束传播可能产生不一致。

## 内置 Demo 说明

当 templates 和 adjacency 均不输入时，自动使用 Diablo 风格地牢瓦片集：

- **16 种瓦片**：N/S/E/W 四方向开口的全部 2^4 种组合，包括全封闭（Fill）和四向全开（NSEW）
- **瓦片尺寸**：7×7，2 格墙壁边框 + 3 格中心房间 + 对应方向的走廊
- **数值**：0 = 墙壁，1 = 通道/房间
- **邻接规则**：边缘匹配 — 相邻瓦片在共享边上的开口状态必须一致（同开或同闭）
- **权重**：2 向通道(4.0) > 3 向(3.0) > 4 向(2.5) > 死胡同(1.5) > Fill(0.3)，偏向生成连通走廊

## 注意事项

1. **模版尺寸一致**：所有模版必须是相同尺寸的二维网格
2. **邻接规则对称**：确保邻接关系双向一致（A 的北邻包含 B ↔ B 的南邻包含 A）
3. **矛盾处理**：如果规则过于严格导致无解，会在 maxRetries 次后返回错误，建议放宽规则或增加重试次数
4. **性能**：行列数过大（如 64x64）配合大量模版时计算较慢，建议先用小网格调试规则
