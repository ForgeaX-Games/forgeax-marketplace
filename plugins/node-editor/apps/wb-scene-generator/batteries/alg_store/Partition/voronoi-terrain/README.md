# Voronoi 地形 (voronoi_terrain)

基于 Voronoi 镶嵌（泰森多边形）生成区域划分网格。每个种子点 `[x, y]` 定义一个多边形区域，自动分配唯一的区域 ID（从 1 开始），支持 Lloyd 松弛使单元分布更均匀，以及平滑插值产生自然的区域过渡。

## 典型应用场景

- **生物群系划分**：每个区域自动获得唯一 ID，用硬边界（smooth=0）产生清晰的群系分界
- **区域/势力划分**：区域 ID 对应阵营或领地编号，用于游戏地图的领地分配
- **地形区块划分**：将地图切分为不规则多边形区块，下游电池可对每个区块分别赋值
- **岛屿/大陆生成**：开启有界模式（bounded=true），多边形被背景包围形成类似 DST 风格的有限区域

## 功能特点

1. **自定义种子点**：可指定 `[x, y]` 种子点列表（绝对坐标）精确控制每个区域的位置，也可随机生成
2. **Lloyd 松弛**：迭代将种子移向其 Voronoi 单元质心，使多边形大小更均匀
3. **平滑过渡**：`smooth > 0` 时使用反距离加权（Shepard）插值，消除硬边界产生自然的区域渐变
4. **有界模式**：`bounded = true` 时多边形不会延伸到地图边缘，而是被背景区域包围，形成有限大小的多边形区域
5. **多种距离度量**：支持欧几里得（圆形单元）、曼哈顿（菱形单元）、切比雪夫（方形单元）
6. **确定性随机**：相同种子产生相同结果，便于调试和复现
7. **区域 ID 输出**：每个种子点按顺序自动获得从 1 开始的唯一 ID，输出网格直接使用该 ID

## 算法原理

### Voronoi 镶嵌

给定一组种子点，Voronoi 镶嵌将平面划分为多个区域，每个区域包含离对应种子点最近的所有点。每个区域被赋予从 1 开始递增的唯一 ID。

### Lloyd 松弛

Lloyd 松弛通过反复执行以下两步使 Voronoi 单元趋于均匀：

1. **计算 Voronoi 划分**：将每个像素分配给最近的种子
2. **移动种子至质心**：将每个种子移到其单元的几何中心

每次迭代后单元形状更接近正多边形，区域 ID 在松弛过程中保持不变。

### 平滑插值

当 `smooth > 0` 时，使用 Shepard 反距离加权插值：

$$h(x, y) = \frac{\sum_i w_i \cdot v_i}{\sum_i w_i}, \quad w_i = \frac{1}{(d_i + \epsilon)^p}$$

其中 $p = 2 / \text{smooth}$，smooth 越大 p 越小，混合范围越广，区域过渡越平滑。

### 有界模式（边界种子点技术）

灵感来自 Don't Starve Together (DST) 的世界生成算法（`WorldSim:WorldGen_VoronoiPass`）。

标准 Voronoi 中，边缘种子点的多边形会无限延伸到地图边界。有界模式通过以下方式解决：

1. **生成边界种子点**：在整个地图上布置一个均匀网格的"隐形"边界种子点，并添加哈希噪声抖动使边界不规则
2. **保护半径过滤**：距离任何真实种子点太近的边界种子点被移除，在每个真实种子点周围留出缓冲区（由 `boundaryGap` 控制）
3. **联合 Voronoi 计算**：真实种子点和边界种子点一起参与 Voronoi 计算
4. **背景标记**：分配给边界种子点的像素填充为 `boundaryValue`

效果是每个真实种子点的多边形被压缩为有限大小，被背景区域包围，形成类似 DST 世界地图中的多边形"岛屿"效果。

```

┌────────────────────────┐       ┌────────────────────────┐
│  A    │     B          │       │ ·  ·  ·  ·  ·  ·  · · │
│       │                │       │ · ╔══╗   ╔══╗         │
│───────┼────────────────│       │ · ║A ║   ║B ║  ·  ·  ·│
│       │                │       │ · ╚══╝   ╚══╝         │
│  C    │     D          │       │ · · ·╔══╗╔══╗· ·  ·  │
│       │                │       │ ·    ║C ║║D ║         │
└────────────────────────┘       │ · · ·╚══╝╚══╝· ·  ·  │
                                 │ ·  ·  ·  ·  ·  ·  · · │
多边形占满整个地图                  └────────────────────────┘

```

## 基本使用方法

1. 设置 `width` 和 `height` 确定网格大小
2. 提供 `seeds` 列表或设置 `numSeeds` 随机生成种子点
3. 调整 `relaxIterations` 控制 Voronoi 单元均匀度
4. 设置 `smooth` 控制边界过渡效果（离散区域划分建议 smooth=0，连续属性建议 smooth>0）
5. 设置 `bounded = true` 使多边形不延伸到边缘，配合 `boundaryGap` 调节大小
6. `grid` 输出中每个区域的值为种子点的 ID（从 1 开始递增）

## 输入参数

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| width | number | 50 | 8–1024 | 网格宽度 |
| height | number | 50 | 8–1024 | 网格高度 |
| seeds | array | [] | - | 种子点 [[x,y],...] x/y 为网格绝对坐标 |
| numSeeds | number | 20 | 2–500 | 随机种子点数（seeds 为空时生效） |
| relaxIterations | number | 3 | 0–50 | Lloyd 松弛迭代次数 |
| distanceType | string | euclidean | euclidean/manhattan/chebyshev | 距离度量 |
| smooth | number | 0 | 0–10 | 平滑度，0=硬边界 |
| bounded | boolean | false | - | 有界模式，多边形不延伸到边缘 |
| boundaryValue | number | -1 | - | 有界模式下背景区域的填充值 |
| boundaryGap | number | 0.8 | 0.1–2.0 | 边界间距（越大多边形越大） |
| seed | number | 0 | - | 随机种子，0=默认 |

### seeds 格式示例

假设网格尺寸为 50×50：

```json
[
  [25, 38],
  [90, 64],
  [64, 102]
]
```

每个元素 `[x, y]`：
- `x`：横坐标，范围 0 ~ width-1（0=左，width-1=右）
- `y`：纵坐标，范围 0 ~ height-1（0=上，height-1=下）
- 每个种子点自动分配区域 ID（从 1 开始递增）

### distanceType 对比

| 距离度量 | 单元形状 | 适用场景 |
|----------|----------|----------|
| euclidean | 自然圆形 | 自然地形、生物群系 |
| manhattan | 菱形/45°旋转方块 | 城市风格、网格化区域 |
| chebyshev | 正方形 | 棋盘风格、方块化区域 |

### boundaryGap 效果对比

| 值 | 效果 |
|----|------|
| 0.2 | 极小保护半径，多边形很小，大部分为背景 |
| 0.5 | 较小多边形，多边形之间有明显间隔 |
| 0.8 | 默认值，类似 DST 的比例，多边形间有适中间隔 |
| 1.2 | 较大多边形，间隔较窄 |
| 2.0 | 极大保护半径，多边形几乎相接，接近无界效果 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| grid | grid | Voronoi 区域网格，值为种子点的 ID（1, 2, 3...），有界模式下背景为 boundaryValue |

## 注意事项

1. **种子坐标为绝对值**：seeds 中的 x/y 坐标直接使用网格绝对坐标（0 ~ width-1, 0 ~ height-1），不做归一化缩放
2. **区域 ID 输出**：每个种子点按在列表中的顺序自动分配 ID（1, 2, 3...），输出网格中对应区域的值即为该 ID
3. **渲染显示**：渲染器根据网格中非零唯一值的数量自动选择渲染模式——若唯一值 ≤16 个（如少量区域），使用多色分类渲染（每个值一种颜色）；若唯一值超过 16 个（如启用 smooth 后产生的连续值），自动切换为单色 + 透明度渐变显示
4. **硬边界 vs 平滑**：smooth=0 产生硬边界（适合区域划分）；smooth=1~2 产生自然渐变（适合高度图、气候分布）；smooth>5 趋近于全局平均
5. **有界模式与平滑**：有界模式下 smooth>0 时，平滑插值仅在真实种子点之间进行，背景区域始终为 boundaryValue，形成清晰的区域-背景分界
5. **有界模式下的背景值**：默认 boundaryValue=-1，与正常的 v 值（通常 ≥0）天然区分，下游电池可通过判断 -1 来识别背景
6. **Lloyd 松弛在有界模式前执行**：松弛只作用于真实种子点，边界种子点在松弛完成后添加
7. **性能**：128×128 + 20 种子点几乎即时完成；有界模式会增加种子点数量，大网格时可能稍慢
8. **与其他电池组合**：输出可通过 blend 电池与 Perlin 噪声等叠加，产生更丰富的细节；有界模式的背景区域可用作 mask
