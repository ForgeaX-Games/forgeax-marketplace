# 连通分量标记 (connected_components)

对二维网格中所有非零区域进行连通分量检测与标记。每个连通分量的格子被赋予唯一正整数 ID（从 1 开始），背景（值为 0 的格子）保持为 0。

## 功能特点

1. **两种连通模式**：支持 4 邻域（上下左右）和 8 邻域（含对角线）
2. **栈式 DFS 标记**：使用显式栈做深度优先搜索逐一标记每个连通分量（结果与 BFS 等价）
3. **ID 从 1 递增**：扫描顺序为从左到右、从上到下，先遇到的分量 ID 更小

## 适用情况

- 统计地图中独立区域（岛屿、房间、碎片）的数量
- 对二值化后的网格进行区域分割
- 配合其他电池做连通性分析（如检测走廊是否将所有房间连通）

## 基本使用方法

1. 将网格连接到 `输入网格` 端口（非零格子视为前景）
2. 从下拉框选择 `连通方式`（4 或 8）
3. 从 `标记网格` 端口获取标记结果，从 `分量数量` 端口获取连通分量总数

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grid | grid | — | 待标记的二维网格，非零格子为前景 |
| connectivity | string | 4 | 连通方式：4=四邻域，8=八邻域（含对角线） |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| grid | grid | 标记后的网格，每个连通分量为唯一正整数ID，背景为0 |
| numComponents | number | 检测到的连通分量总数 |

## 注意事项

1. **非零即前景**：所有非零值的格子都被视为前景参与连通判定，不区分具体数值
2. **原始值不保留**：输出网格中的值为连通分量 ID（1, 2, 3...），而非输入的原始值
3. **4 vs 8 连通**：对角线相邻的格子在 4 连通模式下不算相连，在 8 连通模式下算相连。选择取决于具体场景需求
4. **空网格**：输入空网格时返回错误提示
