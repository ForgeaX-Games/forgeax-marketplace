# 王氏瓦片 (wang_tile)

基于王氏瓦片（Wang Tile）的约束拼接地形生成器。内置 Noita 游戏 `inside.png` 图集的 50 张瓦片（25 水平 + 25 垂直）作为默认模版，支持自定义瓦片和边缘标签匹配。

## 输出语义

| 值 | 含义 |
|----|------|
| 0 | 墙壁（实体/岩石） |
| 1, 2, 3, ... | 不同的连通区域 ID |

**区域划分**：所有非墙壁像素按连通性（上下左右四连通）自动分组，每个连通分量分配唯一递增 ID。

## 两种输出

| 输出 | 说明 | 用途 |
|------|------|------|
| `grid` | 0=墙壁，1,2,3...=区域 ID | 分区，知道每个像素属于哪个独立区域 |
| `valueGrid` | 0=墙壁，1=通道，2=地形 | 材质类型，知道每个像素的材质属性 |

两者结合可以实现：「区域 3 中所有地形类型的像素」这样的查询。

## 功能特点

1. **内置 Noita 瓦片**：从 `inside.png` 图集提取并缩小到 25 张水平瓦片（20×10）和 25 张垂直瓦片（10×20），保留复杂洞穴纹理
2. **自动边缘匹配**：通过比较瓦片四条边的像素签名推导邻接规则
3. **连通区域标记**：放置瓦片后自动执行 flood-fill 生成区域 ID
4. **贪心放置**：从左到右、从上到下，速度快、无回溯

## 内置瓦片

从 Noita `inside.png` 图集提取（内部编码：**0=墙壁/实体背景，1=通道，2=地形**）：

- **水平瓦片（默认）**：25 张，尺寸 20×10 像素
- **垂直瓦片**：25 张，尺寸 10×20 像素
- 通过 `defaultSet` 参数选择（默认 `"horizontal"`）

## 基本使用

**快速体验**：不连接 `tiles` 输入，设置 `rows`/`cols` 即可生成复杂洞穴地图

**切换垂直瓦片**：设置 `defaultSet` 为 `"vertical"`

**自定义瓦片**：
1. 准备同尺寸的二维数字网格瓦片（**0=墙壁/实体背景，1=通道，2+=地形**，与内置编码一致）
2. 确保希望相邻的瓦片在共享边上像素完全一致
3. 连接到 `tiles` 输入

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| tiles | array | 内置瓦片 | 瓦片模版数组，每个元素为二维数字网格 |
| edgeLabels | array | — | 边缘标签 `[{N,S,E,W}, ...]`，提供后使用标签匹配 |
| defaultSet | string | horizontal | 内置瓦片组：`horizontal`（20×10）或 `vertical`（10×20） |
| rows | number | 8 | 瓦片网格行数（1~64） |
| cols | number | 8 | 瓦片网格列数（1~64） |
| weights | array | — | 权重数组，值越大越常出现 |
| seed | number | 0 | 随机种子 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| grid | grid | 分区网格：0=墙壁，1,2,3...=连通区域 ID |
| valueGrid | grid | 材质网格：0=墙壁，1=通道，2=地形 |
| regionCount | number | 连通区域总数 |
| numTiles | number | 放置的瓦片总数 |
