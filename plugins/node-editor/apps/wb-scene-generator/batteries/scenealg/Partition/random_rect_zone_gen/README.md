# 随机矩形区域生成 (random_rect_zone_gen)

在输入网格的目标区域内随机摆放若干互不重叠的轴对齐矩形地块；每个地块输出为一张独立的 0/1 网格（地块覆盖的格子=1，其余=0），与输入网格同形状。

## 适用情况

- 在已划好的大区域 mask 上撒子地块（房间、用地、聚落用地等）
- 任何"在目标区域内随机切若干矩形"的 PCG 流程
- 配合 autoIterate：上游给一组 mask 时自动 fanout，每张 mask 都跑一次本电池

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| grid | grid | - | 输入二维网格；矩形只能落在有效区域格内 |
| targetValue | number | 0 | 0 视为"任意非零格"（即把 1 当作区域特征函数）；非 0 时精确匹配该掩码 ID（适用于异形/多值区域） |
| count | number | 5 | 想要放置的矩形总数；若空间不足，实际数量可能少于此值 |
| minSize | number | 13 | 矩形宽和高的最小值（格数） |
| maxSize | number | 19 | 矩形宽和高的最大值（格数） |
| minDistance | number | 2 | 相邻矩形最近边之间的最小格距，0 表示允许紧靠（仍不重叠） |
| dispersion | number | 0.6 | 0=随机集中，1=尽量分散；越大新矩形越倾向放置在远离已有矩形的位置 |
| seed | number | 0 | 随机种子，0 使用当前时间 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| outputGridList | grid (rank=1) | 每个地块对应一张独立的 0/1 网格列表 |
| placedCount | number | 实际成功放置的矩形数量 |

## 算法要点

1. **PRNG**：内置 64-bit LCG，可由 seed 复现
2. **候选锚点**：扫一遍输入 grid，把所有合格格子（按 targetValue 规则）放入候选池
3. **采样锚点**：dispersion=0 时均匀随机；dispersion=1 时按"到最近已放矩形中心的曼哈顿距离"加权抽，越远权重越高；中间值在两者之间线性插值
4. **放置矩形**：在 [minSize, maxSize] 内独立采样宽高，左上角随机偏移到锚点附近，验证 ① 不越界 ② 与已放矩形最近边距 ≥ minDistance ③ 覆盖格全是合格目标格 ④ 没被其他矩形占过；通过则在新建空 grid 上把覆盖格置 1
5. **重试**：每个矩形最多 1000 次尝试；放不下就跳过该次

## 注意事项

- **`targetValue=0` 默认= "在非零格里放"**（不是"在 0 格里放"）。0-1 输入下，"非零"等价于"标记为 1 的特征函数区域"，行为符合直觉。
- 输出 grid 与输入 grid **同形状**，只在地块覆盖的格子置 1。多个地块叠加可以用下游电池（如 batch_max_merge / 网格相加）合成单张 mask。
- 上游传 grid 列表时由 autoIterate fanout，每张 grid 各自独立放矩形；不需要在本电池里处理双形态输入。
