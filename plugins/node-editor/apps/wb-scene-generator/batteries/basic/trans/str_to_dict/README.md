# 字符串转字典 (str_to_dict)

将形如 `{"key":"value",...}` 的 JSON 字符串解析为字典对象。

## 功能特点

1. **标准 JSON 对象解析**：支持符合 JSON 规范的对象字符串，如 `{"name":"张三","age":18}`
2. **嵌套结构保留**：字典中的嵌套对象、数组等复杂结构完整保留
3. **多层嵌套兼容**：支持被多次 `JSON.stringify` 的字符串（最多 3 层解包）
4. **健壮的错误处理**：输入非法时返回空字典和错误说明，不会抛出异常

## 适用情况

- 将 AI 模型输出的 JSON 对象字符串转为可用字典
- 在节点流中进行字符串与字典类型之间的格式转换
- 解析配置、参数等以字符串形式传递的键值对数据

## 基本使用方法

1. 将包含 JSON 对象的字符串连接到 **输入字符串** 端口
2. 执行节点，**字典** 端口输出解析后的字典

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| str | string | - | 形如 `{"key":"value",...}` 的 JSON 字符串 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| dict | dict | 解析后的键值对字典 |

## 使用示例

### 输入示例

```json
{
  "str": "{\"name\":\"张三\",\"age\":18,\"city\":\"北京\"}"
}
```

### 输出示例

```json
{
  "dict": {
    "name": "张三",
    "age": 18,
    "city": "北京"
  }
}
```

## 注意事项

1. **输入必须为合法 JSON 对象**：字符串内容必须符合 JSON 语法，键名需用双引号包裹
2. **不接受数组输入**：若 JSON 解析结果是数组（如 `["a","b"]`），返回空字典并附带 error 字段；如需解析数组请使用「字符串转列表」电池
3. **值类型完整保留**：字典中数字、布尔值、嵌套对象等类型均原样保留，不会被转为字符串
