# 字符串列表爆炸 (list_explode)

输入一个 JSON 数组字符串，将其中每个元素拆分为独立的动态输出端口，端口数量与列表长度一致。

## 功能特点

1. **动态端口数量**：输出端口数量由列表长度决定，执行后自动生成 `[0]`、`[1]`、`[2]`… 等端口
2. **元素原样透传**：字符串元素直接输出；非字符串元素（数字、布尔等）会转为字符串后输出
3. **内置 JSON 解析**：内部自动将字符串解析为数组，支持最多 3 层嵌套字符串化的 JSON，无需前置 `str_to_list`

## 适用情况

- 将 LLM 返回的 JSON 数组字符串逐条拆分输出
- 将批量生成的字符串化列表（如路径列表、名称列表）分发给各自独立的下游节点
- 任何需要把 JSON 数组字符串"一拆多"为独立值传递的场景

## 基本使用方法

1. 将形如 `["a","b","c"]` 的 JSON 数组字符串连接到 `list` 端口
2. 执行节点，动态输出端口 `[0]`、`[1]`… 自动生成
3. 将各端口分别连接到下游节点

> **注意**：请先执行一次以生成真实端口，再进行连线。执行前的灰色占位端口不可连线。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| list | string | — | 形如 `[...]` 的 JSON 数组字符串 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| item_0 | any | 列表第 0 个元素，类型与 JSON 中的元素类型一致 |
| item_1 | any | 列表第 1 个元素 |
| item_N | any | 动态扩展，N 与列表索引对应 |

## 注意事项

1. **先执行再连线**：动态输出端口在执行前为灰色占位，执行后才生成真实端口
2. **元素类型保留**：输出类型为 any，JSON 中的字符串、数字、布尔、对象、数组均原样输出，不做类型转换
3. **端口数量变化**：若两次执行列表长度不同，连接到已消失端口的连线会自动断开
