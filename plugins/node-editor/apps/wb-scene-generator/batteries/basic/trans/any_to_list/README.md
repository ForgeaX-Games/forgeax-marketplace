# 任意值转列表 (any_to_list)

将多个任意类型的输入值按顺序收集为一个列表，支持动态多端口。

## 功能特点

1. **动态多端口**：连接最后一个槽位后自动追加新槽，按需扩展输入数量
2. **任意类型**：每个输入可以是不同类型（数字、字符串、对象、网格等）
3. **顺序收集**：按端口序号 item_0, item_1, … 的顺序组成列表
4. **跳过未连接**：未连接的槽位（值为 undefined）自动跳过，不产生空洞

## 适用情况

- 需要把多个独立值打包成列表传给下游
- 配合「列表单次取值」电池使用，先打包再按需取出
- 将不同来源的数据汇集为统一的列表格式

## 基本使用方法

将任意值依次连接到 `值0`、`值1`… 端口，执行后 `list` 端口输出按序收集的列表。连接最后一个槽位后自动出现新槽。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| item_0 | any | — | 第0个输入值 |
| item_1 | any | — | 第1个输入值；连接后自动追加新槽位 |
| item_N | any | — | 动态追加的后续槽位 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| list | array | 所有已连接输入值按序组成的列表 |

## 注意事项

1. **未连接槽跳过**：未连接的槽位不会在列表中产生 `null` 或空位，直接跳过
2. **顺序即序号**：列表顺序由端口序号决定（item_0 → 列表第0项），与连线顺序无关
3. **类型混合**：列表中允许存在不同类型的元素，下游需自行处理类型兼容性
