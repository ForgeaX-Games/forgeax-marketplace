# 字符串转Grid (str_to_grid)

将形如 `[[1,2],[3,4]]` 的 JSON 字符串解析为 grid（二维整数数组）。

## 功能特点

1. **直接脱壳**：将字符串形式的 grid 还原为真正的 grid 类型，可直接接入其他 grid 输入端口
2. **多层编码兼容**：支持被多次 `JSON.stringify` 编码的字符串（自动解包最多 3 层）
3. **格式校验**：自动验证解析结果是否为合法 grid，非法输入返回明确错误信息

## 适用情况

- 上游节点输出的 grid 以字符串形式传递，需要还原为 grid 类型再继续处理
- AI 节点返回了包含 grid 的 JSON 字符串，需要提取出来连接到下游

## 基本使用方法

将包含 grid 数据的字符串连接到 `str` 输入端，执行后 `grid` 输出端返回解析好的二维整数数组。

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| str | string | — | 形如 `[[1,2],[3,4]]` 的 JSON 字符串 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| grid | grid | 解析后的二维整数数组 |

## 注意事项

1. **必须是合法 JSON**：输入字符串必须能被 JSON 解析为二维数组，否则返回 `error`
2. **必须是纯数字二维数组**：每一行、每个格子都必须是有限数字，含字符串或 null 的数组不被接受
3. **顶层必须为 grid**：字符串解析后的顶层值必须直接是 grid，若 grid 嵌套在对象内请先使用其他解析电池提取
