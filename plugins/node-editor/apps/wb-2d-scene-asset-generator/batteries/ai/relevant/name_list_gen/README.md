# 批量生成名称清单 (name_list_gen)

根据数量和基础名称，批量生成格式为「名称N」的名称清单，输出与 `districts` 格式兼容的条目数组。

## 功能特点

1. **批量生成**：输入数量 N，一次生成 N 条名称条目，无需手动逐条填写
2. **有序编号**：名称格式固定为 `{基础名称}{序号}`，序号从 1 开始，id 与序号一致
3. **districts 兼容**：输出的 `nameList` 与城市地图的 `districts` 字段格式一致，可直接接入下游管线
4. **默认值兜底**：数量为 0 时输出空数组，名称为空时使用「区域」作为前缀

## 适用情况

- 为按连通性拆分的多个区域批量命名（配合 `grid_split_by_connectivity` 使用）
- 快速生成建筑群、房间、地块等有规律命名的名称清单
- 替代手动输入名称列表，提升管线搭建效率

## 基本使用方法

1. 在 `数量` 端口填写要生成的条目数（如 5）
2. 在 `基础名称` 端口填写名称前缀（如「建筑」）
3. 执行后，`名称清单` 输出 `[{id:1, name:"建筑1"}, {id:2, name:"建筑2"}, ...]`

## 输入参数

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| count | number | 1 | 要生成的名称条目数量 |
| name | string | 区域 | 名称前缀，生成结果格式为「名称N」 |

## 输出参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| nameList | array | 名称条目数组，格式为 `[{id, name}, ...]` |

## 使用示例

### 输入

```
count = 3
name  = 建筑
```

### 输出

```json
[
  {"id": 1, "name": "建筑1"},
  {"id": 2, "name": "建筑2"},
  {"id": 3, "name": "建筑3"}
]
```

## 注意事项

1. **序号从 1 开始**：id 和名称后缀均从 1 起步，不从 0 开始
2. **id 与序号一致**：id 值直接等于序号，使用前请确认与下游网格的掩码值匹配
3. **数量为 0**：输出空数组 `[]`，不报错
